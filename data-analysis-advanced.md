## 一、分析方法体系

### 1.1 因果推断方法（核心）

| 方法 | 使用场景 | 核心假设 | 失效信号 |
|---|---|---|---|
| **PSM（倾向得分匹配）** | 观察性数据中验证指标因果关系，无法开AB时 | 无未观测混杂（Unconfoundedness）；重叠假设（Overlap） | 倾向得分分布无重叠；匹配后协变量不平衡 |
| **DML（双重机器学习）** | 高维混杂变量、连续型处理变量的效应估计 | 样本量足够大； nuisance 函数可一致估计 | 样本量 < 10k； nuisance 过拟合 |
| **DID/面板FE** | 消除pre-AA基线差异，实验收益更准确估计 | 平行趋势假设；处理组与对照组趋势一致 | 事件前趋势明显分叉；处理时机非外生 |
| **Uplift 模型** | 异质性处理效应，识别策略真正受益人群 | SUTVA；处理可随机化 | 处理间存在溢出效应；样本在干预组间迁移 |
| **短期替代指标→长期效应** | 实验期内无法观测长期指标（LT30）时 | 短期指标与长期指标稳定相关 | 短期-长期关系不稳定（如季节性） |
| **Causal Impact** | 单时间序列、无对照组的策略效果评估 | 干预前序列可建模；干预后无其他重大变化 | 干预同期发生其他事件；序列非平稳 |
| **反转实验** | 无法直接开AB时，用"撤策略"估计存量价值 | 撤策略无滞后效应；用户行为可逆 | 撤策略后用户永久流失；存在网络效应 |

### 1.2 机器学习建模方法

| 方法 | 使用场景 | 核心假设 | 失效信号 |
|---|---|---|---|
| **XGBoost + SHAP** | 特征重要性排序，理解哪些因素驱动用户行为 | 特征可观测；样本有代表性 | 特征间强共线性；分布漂移 |
| **GBDT+LR（XGB-LR）** | 高维特征 + 强可解释性需求 | GBDT 特征变换后线性可分 | 变换后特征仍高度非线性 |
| **WOE/IV分箱** | 单变量预测能力评估，特征筛选 | 目标变量二分类；单调关系 | 特征与目标非单调；分箱边界敏感 |
| **ALE / SHAP Interaction Value** | 分析特征在不同用户群上的非线性效应 | 特征间存在交互；样本覆盖交互区域 | 交互区域样本稀疏；ALE 计算窗口过大 |

### 1.3 统计分析方法

| 方法 | 使用场景 | 核心假设 | 失效信号 |
|---|---|---|---|
| 相关性系数分析 | 快速验证变量间关联方向和强度 | 线性关系；无异常值 | 关系非线性；存在极端值 |
| IV值分析（单变量） | 评估每个特征对目标变量的预测贡献 | 二分类目标；单调趋势 | 特征与目标关系复杂非单调 |
| 分桶/分层对比 | 对连续变量分桶，观测各桶指标差异 | 分桶边界合理；桶内同质 | 分桶过粗掩盖差异；边界选择偏误 |
| SHAP值分析 | 模型层面特征贡献量化，全局和局部解释 | 模型已收敛；特征可解释 | 模型欠拟合；特征为 ID 类无意义 |

### 1.4 方法选择逻辑

```
有AB实验数据
├── Pre-AA基线有差异 → DID校正
├── 正常分析 → t检验 / Bootstrap
├── 提升灵敏度 → 后分层 / CUPED
└── 估计异质性效应 → Uplift建模

没有AB实验，用观察性数据
├── 有对照组、有面板数据 → DID / 固定效应
├── 有可观测混杂变量 → PSM
├── 处理变量连续 + 高维混杂 → DML
├── 无对照组、时序数据 → Causal Impact
└── 目标是找重要特征/trigger → XGBoost + SHAP + IV

需要解释模型为什么这么预测
├── 全局特征重要性 → SHAP均值条形图
├── 特征值方向效应 → SHAP小提琴图
├── 特征非线性效应 → ALE图
└── 特征交互效应 → SHAP Interaction Value
```

## 二、方法假设与失效条件

### 2.1 因果推断方法假设

| 方法 | 关键假设 | 如何检验 | 失效后替代方案 |
|---|---|---|---|
| **PSM** | 1. 无未观测混杂<br>2. 重叠假设（所有处理组个体都有对照组匹配） | 1. 匹配后 SMD < 0.1<br>2. 倾向得分分布图 | 敏感性分析（Rosenbaum bounds）；工具变量 |
| **DID** | 平行趋势：若无干预，处理组与对照组趋势一致 | 事件前趋势图；安慰剂检验 | 合成控制法；双向固定效应 + 事件研究 |
| **DML** | 1. 样本量足够<br>2. Nuisance 函数可一致估计 | 交叉验证 MSE；样本量检查 | 传统半参方法；降维后线性模型 |
| **Uplift** | SUTVA（个体处理不受他人影响） | 检查处理组间溢出；网络效应检验 | 网络干扰模型；聚类随机化 |
| **Causal Impact** | 1. 干预前序列可建模<br>2. 干预后无其他重大变化 | 干预前预测误差；同期事件排查 | 合成控制；多时间序列模型 |

### 2.2 机器学习方法假设

| 方法 | 关键假设 | 如何检验 | 失效后替代方案 |
|---|---|---|---|
| **XGBoost + SHAP** | 1. 特征可观测<br>2. 样本有代表性 | 分布稳定性监控；特征重要性稳定性 | 加入领域知识特征；迁移学习 |
| **WOE/IV** | 1. 二分类目标<br>2. 单调关系 | IV 值稳定性；分箱单调性检查 | 非单调分箱（如决策树分箱）；连续目标转换 |
| **ALE** | 特征间存在交互；样本覆盖交互区域 | ALE 曲线平滑度；置信区间宽度 | 局部线性近似；子群分析 |

## 三、实验设计前置知识

### 3.1 何时该开 AB 实验

| 场景 | 决策 |
|---|---|
| 策略可随机化；样本量充足；核心指标可观测 | **开 AB** |
| 策略影响全局（如定价算法）；样本间存在干扰 | **不开**，用反转实验或准实验 |
| 样本量不足（< 最小样本量） | **不开**，用观察性分析或扩大实验范围 |
| 长期效应为主，短期指标不稳定 | **开短期 AB + 长期追踪**，或替代指标法 |
| 伦理/业务限制无法随机化 | **不开**，用 PSM/DID/Causal Impact |

### 3.2 实验设计要素

| 要素 | 说明 | 常见错误 |
|---|---|---|
| **随机化单元** | 用户/设备/会话/页面？ | 用户级策略用会话级分流 → 污染 |
| **目标人群** | 全量用户还是子群？ | 人群定义不清 → 结果不可推广 |
| **处理定义** | 实验组具体体验是什么？ | 处理与对照差异模糊 → 归因困难 |
| **指标体系** | 核心指标 + 护栏指标 + 局部指标 | 只看核心指标 → 忽略副作用 |
| **实验周期** | 多久？是否覆盖完整行为周期？ | 周期过短 →  novelty effect 未消退 |
| **分流比例** | 实验组 vs 对照组比例？ | 比例极端 → 方差增大 |

### 3.3 最小样本量与检验力

**核心公式（比例指标）：**

```
n = 16 * σ² / δ²
```

- `σ²`：指标方差
- `δ`：期望检测的最小效应量（MDE）
- 检验力 80%，显著性水平 5%

**实际步骤：**

1. 确定核心指标类型（比例/均值/比率）
2. 估计指标基线值和方差（历史数据）
3. 确定业务可接受的最小效应量（MDE）
4. 计算每组最小样本量
5. 根据日活估算实验天数
6. 考虑分流比例和多重检验调整

**检验力分析（Power Analysis）：**

| 参数 | 含义 | 调整方式 |
|---|---|---|
| 显著性水平 α | 假阳性率 | 降低 α → 样本量增大 |
| 检验力 1-β | 真阳性率 | 提高检验力 → 样本量增大 |
| MDE | 最小可检测效应 | 降低 MDE → 样本量增大 |
| 指标方差 σ² | 数据波动 | 降低方差（CUPED/后分层）→ 样本量减小 |

### 3.4 多重检验校正

| 场景 | 方法 | 适用条件 |
|---|---|---|
| 少量检验（< 10） | Bonferroni | 保守；检验间独立或正相关 |
| 中等检验（10-100） | Holm-Bonferroni | 比 Bonferroni 稍宽松；控制 FWER |
| 大量检验（> 100） | Benjamini-Hochberg (FDR) | 控制假发现率；适合探索性分析 |
| 指标间相关性强 | 分层检验 / 闭合检验 | 利用指标结构减少惩罚 |

**AB 实验中的多重检验：**

- 多个指标 → 核心指标优先，护栏指标单独控制
- 多个子群分析 → 预先定义子群，事后分析声明为探索性
- 多次 peeking → 用序贯检验（如 O'Brien-Fleming）或固定分析点

## 四、因果推断 × 机器学习交叉方法

### 4.1 因果森林（Causal Forest）

**场景：** 估计异质性处理效应，同时处理高维协变量。

**原理：** 在随机森林框架中，分裂准则改为最大化处理效应异质性（如 `τ(x) = E[Y(1) - Y(0)|X=x]` 的方差）。

**优势：**
- 自动发现异质性子群
- 不需要预先指定交互项
- 置信区间可通过聚类自助法获得

**局限：**
- 需要随机化或已知倾向得分
- 样本量要求较高（> 10k）
- 树深度和叶子节点数需调参

**实现：** `grf` 包（R）；`econml` 库（Python）

### 4.2 双重稳健学习（Doubly Robust Learning）

**场景：** 观察性数据中，既要估计处理效应，又要处理高维混杂。

**原理：** 结合倾向得分模型和结果回归模型，只要其中一个正确，估计即一致。

```
估计量 = 结果回归部分 + 倾向得分加权残差部分
```

**优势：**
- 双重稳健性：一个模型错，另一个补救
- 比 PSM 更有效地利用数据
- 可扩展至连续处理变量

**局限：**
- 两个模型都错 → 偏差放大
- 高维场景下 nuisance 函数估计困难
- 置信区间构造复杂

**实现：** `econml` 中的 `DoublyRobustLearner`

### 4.3 Meta-Learners

**场景：** 用现有机器学习模型估计条件平均处理效应（CATE）。

| 类型 | 原理 | 适用条件 |
|---|---|---|
| **S-Learner** | 单一模型，处理变量作为特征输入 | 处理效应同质或弱异质；样本量中等 |
| **T-Learner** | 对照组和处理组分别建模，预测差值 | 处理效应异质；两组样本都充足 |
| **X-Learner** | 两阶段：先交叉估计反事实，再加权组合 | 处理组与对照组样本量不平衡 |
| **R-Learner** | 基于 Robinson 分解，直接优化 CATE | 高维场景；需要较好的 nuisance 估计 |

**选择建议：**
- 样本量小 → S-Learner（减少方差）
- 处理组对照组样本均衡 → T-Learner
- 样本极不平衡（如 1:10）→ X-Learner
- 高维 + 有较好 nuisance 估计 → R-Learner

**实现：** `econml` 库提供全部实现

## 五、分析文档结构规范

### 5.1 标准文档结构

```
# [年月]-[分析主题]

## 背景
（1-2段，说清楚为什么要做这个分析）

## 主要数据结论 / 核心数据结论
（最重要的章节，放在最前面，先总后分）

## 建模过程 / 分析思路 / 数据详情
（方法说明 + 详细数据）
 - 变量选择
 - 建模/分析过程
 - 结果数据

## 落地方案 / 建议（可选）
（如有业务落地，列举具体方案）

## 附录
（数据口径说明、脚注等）
```

### 5.2 各章节规范

**① 背景章节**
- 引用历史文档链接，说明"前置结论"和"未解决的问题"
- 明确研究问题是什么（非描述分析内容）
- 格式：`前序分析[文档名](链接) 发现xxx，因此本文旨在xxx`

**② 核心结论章节（最重要）**
- 必须放在方法/数据详情之前（金字塔原理）
- 每条结论加粗，以数字或Bullet列出
- 结论后紧跟具体数字（如`**+0.164%**`、`**1.85倍**`）
- 结论末尾附落地建议：`建议xxx（已排期xx月）`
- 非严格AB必须声明局限性：`（注：当前结论基于观察性数据统计的相关性，而非因果性）`

**③ 分析思路/建模过程章节**
- 用表格列举：问题定义、样本、目标变量、自变量
- 模型效果必须报告：AUC、准确率、精确率、召回率、F1
- 特征重要性用SHAP小提琴图 + 均值条形图双图展示

## 六、分析逻辑框架

### 6.1 问题发现→假设→验证→落地

```
1. 问题发现：在日常监控/前序分析中发现 [指标异常/增长乏力/用户行为规律]
2. 假设提出：猜测原因是 [XXX]
3. 量化验证：通过 [方法论] 验证假设，得到 [具体数字结论]
4. 异质性拆分：进一步下钻，分 [行业/用户类型/精泛搜/本异地等] 观测差异
5. 落地建议：基于分析结论，给出 [算法侧/产品侧] 的具体优化方向
```

### 6.2 AB 实验分析逻辑链

```
1. 确认实验设计合理（分流验证 + Pre-AA检验）
2. 报告核心指标（QV/QVCTR/访购率）的绝对/相对变化
3. 用CUPED/后分层/DID提升灵敏度
4. 稳健性检验（去极值/换口径/子群一致性）
5. 归因分析（收益来自哪个用户群/场景）
6. 长期效应评估（如有替代指标方法）
```

### 6.3 常用观测维度分层

| 维度 | 典型分组 |
|---|---|
| **用户类型** | 搜索老用户 / 搜索新用户-唤起 / 搜索新用户-非唤起 |
| **精泛搜** | 精搜（Poi搜索）/ 泛搜（类目/Place等） |
| **本异地** | 本地 / 异地-行前 / 异地-行中 |
| **行业** | 到餐 / 服务零售 / 酒旅 / 电影演出赛事 / 医药健康 |
| **活跃度** | 全勤 / 高活 / 中活 / 低活 / 新回 |

## 七、结论描述规范

### 7.1 数字表达

| 指标类型 | 描述方式 | 示例 |
|---|---|---|
| 百分比变化（AB实验） | `+X.XXpp`或`+X.XX%` | QV访购率 +0.031pp |
| 相对变化 | `+X.XX%（相对）` | QV +0.164% |
| 绝对数字对比 | 列出两个数 | 高集中度5.22% vs 低集中度2.82% |
| 倍数关系 | `约X.X倍` | 高/低访购率差距约1.9倍 |
| 相关系数 | 精确到4位小数 | r = 0.0442 |
| 模型效果 | AUC + 准确率 + F1全报 | AUC:0.82，准确率:0.78，F1:0.70 |

### 7.2 结论语气

**强结论（AB实验支撑）：**
> `策略上线后，搜索QV +0.164%，QV访购率 +0.031pp（均显著）`

**弱结论（相关性/观察性数据）：**
> `基于观察性数据，高集中度query的访购率约为低集中度的1.9倍（相关性，非因果）`

**主动声明局限性：**
> `注：当前结论基于观察性数据统计的相关性，而非因果性。如要严格验证增量价值，需上线屏蔽策略实验。`

### 7.3 加粗使用规范

- **核心结论句必须加粗**
- 关键数字加粗：`**+0.164%**`、`**1.9倍**`
- 重要名词加粗
- **不滥用**：非关键信息不加粗

---

## 八、落地建议的颗粒度

### 8.1 算法侧

| 分析结论类型 | 具体建议方向 | 示例 |
|---|---|---|
| 特征重要性排序 | 模型迭代方向 | "Top3 特征为历史访购次数、搜索频次、类目偏好，建议在下版模型中增加时序特征" |
| 特征非线性效应 | 特征工程优先级 | "价格敏感度呈倒U型，建议分桶处理而非连续值输入" |
| 异质性处理效应 | 子群模型策略 | "高活用户处理效应为负，建议对高活用户单独建模或排除策略触达" |
| 模型误差模式 |  bad case 分析方向 | "模型在异地用户上误差大，建议增加地理位置特征或分区域建模" |

### 8.2 产品侧

| 分析结论类型 | 具体建议方向 | 示例 |
|---|---|---|
| 策略效果显著 | 策略阈值与放量节奏 | "实验组访购率+0.5pp且护栏指标无异常，建议全量放量，灰度3天→7天→全量" |
| 策略效果异质 | 用户触达分层 | "新用户效应+1.2pp，老用户-0.1pp，建议仅对新用户生效" |
| 短期指标提升但长期不确定 | 实验周期与追踪方案 | "短期 CTR +2%，但 LT7 无数据，建议延长实验至14天并埋点追踪留存" |
| 策略有负向副作用 | 产品形态调整 | "访购率提升但客单价下降，建议增加凑单引导或满减门槛" |

## 九、写作反模式

| 反模式 | 正确做法 |
|---|---|
| 结论放在文档最后 | 核心结论放在背景后、方法前 |
| 只报相关性不说是否因果 | 主动声明结论的因果性边界 |
| 只说"更高"不给具体数字 | 永远带数字：高5.22% vs 低2.82% |
| 不给落地建议 | 每条分析结论都附"建议xxx" |
| 模型只报AUC | 同时报准确率/精确率/召回率/F1 |
| SHAP只放条形图 | 同时放小提琴图（看方向和分布） |
| 堆砌数字无总结 | 先总结再细节（金字塔原理） |
| 方法不声明假设 | 每个方法附假设检验和失效信号 |
| 实验不报告设计细节 | 文档开头说明随机化单元、分流比例、实验周期 |
| 落地建议空泛 | 区分算法侧/产品侧，给出具体迭代方向 |

## 十、学习路径建议

### 阶段一：基础夯实（2-3周）

1. **因果推断基础**
   - 潜在结果框架（Rubin Causal Model）
   - 随机化实验的黄金标准
   - 观察性数据的挑战：混杂、选择偏误

2. **统计基础**
   - 假设检验：t检验、卡方检验、Bootstrap
   - 回归分析：OLS、Logistic、固定效应
   - 检验力分析与样本量计算

3. **AB 实验全流程**
   - 实验设计 → 分流 → 分析 → 决策
   - Pre-AA 检验、CUPED、后分层
   - 多重检验问题

### 阶段二：方法进阶（3-4周）

1. **因果推断进阶**
   - PSM：匹配方法、重叠检验、敏感性分析
   - DID：平行趋势检验、事件研究法、合成控制
   - DML：Nuisance 函数估计、交叉验证

2. **机器学习建模**
   - XGBoost 原理与调参
   - SHAP 值计算与可视化
   - ALE 图与特征交互

3. **交叉方法**
   - 因果森林：异质性处理效应
   - Meta-Learners：S/T/X/R-Learner
   - 双重稳健学习

### 阶段三：实战整合（持续）

1. **文档写作**
   - 按本规范撰写完整分析文档
   - 每篇文档自检：假设声明、数字、落地建议

2. **案例复盘**
   - 选取 3-5 个历史分析，用新框架重构
   - 识别当时的方法误用和表述问题

3. **持续迭代**
   - 关注因果推断前沿（如双重差分的最新发展）
   - 根据业务场景补充特定方法（如网络干扰、动态处理效应）
