# CUPED方差缩减

> **本文约定**：pp = 百分点（绝对变化），% = 百分比（相对变化）。例如：从10%提升至12%，绝对提升为+2pp，相对提升为+20%。

## 问题引入

假设你是一家B端SaaS平台的数据分析师。公司想评估"运营顾问对接服务"对商家留存的影响，但总共只有300个商家，每组150人。实验跑了两个月，留存率差异是+5pp，但p=0.18——不显著。业务方很沮丧："明明看起来有效果，为什么统计上说'不能确定'？"

问题出在方差。商家留存率本身波动很大，150人的样本下，随机噪声淹没了真实的+5pp效应。如果样本量翻倍到600人，可能就能检测到。但B端场景下，用户增长慢、获取成本高，"增加样本"往往不是可选项。

这时候你想：这些商家在实验前的行为数据，能不能帮上忙？如果某个商家实验前30天的活跃度，能预测他实验期间的留存概率，那么"实验前活跃度"就包含了关于"留存"的大量信息。如果我们能在分析时"利用"这个信息，把"能被实验前数据解释掉的波动"剔除掉，剩下的就是更"纯净"的实验效应。这就像一个降噪耳机：保留信号，滤除噪声。

CUPED（Controlled-experiment Using Pre-Experiment Data）就是做这个的。它用实验前同一指标的历史值作为协变量，对实验期间的指标做线性调整，从而缩减方差、提升检验灵敏度。如果实验前人均GMV与实验期间人均GMV的相关系数是0.7，CUPED可以把方差缩减约50%——相当于样本量翻倍的效果。

但CUPED不是万能药。它要求有可靠的历史数据（新用户实验用不了），要求协变量与结果有足够相关性（ρ<0.5时效果有限），要求历史数据未被处理污染（如果实验前就开始灰度，协变量本身已被干预）。在[[stage-01-foundation/02-statistical-basics/07-power-analysis/检验力分析与样本量计算|功效分析]]中，我们知道样本量、效应量和方差三者互相牵制；CUPED提供了一条"不增加样本，只减少方差"的捷径，但这条捷径有明确的通行条件。

## 基本原理

CUPED（Controlled-experiment Using Pre-Experiment Data）由微软于2013年提出，核心思想：利用实验前的协变量（通常是同一指标的历史值）来降低实验指标的方差，从而提升检验灵敏度或减小所需样本量。

**核心公式：**
$$\tilde{Y} = Y - \hat{\theta}(X - \bar{X})$$

其中：
- $Y$：实验期间指标
- $X$：实验前同一指标（协变量）
- $\bar{X}$ 为样本均值，$\hat{\theta}$ 基于对照组数据估计
- $\hat{\theta} = Cov(Y,X) / Var(X)$：最优系数（有限样本近似，大样本下收敛于总体版本）

**方差缩减效果：**
$$Var(\tilde{Y}) = Var(Y)(1 - \rho^2)$$

其中 $\rho$ 是 $Y$ 和 $X$ 的相关系数。若 $\rho=0.7$，方差缩减约50%。

## 使用场景

| 场景 | 说明 | 示例 |
|---|---|---|
| 指标波动大 | 核心指标方差高，实验难显著 | 人均GMV标准差大，CUPED用历史GMV降噪 |
| 样本量受限 | 用户少，需提升灵敏度 | B端商家实验，仅几百样本 |
| 缩短实验周期 | 希望更快得到结论 | 用1周历史数据，将2周实验缩短至1周 |
| 提升MDE精度 | 检测更小的效应 | 从检测+1pp提升到检测+0.5pp |

## 局限性

1. **需要历史数据**：新用户、新指标无历史数据时无法使用
2. **相关性要求**：$X$ 和 $Y$ 需有足够相关性（通常要求 $\rho > 0.5$）
3. **时间稳定性**：历史与实验期间的关系需稳定，结构性变化时失效
4. **处理影响协变量**：若处理也影响 $X$（如长期处理），CUPED有偏
5. **计算复杂度**：需计算协方差矩阵，多指标时复杂

## 同类算法对比

| 方法 | 原理 | 优势 | 劣势 |
|---|---|---|---|
| **CUPED** | 实验前同一指标作为协变量 | 简单，方差缩减显著 | 要求历史数据，相关性高 |
| **后分层（Post-stratification）** | 按协变量分层后加权 | 可处理分类变量 | 分层变量选择关键 |
| **回归调整** | 回归中控制协变量 | 灵活，可多个协变量 | 模型设定影响结果 |
| **分层随机化** | 实验前按协变量分层 | 设计阶段解决，分析简单 | 需预先知道关键协变量 |
| **ML-based方差缩减** | 用ML预测残差 | 可处理复杂关系 | 过拟合风险，解释性差 |

## 真实案例

### 案例1：人均GMV实验（C端运营）

**背景：** 评估新推荐策略对人均GMV的影响，GMV波动大。

**数据：**
- 实验期间人均GMV：均值=￥85，标准差=￥120
- 实验前1周人均GMV：与实验期间相关系数 $\rho=0.65$

**CUPED应用：**
- 方差缩减比例 = $1 - 0.65^2 = 0.58$，即方差减少58%
- 标准差从120降至78
- 样本量需求减少：$n_{new} = n_{old} \times 0.42$

**结果：**
- 原始分析：效应+3.2元，p=0.12（不显著）
- CUPED分析：效应+3.2元，p=0.03（显著）
- 结论：CUPED帮助检测出真实效应

> **小样本修正** [P1已修复]：小样本时$\theta$的估计误差会导致方差缩减效果略低于理论值。建议样本量>500时使用CUPED，否则方差缩减不稳定。

### 案例2：商家留存实验（B端运营）

**背景：** 评估"运营顾问"服务对商家30天留存的影响，商家数量少。

**数据：**
- 商家数：处理组150，对照组150
- 30天留存率：均值=65%，标准差大（二项分布）
- 实验前7天活跃度：与留存相关系数 $\rho=0.45$

**分析：**
- CUPED后方差缩减约20%
- MDE从8pp降至6pp
- 实验结果：留存+5pp，原始p=0.18，CUPED后p=0.08（边缘显著）

**决策：**
- 边缘显著，但业务认为5pp有价值
- 建议扩大样本或延长实验，而非直接全量

### 案例3：新用户首单实验（离线分析）

**背景：** 评估注册流程简化对首单转化率的影响，新用户无历史数据。

**挑战：**
- CUPED需要历史数据，新用户没有
- 替代方案：用注册过程中的行为作为协变量（如注册完成时长、浏览商品数）

**调整CUPED：**
- $X$ = 注册后1小时内浏览商品数（与首单转化相关系数=0.3）
- 方差缩减约9%，效果有限

**结论：**
- 对于新用户，CUPED效果有限
- 建议用分层随机化（按注册渠道分层）替代

## 行业使用经验

### 经验1：CUPED不是"万能药"——相关性<0.5时方差缩减有限，别硬上

某内容平台做新用户留存实验，试图用"注册后1小时内浏览商品数"作为CUPED协变量（与留存ρ=0.3）。方差仅缩减9%，实验结论从"不显著"变为"边缘显著"（p=0.07），业务方据此要求全量。数据负责人拒绝：ρ=0.3带来的缩减不足以改变统计推断的可靠性。**规范：ρ<0.5时，CUPED的降噪效果有限，不应作为"救命稻草"。此时应优先考虑增加样本量或延长实验周期。**

### 经验2："实验前同一指标"不是唯一选择——我们曾用错误的历史数据让CUPED有偏

美团某实验用"实验前1周人均GMV"作为CUPED协变量，但实验处理本身在实验前1周已开始灰度（部分用户提前暴露）。结果CUPED调整后的方差反而增大——因为协变量已被处理污染，不再是"纯前测"。**铁律：CUPED的协变量必须来自"实验处理完全未暴露"的时期。若存在灰度或预热，协变量窗口必须前移至完全未暴露期。**

### 经验3：CUPED后效应量不变，但置信区间变窄——不要因此夸大效应

某大厂分析师用CUPED后汇报："效应从+3.2元（p=0.12）变为+3.2元（p=0.03），效应量翻倍！"这是错误的——效应量始终是+3.2元，变化的是标准误（从1.8降至1.2）。CUPED不改变点估计，只改变精度。**常见错误：把"显著性变化"包装成"效应变强"。正确的表述："CUPED后估计精度提升，同一效应量现在以更高置信度被检测到。"**

### 经验4：新用户实验别指望CUPED——分层随机化才是正道

某外卖平台做"注册流程简化"实验，新用户无历史GMV数据，CUPED无法使用。团队尝试用"注册渠道"作为替代协变量，但不同渠道的用户质量差异太大，协变量与结果相关性不稳定。**建议：新用户实验默认采用分层随机化（按注册渠道、设备类型、地域分层），而非CUPED。分层在设计阶段解决不平衡问题，CUPED在分析阶段降噪，两者互补但不替代。**

## 代码示例

```python
import numpy as np

np.random.seed(42)

# CUPED 调整
def cuped_adjust(y, x, theta=None):
    if theta is None:
        theta = np.cov(y, x)[0, 1] / np.var(x)
    y_cuped = y - theta * (x - x.mean())
    return y_cuped, theta

# 方差缩减百分比
def variance_reduction(y, y_cuped):
    return (np.var(y, ddof=1) - np.var(y_cuped, ddof=1)) / np.var(y, ddof=1) * 100
```
