# 实验分析决策框架

## 问题引入

假设你是一家出行平台的数据分析师。一个AB实验跑完了：新定价策略使订单量提升了3%（p=0.001，高度显著），但司机满意度下降了2%（p=0.08，不显著），用户投诉率上升了1.5%（p=0.04，显著）。产品负责人问你："全量吗？"

这个问题没有标准答案，但它暴露了实验分析中最核心的张力：统计显著性≠业务正确性。p=0.001的订单量提升很"漂亮"，但如果司机满意度持续下降导致运力流失，三个月后订单量可能反而崩盘。用户投诉率"显著"上升1.5%，但绝对值从0.8%到0.95%，业务上是否"不可接受"？

在[[stage-01-foundation/03-ab-testing/06-multiple-testing/多重检验问题|多重检验]]中，我们知道统计结果需要校正；在[[stage-01-foundation/02-statistical-basics/01-t-test/t检验|假设检验]]中，我们知道"显著"只是"不太可能是噪声"。但这些统计工具只能回答"数据怎么说"，不能回答"我们应该怎么做"。从统计结果到业务决策，中间隔着一道"判断"的鸿沟——这道鸿沟需要综合考虑效应大小、实施成本、风险评估、长期影响、子群差异等多个维度。

更棘手的是，实验环境和生产环境往往不同。小流量实验中的用户可能更价格敏感（低价值用户更容易被实验策略影响），全量后高价值用户响应弱，效应被稀释——"实验效应"只是"全量效应"的上界。还有novelty效应：新策略短期刺激用户点击，但用户习惯后效应消退；学习效应：用户需要时间适应新界面，短期数据不代表长期表现。

实验分析决策框架，就是把"统计显著性"翻译成"业务行动"的翻译器。它要求分析师不仅懂统计，还要懂业务、懂权衡、懂风险。一个优秀的实验报告，不应该止于"p<0.05，建议全量"，而应该回答："这个效应有多大？值不值得做？有什么风险？怎么监控？如果出问题怎么回滚？"

## 基本原理

实验分析不仅是统计检验，更是将统计结果转化为业务决策的过程。需要综合统计显著性、效应大小、业务意义、风险评估等多维度信息。

**决策核心问题：**
1. 统计上：结果是否显著？是否稳健？
2. 业务上：效应是否足够大？是否值得实施？
3. 风险上：是否有负面后果？如何监控？

## 使用场景

| 场景 | 决策类型 | 示例 |
|---|---|---|
| 实验结果显著正向 | 全量/部分放量 | 核心指标+5%，护栏无异常 |
| 实验结果显著负向 | 停止/回滚 | 核心指标-3%，护栏告警 |
| 实验结果不显著 | 继续/停止/扩大 | 效应接近0，或MDE内 |
| 子群效应异质 | 分层策略 | 新用户有效，老用户无效 |
| 长期效应未知 | 延长/替代指标 | 短期有效，长期未知 |

## 局限性

1. **统计显著≠业务重要**：+0.01pp显著但无业务价值
2. **实验环境≠生产环境**：小流量实验可能不同于全量
3. **短期≠长期**：novelty效应、学习效应
4. **局部≠全局**：某场景有效不代表全场景有效
5. **相关指标冲突**：核心指标提升但护栏指标下降

## 同类算法对比

| 决策模式 | 适用条件 | 优势 | 劣势 |
|---|---|---|---|
| **统计驱动** | 指标清晰，风险低 | 客观，可复现 | 忽略业务复杂性 |
| **业务驱动** | 战略决策，长期影响 | 考虑全局 | 主观，易被叙事影响 |
| **混合框架** | 大多数场景 | 平衡统计与业务 | 需要明确的权重规则 |
| **贝叶斯决策** | 有先验信息 | 利用历史数据 | 先验选择主观 |
| **成本效益分析** | 资源有限 | 量化ROI | 成本和收益难量化 |

## 真实案例

### 案例1：显著但效应小的决策（C端运营）

**背景：** 新按钮颜色实验，点击率+0.3%（p=0.001），但绝对效应小。

**分析：**
- 统计：高度显著
- 效应：日均增加点击1000次，但转化漏斗后续环节无变化
- 业务：开发成本已沉没，但全量需QA和发布成本
- 风险：无已知风险

**决策：**
- 若发布成本低 → 全量（"反正没坏处"）
- 若发布成本高 → 不做（"不值得"）
- 本例：发布成本低，全量，但记录"效应小，未来类似实验提高MDE"

### 案例2：指标冲突的权衡（C端运营）

**背景：** 新推荐策略，点击+5%（显著），但下单转化率-0.5%（边缘显著）。

**分析：**
- 核心指标冲突：短期 engagement ↑，但可能牺牲质量
- 假设：用户点击更多但购买意愿下降（标题党效应）
- 护栏指标：人均浏览时长-2%（显著）

**决策：**
- 停止实验，不推广
- 分析bad case：哪些商品点击高但转化低
- 迭代策略：优化推荐质量而非仅点击率

### 案例3：B端功能的不显著决策（商家运营）

**背景：** 新数据分析功能，商家GMV效应+2%（p=0.15，不显著）。

**分析：**
- 统计：不显著，但效应方向为正
- 样本量：仅300商家，MDE=3%，可能灵敏度不足
- 业务：功能开发成本高，但商家反馈好

**决策：**
- 不基于当前实验全量
- 扩大实验：增加商家至1000，延长周期
- 同时收集定性反馈，补充定量分析

```
   - 效应大小：绝对效应 = xxx
   - ROI估算：收益 = xxx，成本 = xxx
   - 实施复杂度：高/中/低

   - 已知风险：xxx

   - [ ] 停止/回滚

5. 后续行动
   - xxx
```

## 行业使用经验

### 经验1："统计显著+业务显著"双门槛缺一不可——我们曾全量了一个+0.3%的"显著"实验

某大厂按钮颜色实验，点击率+0.3%（p<0.001），绝对日均增加点击仅1000次，后续转化漏斗无变化。开发和QA成本却实打实。全量三个月后复盘：ROI为负。**规范：实验决策模板必须包含"绝对效应量"和"ROI估算"栏。效应量<业务最小关注阈值时，即使p<0.001也应标记为"不值得实施"。**

### 经验2：指标冲突时，"停下来分析bad case"比"选边站"更有价值

滴滴某推荐策略实验，CTR+5%（显著），但下单转化率-0.5%（边缘显著）。PM倾向于"CTR是核心指标，转化微降可接受"。数据团队坚持停止，并花两天分析bad case：发现新策略推荐了更多"标题党"内容（点击高但购买意愿低）。迭代后优化推荐质量，CTR+3%、转化+1%，双赢。**教训：指标冲突时，不要急于决策。花1-2天做bad case分析，往往能找到"兼得"的迭代方向。**

### 经验3：不显著≠无效——我们曾因"不显著"而杀死了一个真实有效的策略

美团某功能实验，核心指标+2%（p=0.15，不显著），样本量仅300商家，MDE=3%。业务方据此停止项目。但真实效应可能就是2%，只是实验灵敏度不足。后续扩大样本至1000商家，效应+2.1%（p=0.04），显著。**规范：不显著时必须报告"检验功效"和"置信区间"。若置信区间包含"业务有意义的效应"（如+2%），结论应是"灵敏度不足，建议扩大实验"，而非"策略无效"。**

### 经验4：子群效应异质时，"分层全量"比"一刀切"更安全

某内容平台实验显示整体CTR无变化（+0.1%，不显著），但新用户CTR+8%（显著），老用户CTR-2%（显著）。若整体不显著就停止，会损失新用户增长机会；若全量，会伤害老用户。**正确做法：对显著正向子群（新用户）全量，对显著负向子群（老用户）保持现状。子群分析不是"锦上添花"，而是"必要步骤"。**
