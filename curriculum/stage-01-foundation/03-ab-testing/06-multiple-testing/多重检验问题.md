# 多重检验问题

## 问题引入

假设你是一家内容平台的数据分析师。你们上线了一个新推荐算法的AB实验，同时监控着10个指标：人均点击次数、人均浏览时长、下单转化率、人均下单金额、次日留存、7日留存、分享率、评论率、完播率、人均使用频次。

实验跑了14天，你每天都忍不住去看数据。第5天，你发现"分享率"显著提升（p=0.02），兴奋地准备写报告。但第7天再看，分享率又不显著了。第10天，"评论率"显著（p=0.03）。第14天最终分析，"人均点击次数"显著（p=0.01），但其他9个指标都不显著。

你向老板汇报："核心指标点击次数显著提升，建议全量。"老板问："那其他9个不显著的指标呢？"你回答："不显著说明没影响。"——但这话有问题。

问题的核心在于：当你同时进行多次检验时，"至少出现一次假阳性"的概率会急剧膨胀。10个指标各自用α=0.05检验，即使所有处理真实效应为零，也有约40%的概率至少看到一个"显著"指标。如果你每天看一次数据、看14天，这个概率更高。你第5天看到的"分享率显著"，很可能就是"多重检验下的假阳性噪音"。

更隐蔽的是"peeking"问题：实验过程中多次查看数据，并在"看到显著"时提前停止——这相当于进行了多次检验，严重膨胀了假阳性率。还有"事后选指标"问题：实验前说好看点击率和转化率，实验后发现点击率不显著、转化率显著，就只说转化率——这也是多重检验，只是更隐蔽。

在[[stage-01-foundation/03-ab-testing/01-experiment-design/AB实验设计要素|实验设计]]中，我们强调核心指标和护栏指标要预先定义；在[[stage-01-foundation/03-ab-testing/07-analysis-decision/实验分析决策框架|分析决策]]中，我们需要把统计结果转化为业务决策。但所有这些的前提，是统计结果本身是可靠的。多重检验问题就是统计可靠性的头号威胁。

## 基本原理

当进行多次假设检验时，即使每次检验的显著性水平为α，整体犯第一类错误（假阳性）的概率会急剧增加。

**直观例子：**
- 1次检验，假阳性概率 = 5%
- 10次独立检验，至少1次假阳性概率 = $1 - 0.95^{10}$ = 40%
- 100次独立检验，至少1次假阳性概率 = 99%

> **独立性假设说明** [P1已修复]：上述99%是检验相互独立时的假阳性概率上限。实际AB实验中指标间通常存在相关性，整体假阳性概率会低于此理论上限。

**核心问题：** 在AB实验中，通常同时监控多个指标、多个子群，如何控制整体错误率？

## 使用场景

| 场景 | 检验次数 | 风险 |
|---|---|---|
| 多指标监控 | 核心+护栏+局部指标，5-10个 | 某个指标假阳性导致错误决策 |
| 多子群分析 | 按城市/渠道/用户群拆分，10+个 | 子群效应可能是偶然 |
| 多实验并行 | 团队同时跑10个实验 | 至少几个会"显著" |
| 时间序列监控 | 每天看数据，持续2周 | peeking导致假阳性 |
| 多版本测试 | A/B/C/D/E多组比较 | 两两比较次数爆炸 |

## 局限性

1. **校正后检验力下降**：控制FWER越严格，真阳性越难检出
2. **检验间相关性**：指标间通常相关，Bonferroni过于保守
3. **业务指标 vs 统计指标**：业务上"显著"不等于统计显著
4. **peeking问题**：中期看数据并停止，即使不多重检验也增加假阳性
5. **后验选择**：事后选择"显著的"子群报告，是隐性的多重检验

## 同类算法对比

| 方法 | 控制目标 | 适用场景 | 保守程度 |
|---|---|---|---|
| **Bonferroni** | FWER（族错误率） | 少量独立检验 | 最保守 |
| **Holm-Bonferroni** | FWER | 少量检验，比Bonferroni稍松 | 较保守 |
| **Benjamini-Hochberg (BH)** | FDR（假发现率） | 大量探索性检验 | 适中 |
| **Benjamini-Yekutieli (BY)** | FDR（任意相关） | 检验间相关性强 | 比BH稍保守 |
| **序贯检验** | 整体错误率 | 可中期停止的实验 | 需要预先设计 |
| **分层检验** | 分层错误率 | 有主次关系的指标 | 灵活 |

## 真实案例

### 案例1：多指标实验的校正（C端运营）

**背景：** 新推荐算法实验，监控5个指标。

**指标：**
1. 人均点击次数（核心，α=0.05）
2. 人均浏览时长（护栏，α=0.05）
3. 下单转化率（核心，α=0.05）
4. 人均下单金额（次要，α=0.05）
5. 次日留存（长期，α=0.05）

**问题：**
- 若不做校正，整体假阳性概率 = $1 - 0.95^5$ = 23%

**Bonferroni校正：**
- 每个指标α = 0.05/5 = 0.01
- 结果：点击次数p=0.003（显著），转化率p=0.02（不显著）
- 问题：过于保守，转化率可能是真效应

**分层检验（更优）：**
- 第一层：核心指标（点击+转化），α=0.05
  - 若至少一个显著，进入第二层
- 第二层：护栏指标（时长），α=0.05
  - 若通过护栏，进入第三层
- 第三层：次要指标（金额+留存），α=0.05
- 结果：点击显著 → 时长不显著（通过护栏）→ 金额p=0.03（显著）

### 案例2：子群分析的陷阱（离线分析）

**背景：** 实验整体不显著，但发现"iOS用户"子群显著。

**问题：**
- 按系统（iOS/Android）、城市（10个）、新老用户，共2×10×2=40个子群
- 即使整体无效应，约2个子群"显著"（40×0.05=2）

**正确做法：**
1. 预先定义关键子群（不超过5个）
2. 子群分析声明为"探索性"，不用于决策
3. 若某子群显著，需独立实验验证

**本例：**
- iOS子群显著可能是偶然
- 建议：对iOS用户单独开实验验证

### 案例3：Peeking的代价（运营监控）

**背景：** 实验每天看数据，看到"显著"就停止。

**模拟：**
- 真实效应为0的实验
- 每天检验，持续14天
- 至少1天"显著"的概率 ≈ 50%（远高于5%）

**解决方案：**
- 固定分析点：如只在第7天和第14天分析
- 序贯检验：如O'Brien-Fleming边界
- 简单规则：实验周期内不看数据，到期后分析

## 关键词

- **FWER（Family-Wise Error Rate）**：族错误率，至少一次假阳性的概率
- **FDR（False Discovery Rate）**：假发现率，显著结果中假阳性的比例
- **Bonferroni校正**：α/m，最保守的简单校正
- **Holm校正**：逐步校正，比Bonferroni稍松
- **BH程序**：控制FDR的常用方法
- **Peeking**：实验过程中多次查看数据并可能停止
- **序贯检验**：预先设计多个分析点，控制整体错误率
- **分层检验**：按指标重要性分层，逐层传递α

## 代码示例

```python
import numpy as np
from scipy import stats

np.random.seed(42)

# Bonferroni 校正
pvalues = np.random.uniform(0, 1, 20)
reject_bonf = pvalues < (0.05 / len(pvalues))

# Benjamini-Hochberg 程序
def benjamini_hochberg(pvalues, alpha=0.05):
    n = len(pvalues)
    sorted_idx = np.argsort(pvalues)
    sorted_p = pvalues[sorted_idx]
    thresholds = np.arange(1, n + 1) / n * alpha
    bh_reject = sorted_p <= thresholds
    if bh_reject.any():
        max_k = np.where(bh_reject)[0][-1]
        reject = np.zeros(n, dtype=bool)
        reject[sorted_idx[:max_k + 1]] = True
        return reject
    return np.zeros(n, dtype=bool)
```

## 反常识：多重检验的4个认知盲区

> 你做的不是1个检验，而是14个——只是你自己没意识到。

### 陷阱1：「我只看了1个指标」的幻觉

**直觉**：我只检验了主要指标（OEC），没有多重检验问题。
**真相**：你在分析过程中可能"偷偷"看了多个指标（guardrail、细分维度），每次看都增加了假阳性概率。
**案例**：某分析师声称"我只看了留存率"。但实际上，他先看了留存率（不显著），又看了活跃度（不显著），又看了GMV（显著，p=0.04），然后报告"GMV显著提升"。这是典型的"选择性报告"——看了3个指标，假阳性概率=1-(0.95)^3=14.3%。
**自检**：分析过程中实际看了多少个指标？是否进行了多重检验校正？

### 陷阱2：Bonferroni「太保守」所以不用

**直觉**：Bonferroni校正太保守，会漏掉真实效应，所以不用。
**真相**：Bonferroni确实保守，但FDR（Benjamini-Hochberg）是更好的折中。完全不做校正等于接受高假阳性率。
**案例**：某实验检验了10个指标，其中2个"显著"（p=0.03, 0.04）。未校正时直接报告"2个指标显著"。Bonferroni校正后都不显著。FDR校正后1个显著。完全不做校正，实际上有40%的概率至少1个是假阳性。
**自检**：是否报告了校正前后的p值？是否说明了校正方法的选择理由？

### 陷阱3：「事后分组」不算多重检验

**直觉**：我先看整体不显著，然后细分用户群找显著子群，这是"深入挖掘"。
**真相**：事后分组（post-hoc subgroup analysis）是隐性的多重检验。看k个子群，假阳性概率=1-(0.95)^k。
**案例**：某实验整体不显著（p=0.12）。分析师按年龄、性别、城市、设备类型细分，看了16个子群。其中1个子群"显著"（p=0.04）。但这是预期的——16个子群中，期望假阳性数=16×0.05=0.8个。这个"显著"很可能是假阳性。
**自检**：子群分析是预设的还是事后的？是否对子群数量进行了校正？

### 陷阱4：Peeking不是多重检验

**直觉**：我只是在实验过程中"看看数据"，不是正式检验。
**真相**：每次"看看"都是一次隐性的检验。看k次，假阳性概率≈1-(0.95)^k。
**案例**：某实验计划跑2周。第3天、第7天、第10天，老板问了3次"结果怎么样"。每次看都是p<0.05就停止。最终在第10天"显著"停止。但事后模拟显示，这种peeking策略的假阳性率高达30%+。
**自检**：实验过程中看了几次数据？是否使用了序贯检验（sequential testing）或-group sequential方法？

> 💡 **核心原则**：多重检验不是问题，"没意识到自己在做多重检验"才是问题。每次你看数据做决策，都在消耗你的"显著性预算"。
