# 多重检验问题

## 基本原理

当进行多次假设检验时，即使每次检验的显著性水平为α，整体犯第一类错误（假阳性）的概率会急剧增加。

**直观例子：**
- 1次检验，假阳性概率 = 5%
- 10次独立检验，至少1次假阳性概率 = $1 - 0.95^{10}$ = 40%
- 100次独立检验，至少1次假阳性概率 = 99%

> **独立性假设说明** [P1已修复]：上述99%是检验相互独立时的假阳性概率上限。实际AB实验中指标间通常存在相关性，整体假阳性概率会低于此理论上限。

**核心问题：** 在AB实验中，通常同时监控多个指标、多个子群，如何控制整体错误率？

## 使用场景

| 场景 | 检验次数 | 风险 |
|---|---|---|
| 多指标监控 | 核心+护栏+局部指标，5-10个 | 某个指标假阳性导致错误决策 |
| 多子群分析 | 按城市/渠道/用户群拆分，10+个 | 子群效应可能是偶然 |
| 多实验并行 | 团队同时跑10个实验 | 至少几个会"显著" |
| 时间序列监控 | 每天看数据，持续2周 | peeking导致假阳性 |
| 多版本测试 | A/B/C/D/E多组比较 | 两两比较次数爆炸 |

## 局限性

1. **校正后检验力下降**：控制FWER越严格，真阳性越难检出
2. **检验间相关性**：指标间通常相关，Bonferroni过于保守
3. **业务指标 vs 统计指标**：业务上"显著"不等于统计显著
4. **peeking问题**：中期看数据并停止，即使不多重检验也增加假阳性
5. **后验选择**：事后选择"显著的"子群报告，是隐性的多重检验

## 同类算法对比

| 方法 | 控制目标 | 适用场景 | 保守程度 |
|---|---|---|---|
| **Bonferroni** | FWER（族错误率） | 少量独立检验 | 最保守 |
| **Holm-Bonferroni** | FWER | 少量检验，比Bonferroni稍松 | 较保守 |
| **Benjamini-Hochberg (BH)** | FDR（假发现率） | 大量探索性检验 | 适中 |
| **Benjamini-Yekutieli (BY)** | FDR（任意相关） | 检验间相关性强 | 比BH稍保守 |
| **序贯检验** | 整体错误率 | 可中期停止的实验 | 需要预先设计 |
| **分层检验** | 分层错误率 | 有主次关系的指标 | 灵活 |

## 真实案例

### 案例1：多指标实验的校正（C端运营）

**背景：** 新推荐算法实验，监控5个指标。

**指标：**
1. 人均点击次数（核心，α=0.05）
2. 人均浏览时长（护栏，α=0.05）
3. 下单转化率（核心，α=0.05）
4. 人均下单金额（次要，α=0.05）
5. 次日留存（长期，α=0.05）

**问题：**
- 若不做校正，整体假阳性概率 = $1 - 0.95^5$ = 23%

**Bonferroni校正：**
- 每个指标α = 0.05/5 = 0.01
- 结果：点击次数p=0.003（显著），转化率p=0.02（不显著）
- 问题：过于保守，转化率可能是真效应

**分层检验（更优）：**
- 第一层：核心指标（点击+转化），α=0.05
  - 若至少一个显著，进入第二层
- 第二层：护栏指标（时长），α=0.05
  - 若通过护栏，进入第三层
- 第三层：次要指标（金额+留存），α=0.05
- 结果：点击显著 → 时长不显著（通过护栏）→ 金额p=0.03（显著）

### 案例2：子群分析的陷阱（离线分析）

**背景：** 实验整体不显著，但发现"iOS用户"子群显著。

**问题：**
- 按系统（iOS/Android）、城市（10个）、新老用户，共2×10×2=40个子群
- 即使整体无效应，约2个子群"显著"（40×0.05=2）

**正确做法：**
1. 预先定义关键子群（不超过5个）
2. 子群分析声明为"探索性"，不用于决策
3. 若某子群显著，需独立实验验证

**本例：**
- iOS子群显著可能是偶然
- 建议：对iOS用户单独开实验验证

### 案例3：Peeking的代价（运营监控）

**背景：** 实验每天看数据，看到"显著"就停止。

**模拟：**
- 真实效应为0的实验
- 每天检验，持续14天
- 至少1天"显著"的概率 ≈ 50%（远高于5%）

**解决方案：**
- 固定分析点：如只在第7天和第14天分析
- 序贯检验：如O'Brien-Fleming边界
- 简单规则：实验周期内不看数据，到期后分析

## 关键词

- **FWER（Family-Wise Error Rate）**：族错误率，至少一次假阳性的概率
- **FDR（False Discovery Rate）**：假发现率，显著结果中假阳性的比例
- **Bonferroni校正**：α/m，最保守的简单校正
- **Holm校正**：逐步校正，比Bonferroni稍松
- **BH程序**：控制FDR的常用方法
- **Peeking**：实验过程中多次查看数据并可能停止
- **序贯检验**：预先设计多个分析点，控制整体错误率
- **分层检验**：按指标重要性分层，逐层传递α
