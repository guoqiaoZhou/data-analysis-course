# 观察性数据的挑战

## 问题引入

假设你是一家在线教育平台的数据分析师。公司新上线了一款 AI 辅导产品，产品团队声称："使用 AI 辅导的学生，期末考试成绩平均提高了 15 分！" 管理层准备追加千万级预算全面推广。作为数据分析师，你的直觉告诉你——这 15 分的提升真的完全来自 AI 辅导吗？

你进一步查看数据，发现了一个令人不安的模式：使用 AI 辅导的学生中，有 78% 来自一线城市、家长学历本科以上、且入学基线成绩本身就高于平均水平。而那些没有使用 AI 辅导的学生，大多来自教育资源相对匮乏的地区。你开始怀疑，这 15 分的"提升"可能只是"好学生本来就会用，也本来就会考得好"——这是典型的**自选择偏差**。

更棘手的是，当你试图"控制"一些变量来"净化"这 15 分的效果时，情况反而变得更糟。你控制了"学生每周自习时长"，发现 AI 辅导的效果从 15 分骤降到 3 分。但产品团队反驳说："AI 辅导正是通过让学生更愿意自习来起作用的，你控制自习时长等于把我们的产品机制给抹掉了！" 你意识到，自己可能犯了一个更隐蔽的错误——**过度控制（Over-control）**。

这还不是全部。你又尝试控制"学生是否参加了课外补习班"，结果 AI 辅导的效果突然变成了负 5 分。你困惑不已，直到一位资深同事指出："参加补习班的学生，往往是那些同时愿意尝试 AI 辅导、又本身学习动力强的学生。补习班是'学习投入'和'使用 AI'共同的结果，控制它反而打开了一条虚假的因果路径。" 这就是**碰撞变量偏差（Collider Bias）**。

这三个场景——自选择、过度控制、碰撞变量——构成了观察性数据因果推断的核心挑战。在 A/B 测试中，随机化保证了实验组和对照组在期望上是可比的；但在现实业务中，绝大多数决策场景（用户自然行为、政策自然实施、疾病自然发生）都不具备随机化的条件。我们无法"重新分配"谁用 AI 辅导、谁不用；我们只能观察已经发生的事情。而观察到的数据中，处理变量（Treatment）的分配从来不是随机的——它受到无数可见和不可见因素的影响。这正是观察性研究面临的根本困境：**因果效应的识别，本质上是在非随机化的世界中寻找随机化的影子**。

理解这一挑战，是每一位数据分析师从"相关性描述"走向"因果推断"的必经之路。

## 核心概念

### 1. 混杂变量（Confounder）

**定义**：混杂变量 $Z$ 是同时影响处理变量 $T$ 和结果变量 $Y$ 的变量，它在 $T$ 和 $Y$ 之间制造了一条虚假的统计关联。

**数学表达**：

在潜在结果框架下，因果效应定义为：

$$
\tau = \mathbb{E}[Y(1) - Y(0)]
$$

但在观察性数据中，我们实际估计的是：

$$
\mathbb{E}[Y \mid T=1] - \mathbb{E}[Y \mid T=0] = \tau + \underbrace{\mathbb{E}[Y(0) \mid T=1] - \mathbb{E}[Y(0) \mid T=0]}_{\text{选择偏差}}
$$

当存在混杂变量 $Z$ 时，$T \not\perp Y(0), Y(1)$，选择偏差不为零。

**DAG 表示**：

```
Z → T → Y
Z → Y
```

$Z$ 是 $T$ 和 $Y$ 的共同原因，不对 $Z$ 进行控制会导致 $T$ 和 $Y$ 之间出现虚假的边。

**业务实例**：学生家庭社会经济地位（SES）同时影响是否使用 AI 辅导（$T$）和考试成绩（$Y$）。如果不控制 SES，AI 辅导的效果会被高估。

### 2. 碰撞变量（Collider）

**定义**：碰撞变量 $C$ 是同时被处理变量 $T$ 和结果变量 $Y$（或它们的后代）影响的变量。在 DAG 中，$T \rightarrow C \leftarrow Y$ 形成碰撞结构。**控制碰撞变量会打开一条原本封闭的虚假路径**。

**数学直觉**：

设 $T$ 和 $Y$ 原本独立（无因果效应），但共同影响 $C$。当我们以 $C=c$ 为条件进行子集分析时：

$$
P(T, Y \mid C=c) \neq P(T \mid C=c) \cdot P(Y \mid C=c)
$$

$T$ 和 $Y$ 在给定 $C$ 的条件下变得相关，产生虚假的因果关联。

**DAG 表示**：

```
T → C ← Y
```

原始路径 $T$ 和 $Y$ 之间无通路；控制 $C$ 后，路径 $T \rightarrow C \leftarrow Y$ 被"打开"，产生虚假关联。

**业务实例**："参加课外补习班"（$C$）可能同时被"使用 AI 辅导"（$T$）和"学习动力强"（影响 $Y$）所影响。控制 $C$ 会在 $T$ 和 $Y$ 之间制造虚假关联。

### 3. 中介变量（Mediator）

**定义**：中介变量 $M$ 位于处理变量 $T$ 到结果变量 $Y$ 的因果路径上，$T$ 通过 $M$ 间接影响 $Y$。

**DAG 表示**：

```
T → M → Y
```

**关键问题**：控制中介变量会阻断间接效应，导致**总效应被低估**。

**效应分解**：

$$
\underbrace{\mathbb{E}[Y(1) - Y(0)]}_{\text{总效应}} = \underbrace{\mathbb{E}[Y(1, M(1)) - Y(1, M(0))]}_{\text{直接效应}} + \underbrace{\mathbb{E}[Y(1, M(0)) - Y(0, M(0))]}_{\text{间接效应}}
$$

其中 $Y(t, m)$ 表示在 $T=t$ 且 $M=m$ 时的潜在结果。

**业务实例**：AI 辅导（$T$）通过增加学生自习时长（$M$）提升成绩（$Y$）。如果分析时控制了自习时长，就会阻断这条间接路径，低估 AI 辅导的真实效果。

### 4. 控制变量选择原则

基于 DAG 的后门准则（Back-Door Criterion），一个变量集合 $Z$ 满足后门准则，如果：

1. $Z$ 阻断了所有从 $T$ 到 $Y$ 的后门路径（即所有以指向 $T$ 的箭头开始的路径）；
2. $Z$ 不包含 $T$ 的任何后代。

**选择策略**：

| 变量类型 | 是否控制 | 原因 |
|---|---|---|
| 混杂变量（Confounder） | ✅ 控制 | 阻断虚假的后门路径 |
| 碰撞变量（Collider） | ❌ 不控制 | 控制会打开虚假路径 |
| 中介变量（Mediator） | ❌ 不控制（估计总效应时） | 控制会阻断间接效应 |
| 工具变量（Instrument） | ❌ 不控制 | 控制会降低统计效率 |
| 对撞后代（Collider descendant） | ❌ 不控制 | 与碰撞变量同理 |

## 关键假设

观察性因果推断依赖于以下核心假设：

1. **无混淆假设（Unconfoundedness / Ignorability）**：
   $$
   (Y(0), Y(1)) \perp T \mid X
   $$
   给定观察到的协变量 $X$，潜在结果与处理分配独立。这是观察性研究中最强的假设，通常无法直接验证。

2. **正值假设（Positivity / Overlap）**：
   $$
   0 < P(T=1 \mid X=x) < 1, \quad \forall x
   $$
   对于每一个协变量组合 $x$，处理组和对照组都有非零的概率被观察到。违反此假设会导致外推风险。

3. **稳定单位处理值假设（SUTVA）**：
   - 无干扰：一个单位的处理不影响其他单位的潜在结果；
   - 一致性：$Y = T \cdot Y(1) + (1-T) \cdot Y(0)$。

4. **正确模型设定**：倾向得分模型或结果回归模型被正确设定（对于基于模型的方法）。

## 常见误区

| 误区 | 正确理解 |
|---|---|
| "变量越多越好，多控制总不会错" | 控制碰撞变量或中介变量会引入偏差或低估效应。变量选择应基于因果图（DAG）而非数据驱动。 |
| "相关性等于因果性，回归显著就说明有因果效应" | 回归系数反映的是条件关联，除非满足无混淆假设，否则不能解释为因果效应。 |
| "控制了所有观察到的变量，就没有遗漏变量偏误了" | 未观察到的混杂变量始终存在威胁。敏感性分析是评估稳健性的必要步骤。 |
| "子群分析（Subgroup Analysis）总是更精确" | 子群分析往往隐含地对碰撞变量进行了条件化，可能引入碰撞偏差。 |
| "工具变量法不需要假设" | 工具变量需要满足相关性、排他性和独立性假设，其中排他性通常无法直接检验。 |
| "匹配后数据就等价于随机实验了" | 匹配只能控制观察到的混杂变量，且匹配质量依赖于共同支撑域和正确设定。 |

## 行业使用经验

### 经验1：先画 DAG，再跑回归

在动手分析之前，先与业务方一起画出因果图（DAG）。DAG 的价值不在于"正确"，而在于**显式地暴露假设**。当团队对某个变量是混杂还是碰撞产生分歧时，DAG 能迫使大家把假设摆到桌面上讨论。我们推荐使用 [dagitty.net](http://dagitty.net/) 或 Python 的 `pgmpy` / `causalgraphicalmodels` 库来绘制和检验 DAG。

### 经验2：永远报告敏感性分析

在观察性研究中，"无未观察混杂"是一个无法证实的假设。因此，任何因果效应估计都应附带敏感性分析：如果存在一个未观察到的混杂变量，其强度需要达到多大才能推翻当前结论？常用的方法包括 Rosenbaum 边界法、E-value 法等。在业务报告中，明确写出"一个与处理组和结果都中度相关的未观察变量即可使效应消失"，比单纯报告点估计更有决策价值。

### 经验3：我们曾犯的错误——控制了"用户活跃度"作为协变量

在一次评估"新用户引导流程改版"（$T$）对"长期留存"（$Y$）效果的项目中，我们控制了"用户首周活跃度"（$C$）。结果改版效果不显著，团队差点放弃这个方向。后来复盘发现：新引导流程正是通过提升首周活跃度来影响长期留存的，同时高留存潜力的用户本身首周就更活跃。"首周活跃度"是一个典型的碰撞变量（受 $T$ 和 $Y$ 共同影响）。去除该控制变量后，改版效果显著为正。这个教训让我们建立了"控制变量白名单"机制：任何进入回归的变量必须在 DAG 上有明确身份。

### 经验4：中介分析需要预先设定，而非事后探索

在一次营销活动中，我们想评估"短信触达"（$T$）对"下单转化"（$Y$）的效果，并探索"打开 App"（$M$）是否起中介作用。如果我们先跑总效应回归，发现不显著，再跑控制 $M$ 的回归"凑"出一个显著的直接效应，这是**数据挖掘（P-hacking）**。正确做法是：在实验设计阶段就明确要估计的是总效应、直接效应还是间接效应，并选择对应的方法（如因果中介分析）。

## 代码示例

```python
"""
观察性数据挑战的核心模拟：混杂、碰撞、中介
对应完整可运行版本见同目录 code/simulation.py
"""
import numpy as np
import pandas as pd

np.random.seed(42)
n = 10000

# ---------- 1. 混杂变量场景 ----------
Z = np.random.normal(0, 1, n)           # 混杂：能力/SES
T = (Z + np.random.normal(0, 1, n) > 0).astype(int)  # 处理受Z影响
Y = 2 * T + 3 * Z + np.random.normal(0, 1, n)        # 结果受T和Z影响

# 不控制Z：高估；控制Z：正确估计 ~2
from sklearn.linear_model import LinearRegression
X_naive = T.reshape(-1, 1)
X_adjusted = np.column_stack([T, Z])
print("混杂场景 — 不控制Z:", LinearRegression().fit(X_naive, Y).coef_[0])
print("混杂场景 — 控制Z:  ", LinearRegression().fit(X_adjusted, Y).coef_[0])

# ---------- 2. 碰撞变量场景 ----------
T2 = np.random.normal(0, 1, n)          # 处理
Y2 = np.random.normal(0, 1, n)          # 结果（真实无因果效应）
C = (T2 + Y2 + np.random.normal(0, 1, n) > 0).astype(int)  # 碰撞变量

# 控制C：制造虚假效应
mask_c = C == 1
print("\n碰撞场景 — 控制C后T与Y的相关系数:", np.corrcoef(T2[mask_c], Y2[mask_c])[0, 1])

# ---------- 3. 中介变量场景 ----------
T3 = (np.random.normal(0, 1, n) > 0).astype(int)
M = 2 * T3 + np.random.normal(0, 1, n)  # 中介：自习时长
Y3 = 1.5 * T3 + 2.0 * M + np.random.normal(0, 1, n)  # 总效应 = 1.5 + 2*2 = 5.5

# 不控制M：估计总效应 ~5.5；控制M：仅直接效应 ~1.5
X_total = T3.reshape(-1, 1)
X_direct = np.column_stack([T3, M])
print("\n中介场景 — 不控制M (总效应):", LinearRegression().fit(X_total, Y3).coef_[0])
print("中介场景 — 控制M (直接效应): ", LinearRegression().fit(X_direct, Y3).coef_[0])
```

> 💻 **完整可运行代码**：见同目录下 `code/simulation.py`，包含可视化与完整输出。建议在 VS Code / PyCharm 中打开运行，或命令行执行 `python simulation.py`。
