# 好问练习解答 - 因果与相关的本质区别

> 对应知识点：[[stage-01-foundation/01-causal-inference-basics/04-causation-vs-correlation/因果与相关的本质区别|因果与相关的本质区别]]

---

### Q1: 为什么"冰淇淋销量和溺水事故同时上升"不等于"冰淇淋导致溺水"？

**Hints:**
- 思考"气温"在因果图中的位置
- 回忆混杂的定义：同时影响两个变量的第三方因素
- 如果禁止冰淇淋销售，溺水事故会减少吗？

**Answer:**
冰淇淋销量和溺水事故正相关，是因为两者有共同原因"气温升高"——天热人们既爱吃冰淇淋也爱游泳。气温是混杂变量：气温→冰淇淋销量，气温→溺水事故。禁止冰淇淋销售不会减少溺水，因为溺水的真正原因是游泳（由气温驱动），不是吃冰淇淋。

"相关强≠因果"的反驳：相关是"一起看"，因果是"如果改变"。$P(Y|X) \neq P(Y)$ 只说明知道冰淇淋销量能预测溺水，但 $P(Y|do(X=x)) = P(Y)$ 说明改变冰淇淋销量不会改变溺水分布。Pearl的$do$-算子明确区分了"观测"和"干预"。

**决策规则：**
- 发现相关后 → 先问"是否存在共同原因Z同时影响两者？"
- 如果存在强混杂（如气温、季节、用户活跃度） → 相关≠因果，禁止直接推断因果
- 任何"因为A和B相关，所以A导致B"的推理 → 必须经过"混杂检验""方向检验""机制检验"
- 向非技术背景解释 → 用"共同原因"类比（如" umbrellas和raincoat sales相关，不是因为umbrellas导致raincoat"）

### Q2: 混杂和反向因果有什么区别？怎么从数据中区分？

**Hints:**
- 画因果图：混杂是Z→X, Z→Y；反向因果是Y→X
- 数据能告诉你"X和Y相关"，但能告诉你"X→Y还是Y→X"吗？
- 时间顺序能否帮助区分？

**Answer:**
**混杂**：Z→X且Z→Y，X和Y无直接因果，但Z创造虚假相关。如"气温→冰淇淋销量"和"气温→溺水"。

**反向因果**：Y→X，因果方向与假设相反。如"死亡率高→建更多医院"（而非"医院多→死亡率高"）。

数据本身**无法区分**混杂和反向因果，因为两者都产生$P(Y|X) \neq P(Y)$。区分需要：
1. **时间顺序**：如果X发生在Y之后，不可能是X→Y（但时间先后≠因果先后）
2. **领域知识/机制**：医院不可能导致死亡，但死亡可能促使建医院
3. **实验/工具变量**：随机化或准自然实验能打破混杂和反向因果的模糊性

**决策规则：**
- 相关发现后 → 必须列出三种解释：X→Y、Y→X、Z→X且Z→Y
- 有时间序列数据 → 检查Granger因果（预测性，非真正因果）
- 有领域知识 → 优先排除机制上不可能的因果方向
- 任何情况下 → 只有随机化实验或强工具变量能"确定"因果方向

### Q3: 为什么"控制碰撞变量"会让原本无关的变量产生虚假相关？

**Hints:**
- 思考"入选精英计划"的入选标准
- 如果入选需要"质量高或频率高之一"，已知入选且质量低意味着什么？
- 控制碰撞变量后，估计的是什么效应？

**Answer:**
"入选精英计划"是碰撞变量：内容质量→入选←更新频率。入选标准是"质量≥8分或频率≥每周5更之一"——高质量创作者不需要高频更新也能入选，高频率创作者不需要高质量也能入选。

无条件时，质量和频率独立（无因果路径连接）。但控制"入选=1"后：如果某创作者质量低，则频率必须高才能入选；反之亦然。这创造了**条件负相关**——控制碰撞变量引入了原本不存在的关联。

如果产品团队据此降低高质量创作者的更新权重：
1. 策略基于虚假相关，无因果基础
2. 高质量创作者更新减少，长期内容供给下降
3. 平台内容生态受损

**决策规则：**
- 任何分析前 → 画DAG图，识别碰撞变量（被处理和结果共同影响）
- 碰撞变量及其后代 → **绝不控制**
- 控制后出现"意外相关" → 立即检查是否控制了碰撞变量
- 策略建议必须基于因果推断，而非条件相关

### Q4: 发现"搜索用户留存率高3倍"后，该做什么？不该做什么？

**Hints:**
- 思考"使用搜索的用户"和"不使用搜索的用户"是否是同一群人
- PSM能解决"观测混杂"，但能解决"未观测混杂"吗？
- A/B测试的"随机让部分用户看到搜索引导"是否存在伦理或用户体验问题？

**Answer:**
**标准路径：相关发现 → 因果假设 → 实验验证。**

**不该做方案A（全量推送）**：搜索用户留存高15%可能是健康用户偏误——高活跃度用户才使用搜索，搜索是"活跃度"的结果而非原因。强制低活跃度用户使用搜索不会提高其活跃度，可能反而干扰用户体验。美团案例：全量推搜索入口后留存无变化，三个月资源白费。

**可做方案B（PSM）**：用现有数据估计因果效应，但只能处理观测混杂（如历史活跃度、使用时长）。"用户内在动机"等未观测因素仍可能偏倚结果。PSM结果应标注"基于观测数据，存在未观测混杂风险"。

**优先方案C（A/B测试）**：随机让部分用户看到搜索引导，是因果推断的黄金标准。虽然"最慢"，但能给出确定性结论。如果实验成本高，可先小流量测试。

**决策规则：**
- 相关发现 → 先声明"相关"，提出多种解释
- 如果决策成本高（如全量功能开发） → 必须经过实验验证
- 如果决策成本低（如文案调整） → 可用PSM快速估计，但需标注置信度
- 任何基于观察性数据的因果推断 → 必须过"实验关卡"才能转化为策略

### Q5: 多个"相关发现"同时出现时，怎么判断哪个值得投入实验验证？

**Hints:**
- 思考Hill准则中哪些标准在业务场景中最实用
- "时间性"（因在前果在后）为什么是必要的？
- "实验证据"和"一致性"哪个更重要？

**Answer:**
**优先验证标准（按重要性排序）：**

1. **时间性（必要性）**：如果"因"发生在"果"之后，不可能是因果。如"收藏"发生在"购买"之前，可能因果；"死亡率"和"医院床位"无明确时间顺序，难判断。

2. **机制合理性**：是否存在可信的因果路径？如"新手任务→熟悉产品→留存"合理；"搜索→留存"的机制是"搜索满足需求"，但健康用户偏误更强。

3. **实验证据（高优先级）**：能否设计低成本实验验证？如"新手任务"可用小流量A/B测试快速验证；"医院床位"无法随机化。

4. **一致性**：不同场景是否重复发现？如"高级报表→续约"只在SaaS发现，一致性弱；"任务完成→留存"在多个产品重复，一致性强。

5. **特异性（可忽略）**：现代观点认为特异性不重要——一个因素可以有多个结果。

**五个案例的优先级判断：**
- 案例5（医院床位/死亡率）：反向因果明显，排除
- 案例3（评论数/播放量）：碰撞变量陷阱，排除
- 案例2（收藏/GMV）：反向因果可能（高购买意愿才收藏），低优先级
- 案例4（高级报表/续约）：健康用户偏误，低优先级
- 案例1（新手任务/留存）：时间性清晰、机制合理、可实验验证，**最高优先级**

**决策规则：**
- 时间性不满足 → 排除
- 无合理机制 → 低优先级
- 可低成本实验验证 + 机制合理 → 最高优先级
- 不可实验 + 观察性结论 → 必须附带敏感性分析
