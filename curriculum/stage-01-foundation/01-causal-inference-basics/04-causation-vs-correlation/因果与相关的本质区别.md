# 因果 vs 相关：本质区别

## 问题引入

夏天到了，冰淇淋销量和溺水事故同时上升。某媒体据此发文呼吁"限制冰淇淋销售以减少溺水"。你作为数据分析师，一眼就能看出问题所在：两者背后共同的原因是"气温升高"——天热人们既爱吃冰淇淋也爱去游泳，冰淇淋和溺水只是"一起发生"，而非"一个导致另一个"。

但在真实的业务场景中，"相关≠因果"的陷阱远比冰淇淋案例隐蔽。某内容平台发现"使用搜索功能的用户留存率是未使用者的3倍"，产品团队据此投入三个月开发"搜索引导功能"。上线后留存无变化。问题出在哪？使用搜索的用户本身就是高活跃度用户——"搜索行为"和"高留存"都是"高活跃度"的结果，而非因果关系。用户不是因为搜索而留存，而是因为活跃才既搜索又留存。

人脑天生喜欢因果叙事。看到A和B一起出现，我们本能地想"A导致B"。这种认知偏误在数据分析中被放大：统计软件能轻松跑出"显著相关"，但它不会告诉你这个相关是因果、反向因果、混杂还是纯粹的巧合。在[[stage-01-foundation/01-causal-inference-basics/01-potential-outcomes/潜在结果框架|潜在结果框架]]中，我们知道因果需要反事实；在[[stage-01-foundation/01-causal-inference-basics/03-observational-challenges/观察性数据的挑战|观察性研究]]中，我们知道混杂无处不在。但回到最根本的问题："相关"和"因果"在数学上到底有什么区别？为什么数据只能给我们相关，却给不了因果？有没有一个清晰的框架，能帮我们判断"这个相关能不能当成因果来用"？

这些问题不只是哲学思辨，而是每天都在烧钱烧资源的业务现实。把相关当因果，三个月迭代白费；把因果当相关，错失真正的增长机会。理解两者的本质区别，是数据分析师的第一课，也是最难的一课。

## 一句话总结

> **相关是"一起看"，因果是"如果改变"。**

## 形式化定义

### 相关性（Association）

$$P(Y|X) \neq P(Y)$$

- 知道 $X$ 的值，改变了我们对 $Y$ 的信念
- **对称的**：$X$ 与 $Y$ 相关 = $Y$ 与 $X$ 相关

### 因果性（Causation）

$$P(Y | do(X=x)) \neq P(Y)$$

- **干预** $X$ 为 $x$，改变了 $Y$ 的分布
- **不对称的**：$X$ 导致 $Y$ 不意味着 $Y$ 导致 $X$
- Pearl 的 $do$-算子明确表示干预

## 相关 ≠ 因果的常见原因

### 1. 混杂（Confounding）

```
Z → X
Z → Y
```

$X$ 和 $Y$ 相关，但改变 $X$ 不改变 $Y$（因为 $Z$ 是共同原因）。

**经典例子：**
- 冰淇淋销量与溺水事故正相关
- 共同原因：气温
- 结论：禁售冰淇淋不会减少溺水

### 2. 反向因果（Reverse Causality）

```
Y → X
```

$X$ 和 $Y$ 相关，但因果方向相反。

**例子：**
- 医院床位越多，地区死亡率越高
- 真相：死亡率高的地区建更多医院

### 3. 共同结果（Collider）

```
X → Z ← Y
```

控制 $Z$ 后，$X$ 和 $Y$ 产生虚假相关。

**例子：**
- 控制"入选精英项目"后，技能和努力负相关
- 真相：入选需要技能或努力之一即可

### 4. 偶然相关（Spurious Correlation）

- 纯粹巧合
- Tyler Vigen 的《Spurious Correlations》网站有大量例子
- 例如：人均奶酪消费量与床单缠绕致死人数

## 为什么区分这么难

| 挑战 | 说明 |
|---|---|
| **数据只给相关** | 观测数据永远显示 $P(Y|X)$，不显示 $P(Y|do(X))$ |
| **反事实不可观测** | 同一时刻只能看到一种状态 |
| **人脑爱因果叙事** | 进化让我们偏好因果解释，即使数据不支持 |
| **统计方法不保证因果** | 回归系数 ≠ 因果效应，除非满足强假设 |

## 从相关到因果的路径

```
相关发现
    ↓
提出因果假设（基于理论/机制）
    ↓
设计研究消除替代解释
    ↓
随机化实验（黄金标准）
    ↓
或：观察性研究 + 强假设 + 敏感性分析
    ↓
因果结论（附置信度声明）
```

## 实践中的判断框架

### 三个问题

1. **是否有混杂？** 是否有第三方因素同时影响两者？
2. **方向是否明确？** 时间顺序是否清晰？
3. **机制是否合理？** 是否存在可信的因果路径？

### Hill 因果准则（1965）

Bradford Hill 提出的评估因果关系的标准：

| 准则 | 说明 | 重要性 |
|---|---|---|
| 强度 | 关联强度大 | 中等 |
| 一致性 | 不同研究重复发现 | 高 |
| 特异性 | 特定暴露 → 特定结果 | 低（现代观点） |
| 时间性 | 因在前，果在后 | **必要** |
| 生物梯度 | 剂量-反应关系 | 高 |
| 合理性 | 与现有知识一致 | 中等 |
| 实验证据 | 实验支持 | **高** |
| 类比 | 类似因素已知因果 | 低 |

## 数据分析师的实用建议

### 当发现相关时

1. **先声明"相关"**："X 与 Y 正相关（r=0.3）"
2. **提出可能解释**：
   - "可能 X 导致 Y（机制是...）"
   - "可能 Y 导致 X"
   - "可能 Z 导致两者"
3. **说明验证方式**："要验证因果，需要..."
4. **避免**："X 导致 Y"（除非有实验支持）

### 文档写作模板

**观察性数据：**
> 基于观察性数据，X 与 Y 呈正相关（r=0.3, p<0.01）。**可能解释**包括：X 影响 Y（通过...机制）、存在未观测混杂、或反向因果。**严格验证**需随机化实验。

**实验数据：**
> 随机化实验显示，X 使 Y 增加 0.5pp（95% CI: [0.2, 0.8]），结果稳健。

## 行业使用经验

### 经验1：搜索功能与留存的"健康用户偏误"

美团外卖2023年Q2，我们发现"使用搜索功能的用户30日留存率比未使用用户高15%"，产品团队据此全量推大搜索入口。上线后留存无变化。复盘发现：会使用搜索的用户本身就是高活跃度用户（健康用户偏误），搜索行为是结果而非原因。把相关当因果，三个月迭代资源白费。

### 经验2：收藏功能与GMV的反向因果

某电商2022年大促前，数据发现"收藏商品的用户GMV是未收藏用户的3倍"，运营强推"一键收藏领券"功能。结果大促期间GMV反而下降8%。真相是：高购买意愿用户才会主动收藏，强制收藏打断了自然购买路径，造成反向因果的干预灾难。

### 经验3：评论数与播放量的碰撞变量陷阱

某短视频平台2021年发现"评论数>100的视频平均播放量是评论数<10视频的5倍"，产品引导用户"先评论再观看"。上线一周后核心创作者反馈播放量下降30%。问题出在碰撞变量——只有高播放量的视频才会积累大量评论，强行干预评论行为破坏了内容消费的自然链路。

### 经验4：我们曾犯的错误——把相关当因果烧了三个月预算

2020年我们负责某内容平台的用户增长，发现"完成新手任务的用户7日留存比未完成用户高40%"，据此投入三个月研发资源重做新手引导流程。上线后留存曲线纹丝不动。复盘时才意识到：愿意做新手任务的用户本身就是高动机用户，任务完成度只是自选择的结果。三个月团队资源、服务器成本、机会成本全部沉没。这个教训让我们建立了"相关发现→因果假设→实验验证"的强制流程，任何基于观测数据的因果推断必须过实验关卡。
