# OLS回归（普通最小二乘法）

## 问题引入

假设你是一家外卖平台的数据分析师。运营团队想知道："配送费每涨1块钱，订单量会下降多少？"你拉了一张散点图，横轴是配送费，纵轴是订单量，发现大致呈一条向下的直线趋势。但这条线怎么画才最"合理"？肉眼随手画一条肯定不行，你需要一个客观的标准。

更复杂的场景是：订单量不仅受配送费影响，还受天气、时段、用户历史活跃度、商家评分等一堆因素共同作用。运营负责人追问："在控制其他因素不变的情况下，配送费的独立影响是多少？"这就需要同时考虑多个变量对结果的联合影响。

OLS回归就是回答这类问题的标准工具。它的核心思想极其朴素：找到一条直线（或多个变量构成的超平面），让所有数据点到这条线的"垂直距离"（残差）的平方和最小。为什么用平方和而不是绝对值？因为平方对极端值更敏感，同时有漂亮的解析解。这条"最小二乘"线，就是数据中最能代表变量间线性关系的估计。

但OLS的"简单"背后藏着严格的假设。它假设X和Y的关系是线性的，假设误差项与自变量不相关（外生性），假设误差方差恒定（同方差），假设自变量之间不太相关（无多重共线性）。在真实业务中，这些假设经常被违反：配送费和订单量的关系可能是非线性的（涨到一定程度后订单量断崖式下降）；高配送费区域的订单量波动可能更大（异方差）；配送费和距离高度相关（多重共线性）；最重要的是——配送费不是随机分配的，商家定价高的区域可能本身需求就弱（内生性）。

理解OLS什么时候可靠、什么时候会骗人，是数据分析师的核心能力。OLS回归给出的系数，在观察性数据中永远只是"条件相关"，不是"因果效应"。要谈因果，你需要[[stage-01-foundation/01-causal-inference-basics/02-randomized-experiments/随机化实验原理|随机化实验]]或更精巧的识别策略。

## 基本原理

OLS 通过最小化残差平方和来估计线性关系。核心思想：找到一条直线（或超平面），使得所有数据点到该直线的垂直距离（残差）的平方和最小。

**目标函数：**
$$\min_{\beta} \sum_{i=1}^n (Y_i - \beta_0 - \beta_1 X_{i1} - ... - \beta_k X_{ik})^2$$

**解析解（矩阵形式）：**
$$\hat{\beta} = (X'X)^{-1}X'Y$$

## 使用场景

| 场景 | 说明 | 示例 |
|---|---|---|
| 预测连续变量 | 基于特征预测数值结果 | 预测用户LTV（生命周期价值） |
| 量化关系强度 | 估计X对Y的边际效应 | 每增加1次访问，下单概率增加多少 |
| 控制混杂变量 | 在回归中加入协变量 | 评估活动效果时控制用户历史活跃度 |
| 趋势分析 | 时间序列的趋势拟合 | 月度GMV的线性趋势 |

## 局限性

1. **线性假设**：X与Y的关系必须是线性的，非线性关系需变换或改用其他方法
2. **外生性假设**：误差项与自变量不相关（$E[\epsilon|X]=0$），违反则估计有偏
3. **同方差性**：误差方差恒定，异方差时标准误无效
4. **无多重共线性**：自变量间高度相关会导致估计不稳定
5. **异常值敏感**：极端值对系数估计影响大

## 同类算法对比

| 方法 | 适用条件 | 优势 | 劣势 |
|---|---|---|---|
| **OLS** | 线性关系，连续因变量 | 解析解，可解释性强 | 对异常值敏感，假设严格 |
| **Ridge回归** | 多重共线性，预测为主 | L2正则化，稳定 | 系数有偏，解释性降低 |
| **Lasso回归** | 高维特征，特征选择 | L1正则化，自动选择特征 | 系数有偏，标准误难计算 |
| **稳健回归** | 有异常值 | 对异常值不敏感 | 效率损失，计算复杂 |
| **分位数回归** | 关注条件分位数 | 描述完整分布 | 计算量大，解释复杂 |

## 真实案例

### 案例1：LTV预测模型（C端运营）

**背景：** 基于用户早期行为预测其12个月LTV，用于新用户分层运营。

**变量：**
- 因变量：12个月LTV（元）
- 自变量：首月访问次数、首月下单金额、注册渠道、是否使用优惠券

**模型：**
```
LTV = 50 + 3.2×首月访问 + 0.8×首月下单 + 15×优惠券使用 + ε
```

**分析：**
- R² = 0.42，说明模型解释了42%的变异
- 每多访问1次，LTV增加3.2元（控制其他变量后）
- 使用优惠券的用户LTV高15元（可能因优惠券吸引了高价值用户）

### 案例2：活动效果评估（离线分析）

**背景：** 评估"满减活动"对客单价的影响，控制用户历史行为。

**模型：**
```
客单价 = β₀ + β₁×活动参与 + β₂×历史客单价 + β₃×会员等级 + ε
```

**结果：**
- 活动参与系数 = +28.5（p<0.001）
- 但R²仅0.15，说明活动只解释了客单价变异的少量部分
- 结论：活动有显著效果，但用户自身差异是主要因素

### 案例3：商家流失预警（B端运营）

**背景：** 预测商家未来3个月是否流失（转化为连续概率）。

**变量：**
- 因变量：未来3个月GMV（连续，接近0则流失）
- 自变量：近30天登录次数、订单量、投诉次数、行业类目

**分析：**
- 登录次数系数 = +120（每多登录1次，GMV增加120元）
- 投诉次数系数 = -800（每多1次投诉，GMV减少800元）
- 用于构建流失评分：低预测GMV的商家优先干预

## 行业使用经验

### 经验1：R²=0.42不是"模型很好"，而是"还有58%的东西你没解释"

某大厂LTV预测模型R²=0.42，团队庆祝"解释了42%的变异"。但业务方问："那剩下的58%是什么？"——可能是未观测的用户偏好、竞品行为、宏观经济波动。更关键的是：模型对高LTV用户的预测系统性偏低（残差异方差），导致运营资源错配。**建议：报告R²时必须同时展示残差图。若残差呈现模式（如漏斗形），说明模型设定有问题，R²再高也不可信。**

### 经验2：多重共线性不会"让系数变号"，但会让标准误膨胀到无法判断

某外卖平台用OLS估计"配送费"对"订单量"的影响，同时控制了"客单价""距离""时段"。发现"配送费"系数为正（+0.5，p=0.3），与直觉相反。排查VIF发现"客单价"和"距离"的VIF=18（严重共线性）。移除"距离"后，配送费系数变为-2.1（p<0.01），符合预期。**规则：回归前先跑VIF矩阵，任何VIF>10的变量必须处理（删除、PCA或正则化），否则系数解读是危险的。**

### 经验3：我们曾犯的错误——把"显著"的系数当成"因果"上报

某内容平台用OLS估计"使用搜索功能"对"留存"的影响，控制了一堆用户特征后，系数显著为正（+15%）。产品团队据此全量推广搜索入口。但问题在于：用户"使用搜索"不是随机分配的——高活跃用户更可能搜索，而高活跃用户本身就更容易留存。OLS的外生性假设（误差项与自变量不相关）被违反，系数是偏的。**事后复盘：观察性数据中的OLS系数，永远只是"条件相关"，不是"因果效应"。要谈因果，必须有随机实验、工具变量或DAG支持的识别策略。**

### 经验4：异方差不是"小问题"，White标准误是必做项

滴滴某实验离线分析用OLS估计处理效应，残差图显示"处理组残差方差是对照组的3倍"（异方差）。普通标准误下p=0.04（"显著"），White稳健标准误下p=0.11（不显著）。若按普通标准误上报，可能导致错误全量。**规范：任何OLS回归，默认使用稳健标准误（HC3）。在Python的statsmodels中，添加cov_type='HC3'只需一个参数，但能避免大量错误决策。**
