# Logistic回归

## 问题引入

假设你是一家SaaS平台的数据分析师。市场团队想知道："哪些因素最能预测一个注册用户会在7天内完成首单？"你拉出的数据中，因变量只有两种结果——"转化"或"未转化"。如果你用OLS回归，预测值可能跑出负数或大于1的概率，这在数学上毫无意义。你需要一种专门处理"是/否"结果的回归方法。

更棘手的是业务解释问题。运营负责人不只想知道"哪些因素重要"，还想知道"每多一个因素，转化的几率增加多少倍"。比如"领过优惠券的用户转化几率是未领券用户的多少倍"——这种"几率比"（Odds Ratio）的解释方式，在业务沟通中远比"系数+0.3"直观。

Logistic回归就是为二分类问题设计的。它通过一个S形的Sigmoid函数，把线性组合映射到(0,1)的概率区间，保证预测值永远是合理的概率。同时，它的系数经过指数化后，直接变成Odds Ratio，业务方一听就懂："加购行为的Odds Ratio是3.3，意味着加购用户转化的几率是未加购用户的3.3倍。"

但Logistic回归也有它的问题。它只能学习线性的决策边界，复杂的非线性关系需要大量的特征工程；当某个特征能完美预测结果时（比如"未实名认证的用户100%不能发布内容"），模型会遭遇"完全分离"，系数趋向无穷大；在类别极度不平衡的数据中（比如流失率只有5%），伪R²可能很好看，但模型的实际区分能力很差。

和[[stage-01-foundation/02-statistical-basics/04-ols-regression/OLS回归|OLS回归]]一样，Logistic回归在观察性数据中给出的系数也只是"条件相关"，不是"因果效应"。如果你用它来做Uplift Modeling（估计"发券带来的增量转化"），可能会因为混杂因素而严重偏误。理解它的能力和边界，才能避免"模型AUC很高，上线后ROI为负"的悲剧。

## 基本原理

Logistic回归用于二分类问题，通过Sigmoid函数将线性组合映射到(0,1)概率区间。

**模型形式：**
$$P(Y=1|X) = \frac{1}{1 + e^{-(\beta_0 + \beta_1 X_1 + ... + \beta_k X_k)}}$$

**对数几率（Logit）：**
$$\log\frac{P}{1-P} = \beta_0 + \beta_1 X_1 + ... + \beta_k X_k$$

**估计方法：** 最大似然估计（MLE），无解析解，需迭代优化。

## 使用场景

| 场景 | 说明 | 示例 |
|---|---|---|
| 二分类预测 | 预测事件发生概率 | 用户是否会在7日内下单 |
| 风险评估 | 估计风险概率 | 商家未来30天流失概率 |
| 效果评估（概率尺度） | 控制混杂后估计处理效应 | 活动参与对转化率的影响 |
| 特征重要性排序 | 系数符号和大小指示方向强度 | 哪些因素驱动用户升级 |

## 局限性

1. **线性决策边界**：在特征空间中只能学习线性边界，需特征工程处理非线性
2. **完全分离问题**：某特征完全预测结果时，MLE不收敛
3. **样本不平衡**：正负样本比例极端时，预测偏 Majority Class
4. **多重共线性**：与OLS类似，影响系数稳定性
5. **遗漏变量偏误**：遗漏相关变量导致估计有偏

## 同类算法对比

| 方法 | 适用条件 | 优势 | 劣势 |
|---|---|---|---|
| **Logistic回归** | 二分类，需要可解释性 | 系数可解释，计算快 | 只能线性边界 |
| **Probit回归** | 二分类，误差正态假设 | 与Logit结果接近 | 计算稍复杂 |
| **决策树** | 非线性关系 | 自动学习交互和非线性 | 不稳定，过拟合 |
| **SVM** | 高维数据 | 泛化好，核技巧处理非线性 | 黑盒，训练慢 |
| **XGBoost** | 复杂模式，预测为主 | 精度高，自动特征交互 | 黑盒，需调参 |

## 真实案例

### 案例1：用户转化预测（C端运营）

**背景：** 预测注册用户7日内是否完成首单，用于精准营销。

**变量：**
- 因变量：7日内首单（0/1）
- 自变量：注册渠道、首小时浏览次数、是否加购、是否领券、设备类型

**模型结果：**
```
logit(P) = -2.1 + 0.8×领券 + 1.2×加购 + 0.5×首小时浏览3次+ + 0.3×iOS
```

**解释：**
- 领券用户转化 odds 是未领券的 e^0.8 ≈ 2.2 倍
- 加购用户转化 odds 是未加购的 e^1.2 ≈ 3.3 倍
- 模型AUC=0.78，用于分层触达（预测概率>0.6的用户push提醒）

### 案例2：商家流失预警（B端运营）

**背景：** 预测商家未来30天是否流失（GMV<100元），提前干预。

**变量：**
- 因变量：30天流失（0/1）
- 自变量：近7天登录次数、订单量环比下降、投诉次数、合同到期天数

**分析：**
- 订单环比下降>50%：流失 odds 增加 4 倍
- 近7天未登录：流失 odds 增加 3.5 倍
- 用于生成"流失风险评分"，高优先级商家分配运营跟进

### 案例3：活动响应预测（离线分析）

**背景：** 预测用户对"双11活动"的响应概率，优化触达策略。

**变量：**
- 因变量：活动期间是否下单（0/1）
- 自变量：历史活动参与度、月均消费、品类偏好匹配度、优惠券敏感度

**应用：**
- 预测概率<0.2的用户：不触达（节省成本）
- 预测概率0.2-0.6的用户：短信触达
- 预测概率>0.6的用户：APP推送+短信双重触达

## 关键词

- **Odds**：发生比，P/(1-P)，如 odds=2 表示发生概率是2倍于不发生
- **Odds Ratio**： odds 的比值，e^β，系数指数化后的解释
- **Logit**：对数 odds，线性部分，系数直接可加
- **Sigmoid**：S形函数，将实数映射到(0,1)
- **最大似然估计**：通过最大化观测数据的似然函数来估计参数
- **伪R²**：McFadden R²等，类比线性回归R²，但解释不同
- **Hosmer-Lemeshow检验**：拟合优度检验，评估预测概率与实际频率的一致性

## 行业使用经验

### 经验1：训练集AUC=0.85但测试集仅0.72的过拟合教训

某出行平台2022年Q2用Logistic回归预测司机30日流失，训练集AUC=0.85，业务方很满意。上线后发现预测概率集中在0.1-0.3区间，几乎无法区分高低风险。测试集AUC=0.72，与训练集差距13个百分点。原因是：使用了"近30日完单量"等未来信息作为特征（数据泄露），训练时模型"作弊"看到了结果。修正后AUC降至0.68，但稳定可复现。这个案例让我们建立"时间切分测试集"的强制流程，禁止随机切分。

### 经验2：用Logistic回归做Uplift Modeling的失败

某电商平台2021年尝试用Logistic回归做"优惠券发放"的Uplift Modeling，分别建"发券组转化模型"和"不发券组转化模型"，两者相减得Uplift。结果上线后ROI为负。根本问题：两组的Logistic模型各自优化的是"给定发券/不发券条件下的转化率"，而非"发券带来的增量"。发券组模型学习到的混杂因素（如高价值用户更可能收到券）被误当成效应。后改用S-Learner和因果森林，Uplift估计才趋于合理。Logistic回归不是为因果效应设计的。

### 经验3：完全分离问题的检测与Firth回归

某内容平台2023年分析"是否完成实名认证"对"能否发布内容"的影响，发现所有未认证用户都被禁止发布（平台规则）。Logistic回归MLE不收敛，系数趋向无穷大。团队最初以为是代码bug，后识别为完全分离（Complete Separation）。改用Firth回归（惩罚似然）后，系数估计稳定且有限，给出合理的效应方向。这个经验让我们建立规则：二分类模型训练前必须检查分离表，一旦发现零单元格立即切换Firth或贝叶斯方法。

### 经验4：我们曾犯的错误——用训练集伪R²说服业务，上线后预测概率全部偏向一侧

2020年我们向业务方汇报用户流失预警模型，展示McFadden R²=0.42"解释力很强"。业务方据此投入运营人力对"高风险用户"进行电话挽留。上线后发现：模型预测概率90%用户<0.2，10%用户>0.8，几乎没有中间地带，运营团队无法分层触达。更深层的问题是：伪R²在类别不平衡数据下（流失率仅5%）天然偏高，与模型的实际区分能力无关。我们用伪R²的"好看数字"掩盖了模型校准（Calibration）的失败。此后我们禁止用伪R²作为模型评估指标，强制要求校准曲线和Brier Score。

## 代码示例

```python
import numpy as np
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import confusion_matrix, accuracy_score

np.random.seed(42)

# 生成二分类数据
x = np.random.normal(loc=5, scale=2, size=200).reshape(-1, 1)
logit = -3.0 + 0.8 * x.flatten()
prob = 1 / (1 + np.exp(-logit))
y = (np.random.uniform(size=200) < prob).astype(int)

# 拟合 Logistic 回归
model = LogisticRegression(solver='lbfgs')
model.fit(x, y)

beta0 = model.intercept_[0]
beta1 = model.coef_[0][0]
print(f"估计模型：logit(p) = {beta0:.3f} + {beta1:.3f} * x")
print(f"优势比 OR = exp({beta1:.3f}) = {np.exp(beta1):.3f}")

# 混淆矩阵
y_pred = model.predict(x)
cm = confusion_matrix(y, y_pred)
print(f"准确率 = {accuracy_score(y, y_pred):.3f}")
print("混淆矩阵：")
print(cm)
```
