# 好问练习解答 - Logistic回归

> 对应知识点：[[stage-01-foundation/02-statistical-basics/05-logistic-regression/Logistic回归|Logistic回归]]

---

### Q1: 预测用户"转化/未转化"，OLS跑出预测值1.3和-0.2——为什么概率不能大于1或小于0？

**Hints:**
- OLS的线性预测 $X\beta$ 取值范围是什么？概率的取值范围是什么？
- 需要什么样的函数，能把 $(-\infty, +\infty)$ 映射到 $(0, 1)$？

**Answer:**
OLS不适合二分类问题的核心原因：线性预测 $X\beta$ 可以取任意实数值，但概率必须在(0,1)区间内。强行用OLS会导致：
- 预测值超出[0,1]，数学上无意义
- 误差项不服从正态分布（0/1数据的误差是异方差的）
- 线性假设不合理：X从10变到11对概率的影响，与X从100变到101的影响应该不同（边际效应递减）

Logistic回归通过Sigmoid函数解决：
$$P(Y=1|X) = \frac{1}{1 + e^{-X\beta}}$$

**决策规则：**
- 因变量是二分类（0/1） → 必须用Logistic回归或Probit回归，禁止OLS
- 需要预测概率且解释边际效应 → Logistic回归
- 需要可解释的"几率比"（Odds Ratio） → Logistic回归（Probit无此性质）
- 因变量是连续数值 → OLS，不用Logistic

### Q2: Logistic回归和Probit回归都处理二分类，面对"用户是否转化"该用哪个？

**Hints:**
- Logistic分布和正态分布的尾部厚度有什么区别？
- 在实际业务数据中，这个尾部差异是否足以影响结论？

**Answer:**
绝大多数情况下，两者结论几乎相同。选择依据：

| 维度 | Logistic回归 | Probit回归 |
|------|-------------|-----------|
| **链接函数** | Sigmoid（Logistic分布CDF） | 正态分布CDF |
| **尾部厚度** | 更厚（类似t分布） | 更薄（指数衰减） |
| **系数解释** | 可解释为Log Odds和Odds Ratio | 无直观解释 |
| **计算** | 最大似然估计，标准实现 | 同样需要MLE |

**决策规则：**
- 需要向业务方解释系数 → 用Logistic回归（Odds Ratio直观）
- 不需要解释系数，只关心预测概率 → 两者均可，选更熟悉的那个
- 数据有极端尾部（如金融违约数据） → Logistic更稳健，因为厚尾对极端观测不敏感
- 做经济学研究需要与文献对比 → 经济学传统多用Probit

**关键原则：** 如果Logistic和Probit给出矛盾结论，说明模型设定有问题（如遗漏关键变量、函数形式错误），而不是"选错了方法"。

### Q3: 训练集AUC=0.85，测试集仅0.72——我用了"近30日完单量"作为特征，为什么上线后模型失效？

**Hints:**
- "近30日完单量"在训练时是否包含了"未来30日"的信息？
- 数据泄露（Data Leakage）的本质是什么？是"特征太强"还是"时间结构被破坏"？

**Answer:**
你犯了**数据泄露**错误：特征"近30日完单量"在训练时包含了预测目标时间窗口内的信息。模型"作弊"看到了部分结果，训练集表现虚高。

**决策规则：**
- 任何时序数据建模，必须按**时间切分**训练/测试集：用t时刻之前的数据训练，预测t时刻之后的目标
- 禁止随机切分（shuffle split），这会破坏时间结构，导致未来信息泄露到训练集
- 特征工程时必须明确每个特征的"时间戳"：该特征在预测时刻是否已知？
- 建立强制流程：训练前审查每个特征的时间有效性，"未来特征"一律删除

**检测方法：**
- 如果某个特征的重要性异常高 → 怀疑数据泄露
- 如果训练集和测试集AUC差距>10个百分点 → 几乎肯定有泄露
- 如果预测概率分布极端集中（如90%预测<0.2） → 可能是泄露导致模型"过度自信"

### Q4: 流失率仅5%，模型McFadden R²=0.42"很好看"，但预测概率90%用户<0.2——我该信伪R²还是信校准曲线？

**Hints:**
- McFadden R²的基准模型是什么？在类别不平衡时，"击败基准"是否容易？
- 校准曲线（Calibration Curve）反映的是什么？伪R²能否检测校准失败？

**Answer:**
信校准曲线。伪R²在类别不平衡数据下**天然偏高**，与模型实际区分能力无关：

- McFadden R² = 1 - logL_model / logL_null。当流失率仅5%时，基准模型（全预测为"不流失"）已经能"解释"大部分变异，logL_null本身就很接近logL_sat（饱和模型），导致分母很小，R²容易被放大。
- 校准曲线反映"预测概率是否等于实际频率"：如果模型预测某群体流失概率0.3，实际流失率应该是约30%。校准失败意味着概率不可信。

**决策规则：**
- **禁止用伪R²作为模型评估指标**（McFadden、Cox-Snell、Nagelkerke均禁止）
- 必须报告：AUC（区分能力）、校准曲线/Brier Score（概率准确性）、Precision-Recall曲线（不平衡数据）
- 如果预测概率分布极端两极化 → 检查校准，可能是模型过拟合或特征泄露
- 向业务方解释："模型能排序风险高低（AUC=0.78），但预测的具体概率可能不准（Brier Score高），需要校准后使用"

### Q5: Logistic回归做Uplift Modeling上线后ROI为负——我用"发券组模型减不发券组转化模型"得增量，为什么错了？

**Hints:**
- 发券组模型估计的是 $P(Y=1|T=1, X)$，这是"发券条件下的转化率"还是"发券带来的因果效应"？
- 两组模型各自优化的是什么？混杂因素（如高价值用户更可能收到券）进入了哪个部分？

**Answer:**
你犯了**因果混淆**错误：两组Logistic模型各自估计的是"给定处理条件下的转化率"，而非"处理带来的增量"。

根本问题：
- 发券组模型：$P(Y=1|T=1, X)$ —— 高价值用户更可能收到券，模型学到的"发券效应"混杂了用户价值
- 不发券组模型：$P(Y=1|T=0, X)$ —— 同样混杂了用户价值
- 两者相减：$P(Y=1|T=1, X) - P(Y=1|T=0, X)$ —— 这不是因果效应，因为两组用户不可比

**决策规则：**
- 估计因果效应（Uplift） → 必须用因果推断方法：S-Learner、T-Learner、X-Learner、因果森林
- 如果只有观察数据、无随机实验 → 用S-Learner（单一模型加入处理变量交互项），或倾向得分匹配后估计
- 如果有随机实验数据 → 直接用T-Learner或X-Learner
- Logistic回归不是为因果效应设计的，禁止用于Uplift估计

**关键原则：** "条件概率的差异"不等于"因果效应"。除非处理是随机分配的，否则 $P(Y|T=1) - P(Y|T=0)$ 是有偏的。
