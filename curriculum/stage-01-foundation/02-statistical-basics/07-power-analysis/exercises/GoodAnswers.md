# 好问练习解答 - 检验力分析与样本量计算

> 对应知识点：[[stage-01-foundation/02-statistical-basics/07-power-analysis/检验力分析与样本量计算|检验力分析与样本量计算]]

---

### Q1: 产品经理说"最好能检测到0.1%的点击率变化"，你算了算说需要200万用户/组，他立刻改口说"那0.5%吧"

**Hints:**
- 样本量公式 n = 16σ²/δ² 中，δ在分母且是平方关系，这意味着什么？
- 如果MDE从0.1%变为0.5%（5倍），样本量变化多少倍？
- "能检测"和"值得检测"之间差了什么？统计显著性是否等于业务意义？

**Answer:**
样本量与MDE的平方成反比：n ∝ 1/δ²。MDE从0.1%→0.5%（5倍），样本量从200万→8万（缩小25倍）。这是因为检测更小的差异需要更精确地估计均值差异，而精度与√n成正比。

"能检测"是统计概念（给定样本下可可靠检出的最小效应），"值得检测"是业务概念（业务方认为有意义的最小变化）。统计上能检测0.1pp变化，但如果业务上<1pp的变化不值得投入工程资源上线，那么检测0.1pp就是浪费。

**决策规则：**
- MDE必须同时满足：统计可检测 + 业务有意义 + 资源可承受
- 如果三个条件冲突 → 优先"业务有意义"，其次"资源可承受"，调整"统计可检测"
- 永远向业务方展示MDE-样本量-周期的权衡三角，让他们做知情选择

### Q2: 样本量公式 n = 16σ²/δ² 和精确公式，到底该用哪个？

**Hints:**
- "16法则"中16是怎么来的？它假设了什么？
- 精确公式中 Z₁₋α/₂ 和 Z₁₋β 在α=0.05（双侧）、Power=80%时分别是多少？
- 如果α=0.01或Power=90%，"16法则"还适用吗？

**Answer:**

**"16法则"的来源：** α=0.05（双侧）→ Z₁₋α/₂=1.96；Power=80% → Z₁₋β=0.84。2×(1.96+0.84)² = 2×7.84 = 15.68 ≈ 16。

**何时"16法则"会误导：**

| 场景 | 精确系数 | 16法则误差 |
|---|---|---|
| α=0.05, Power=80%（标准） | 15.7 | 无 |
| α=0.01, Power=80% | 21.0 | 低估25% |
| α=0.05, Power=90% | 21.0 | 低估25% |
| α=0.01, Power=90% | 26.7 | 低估40% |

**决策规则：**
- 标准设计（α=0.05双侧, Power=80%）→ "16法则"快速估算足够
- 严格设计（α=0.01或Power=90%）→ 必须用精确公式
- 非标准设计（不等组、非正态、聚类随机化）→ 用模拟法
- 向业务方汇报时，用精确公式展示专业性，用"16法则"快速心算验证

### Q3: 实验跑完了不显著，我说"可能是样本量不够"，老板反问"那你当初为什么没算对？"

**Hints:**
- 历史数据中可能包含什么导致方差估计失真？
- "最佳月份"的方差和"实验计划期"的方差为什么可能不同？
- 方差估计偏高和偏低分别会导致什么后果？

**Answer:**

**错误的根源：** 用"未经清洗的历史数据"或"最佳场景"估计方差，而非"实验计划期的代表性数据"。

- 历史数据含异常值（系统故障、大促）→ 方差被高估或低估
- "最佳月份"（平稳期）方差 < 实验期（淡季/旺季）方差 → 样本量不足
- 结构性变化（用户增长、产品迭代）→ 历史方差不再代表未来

**方差估计偏差的后果：**

| 偏差方向 | 样本量计算 | 实际实验 |
|---|---|---|
| 方差低估 | 计算样本量偏小 | 实际灵敏度不足，假阴性风险↑ |
| 方差高估 | 计算样本量偏大 | 浪费资源，但结论可靠 |

**决策规则：**
- 方差估计必须覆盖"实验计划期的完整季节周期"（至少一个完整周期）
- 禁止用"最佳场景"自欺欺人——用保守（偏高）方差估计
- 实验前做"方差敏感性分析"：展示不同方差假设下的样本量需求
- 建立规则：历史数据必须清洗异常值，方差估计期≥实验计划期的季节覆盖

### Q4: 同时监控5个指标，样本量该怎么调整？

**Hints:**
- Bonferroni校正假设指标间是什么关系？独立还是相关？
- 如果5个指标高度相关，Bonferroni校正会过度惩罚吗？
- "核心指标"和"护栏指标"在决策中的权重是否相同？

**Answer:**

**正确选择：选项C为主，核心指标严格检验，护栏指标探索性分析。**

**原因：**
- 选项A（Bonferroni）：假设指标独立，5个指标α=0.01，样本量需增加约2.6倍。但真实指标通常相关，Bonferroni过度保守，浪费样本。
- 选项B（不校正）：整体FWER=23%，不可接受。
- 选项C（分层检验）：核心指标α=0.05严格检验，护栏指标用FDR控制或仅看效应方向/置信区间，不做严格显著性检验。

**更精细的方案：**
- 明确假设结构：是"所有指标必须同时显著"（合取）还是"任一指标显著即可"（析取）？
- 合取假设：用Bonferroni或Holm校正
- 析取假设：用FDR控制（Benjamini-Hochberg），样本量增加更少
- 护栏指标：不追求显著性，关注"效应方向+置信区间是否包含不可接受的负面效应"

**决策规则：**
- 核心指标（决定上线）→ 严格α控制，单独计算样本量
- 护栏指标（决定不下线）→ 关注效应方向和置信区间，不追求显著性
- 探索性指标 → 不做多重检验校正，但结论标记为"探索性"
- 多重检验校正方法必须与假设结构（合取/析取/混合）匹配

### Q5: 业务方说"变化<1pp不值得动"，统计方说"按1pp算需要跑6周"——MDE到底谁说了算？

**Hints:**
- MDE是纯粹的统计概念吗？它包含哪些非统计要素？
- "有意义"和"可检测"之间如何建立桥梁？
- 如果业务方坚持不合理的MDE，统计方的责任是什么？

**Answer:**

**MDE不是统计问题，是"业务意义 × 资源约束 × 时间窗口"的谈判结果。**

**"MDE三层对齐"流程：**

1. **业务层**：业务方提出"期望检测的效应"（如+5秒）——这是愿望，不一定是约束
2. **统计层**：分析师计算"给定资源下可检测的效应"（如+15秒/1周，或+5秒/6周）——这是物理约束
3. **管理层**：决策层权衡"业务价值 vs 时间成本"，确定最终MDE

**促成对齐的话术：**
- 不说"5秒不可能"，说"检测5秒需要6周，检测15秒只需1周。如果我们1周能上线验证，即使效应是15秒，业务价值是否值得？"
- 展示MDE-周期-样本量的权衡曲线，让业务方做知情选择
- 明确声明："统计上检测5秒是可行的，但资源约束下我们需要判断：等待6周的机会成本 vs 更早知道'至少15秒'的信息价值"

**决策规则：**
- 分析师的责任是展示约束和权衡，不是替业务方做决定
- 如果业务方坚持不可行的MDE → 明确书面记录风险（"按此MDE，实验可能因灵敏度不足而无法检出真实效应"）
- 优先推荐"快速验证大效应"的路径，而非"慢速精确测量小效应"——业务价值通常随时间递减
