# t检验

## 问题引入

假设你是一家电商平台的数据分析师，负责评估新上线的推荐算法是否真正提升了用户的平均下单金额。上线一周后，你拿到了实验组（使用新算法）和对照组（使用旧算法）各 500 名用户的下单金额数据。实验组平均下单金额为 328 元，对照组为 315 元，看起来新算法带来了 13 元的提升。

你兴奋地向业务方汇报，却被技术负责人打断："这 13 元的差异，到底是算法真的有效，还是仅仅是随机波动？"

这个问题直击要害。在数据分析中，**我们观察到的样本差异不等于总体差异**。即使两套算法效果完全相同，由于抽样的随机性，实验组的平均下单金额也完全可能碰巧高于对照组。如果仅凭样本均值的大小就下结论，我们可能会把"运气"当成"效果"，导致错误的业务决策。

这正是 t 检验要解决的问题：**在样本数据的基础上，判断两组均值之间的差异是否具有统计学意义**——也就是说，这种差异是否大到足以让我们相信它反映了真实的总体差异，而非随机误差。

t 检验是假设检验家族中最基础、最常用的工具之一。它由英国统计学家 William Gosset（笔名"Student"）于 1908 年提出，专门用于在小样本情况下判断均值差异的显著性。与 z 检验不同，t 检验不需要事先知道总体标准差，而是利用样本标准差进行估计，这使得它在现实业务场景中更加实用。

在 AB 测试中，t 检验几乎是判断实验效果是否显著的"默认武器"。无论是评估新功能对转化率的影响、新文案对点击率的作用，还是新策略对用户留存率的改变，只要核心指标是连续变量（如金额、时长、次数），t 检验都是首选的分析方法。理解 t 检验的原理和适用场景，是每一位数据分析师的必修课。

## 核心概念

### 1. 单样本 t 检验

单样本 t 检验用于判断**一个样本的均值是否显著不同于某个已知的理论值或目标值**。

**典型场景**：
- 某产品宣称平均加载时间为 2 秒，你抽样测得 30 次加载时间，想验证实际均值是否偏离 2 秒
- 公司设定的 KPI 是人均日活时长 15 分钟，你想检验当前用户群体是否达标

**假设设定**：
- 原假设 $H_0$：$\mu = \mu_0$（样本来自均值为 $\mu_0$ 的总体）
- 备择假设 $H_1$：$\mu \neq \mu_0$（双侧检验）；或 $\mu > \mu_0$ / $\mu < \mu_0$（单侧检验）

**检验统计量**：

$$
t = \frac{\bar{x} - \mu_0}{s / \sqrt{n}}
$$

其中：
- $\bar{x}$ 为样本均值
- $\mu_0$ 为假设的总体均值
- $s$ 为样本标准差
- $n$ 为样本量

**自由度**：$df = n - 1$

### 2. 独立样本 t 检验

独立样本 t 检验用于比较**两个独立组的均值是否存在显著差异**。这是 AB 测试中最常用的检验形式。

**典型场景**：
- 实验组 vs 对照组的平均下单金额比较
- 男性用户 vs 女性用户的平均使用时长比较
- 新用户 vs 老用户的平均客单价比较

**假设设定**：
- 原假设 $H_0$：$\mu_1 = \mu_2$（两组总体均值相等）
- 备择假设 $H_1$：$\mu_1 \neq \mu_2$

#### 2.1 方差齐性假设下的 t 检验（Student's t-test）

当两组方差相等时，使用合并方差（pooled variance）：

$$
s_p^2 = \frac{(n_1 - 1)s_1^2 + (n_2 - 1)s_2^2}{n_1 + n_2 - 2}
$$

$$
t = \frac{\bar{x}_1 - \bar{x}_2}{\sqrt{s_p^2 \left(\frac{1}{n_1} + \frac{1}{n_2}\right)}}
$$

**自由度**：$df = n_1 + n_2 - 2$

#### 2.2 方差不齐时的 t 检验（Welch's t-test）

当两组方差不相等时，使用 Welch 校正：

$$
t = \frac{\bar{x}_1 - \bar{x}_2}{\sqrt{\frac{s_1^2}{n_1} + \frac{s_2^2}{n_2}}}
$$

**自由度**（Welch-Satterthwaite 方程）：

$$
df = \frac{\left(\frac{s_1^2}{n_1} + \frac{s_2^2}{n_2}\right)^2}{\frac{(s_1^2/n_1)^2}{n_1-1} + \frac{(s_2^2/n_2)^2}{n_2-1}}
$$

> **实践建议**：在 AB 测试中，两组方差往往不相等（例如实验组可能引入更大的变异性）。因此，**默认使用 Welch's t-test 是更稳健的做法**。现代统计软件（如 Python 的 `scipy.stats.ttest_ind`）默认已采用 Welch 校正。

### 3. 配对 t 检验

配对 t 检验用于比较**同一组对象在两个不同条件下的测量值**，或**配对样本的差值是否显著不为零**。

**典型场景**：
- 同一批用户在改版前后的留存率变化
- 同一批商品在调价前后的销量变化
- 同一批员工在培训前后的绩效评分变化

**核心思想**：将配对数据转化为差值，然后对差值做单样本 t 检验。

设配对差值为 $d_i = x_{1i} - x_{2i}$，则：

$$
t = \frac{\bar{d}}{s_d / \sqrt{n}}
$$

其中：
- $\bar{d}$ 为差值的均值
- $s_d$ 为差值的标准差
- $n$ 为配对数

**自由度**：$df = n - 1$

> **关键优势**：配对设计消除了个体间的差异，只关注个体内变化，因此**统计功效通常高于独立样本检验**。在可行的情况下，优先采用配对设计。

### 4. t 统计量的构造

所有 t 检验的核心思想是一致的：

$$
t = \frac{\text{观察到的效应}}{\text{估计的标准误}} = \frac{\text{mean\_diff}}{SE}
$$

- **分子**：我们关心的效应大小（均值差）
- **分母**：该效应的抽样波动程度（标准误）

**t 值越大**，说明观察到的效应相对于抽样误差越大，越不可能由随机波动产生。

**p 值的含义**：在 $H_0$ 为真的前提下，观察到当前或更极端 t 值的概率。

- $p < 0.05$：通常认为结果"统计显著"
- $p < 0.01$：高度显著
- $p < 0.001$：极其显著

> **重要**：p 值**不是**原假设为真的概率，也不是实验成功的概率。它仅仅衡量了"在无效假设下，数据出现的罕见程度"。

### 5. 与置信区间的关系

t 检验与置信区间本质上是同一枚硬币的两面：

- **95% 置信区间不包含零** $\Leftrightarrow$ **双侧 t 检验的 $p < 0.05$**

例如，两组均值差的 95% 置信区间为 $[5, 21]$，我们可以同时得出：
1. 差异在 5% 水平上统计显著（因为区间不包含 0）
2. 真实差异有 95% 的概率落在 5 到 21 之间

**在报告结果时，同时给出 p 值和置信区间是最佳实践**。

### 6. t 检验 vs z 检验

| 条件 | 使用 z 检验 | 使用 t 检验 |
|---|---|---|
| 总体标准差 $\sigma$ 已知 | ✅ | — |
| 总体标准差 $\sigma$ 未知，用样本估计 | — | ✅ |
| 大样本（$n > 30$） | 近似可用 | 更精确 |
| 小样本（$n \leq 30$） | 不适用 | ✅ |

> 当样本量很大时，t 分布趋近于标准正态分布，t 检验与 z 检验的结果几乎相同。但在小样本情况下，t 分布的尾部更厚，使用 t 检验能更准确地控制第一类错误率。

## 关键假设

1. **独立性**：观测值之间相互独立（组内独立，组间独立）
2. **正态性**：数据近似服从正态分布，或样本量足够大（中心极限定理）
3. **方差齐性**（仅 Student's t-test）：两组方差相等；若不确定，使用 Welch's t-test

> **正态性检验**：小样本时可用 Shapiro-Wilk 检验或 Q-Q 图检查；大样本时正态性假设相对宽松。若严重偏离正态，考虑非参数替代方法（如 Mann-Whitney U 检验）。

## 常见误区

| 误区 | 正确理解 |
|---|---|
| "p 值越小，效应越大" | p 值反映的是统计显著性，与效应大小无关。大样本下极小的效应也能得到极小的 p 值 |
| "不显著就等于无效果" | 不拒绝 $H_0$ 不等于接受 $H_0$。可能是样本量不足导致的检验功效过低 |
| "t 检验只能用于正态分布数据" | 大样本下中心极限定理保证 t 检验的稳健性；小样本且严重偏态时才需要替代方法 |
| "多重比较不需要校正" | 同时进行多个 t 检验时，族系错误率会膨胀，需使用 Bonferroni 校正等方法 |
| "单侧检验可以随意使用" | 单侧检验需要事先确定方向，不能看了数据后再决定。滥用会夸大显著性 |
| "配对检验和独立检验可以互换" | 配对数据用独立检验会损失功效；独立数据用配对检验会得出错误结论 |

## 行业使用经验

### 经验 1：默认使用 Welch's t-test

在真实的 AB 测试中，实验组和对照组的方差几乎不可能完全相等。Welch's t-test 不需要方差齐性假设，且在方差相等时与 Student's t-test 结果几乎一致。Python 的 `scipy.stats.ttest_ind(equal_var=False)` 和 R 的 `t.test()` 默认都已采用 Welch 校正。**养成默认使用 Welch 的习惯，避免方差不齐带来的风险**。

### 经验 2：关注效应量，而非仅看 p 值

统计显著不等于业务显著。一个 p = 0.001 但均值差只有 0.3% 的实验，可能对业务毫无价值。建议同时报告：
- **Cohen's d**：标准化的效应量，$d = \frac{\bar{x}_1 - \bar{x}_2}{s_p}$，0.2 为小效应，0.5 为中等，0.8 为大效应
- **原始均值差及其置信区间**
- **样本量**：帮助读者判断统计功效是否充足

### 经验 3：我们曾犯的错误——忽略样本量计算就上线实验

某次实验中，我们在只有 200 样本/组的情况下就进行了 t 检验，结果 p = 0.12，"不显著"。但事后计算发现，要检测预期效应量，每组至少需要 800 样本。我们不是因为"没效果"而得到不显著结果，而是因为"样本不够"导致检验功效不足。现在，**所有实验上线前必须做样本量预估**，使用公式：

$$
n \approx \frac{16 \sigma^2}{\delta^2}
$$

其中 $\delta$ 为期望检测的最小效应量，$\sigma^2$ 为方差（对应 80% 功效、5% 显著性水平的经验公式）。

### 经验 4：配对设计能大幅提升检验功效

在评估某功能改版效果时，我们最初采用独立样本设计（随机抽取改版前、改版后各一组用户），结果不显著。后来改用配对设计（追踪同一批用户改版前后的行为），同样的效应量在配对检验中变得显著。原因是配对设计消除了用户个体间的巨大差异，使标准误大幅降低。**当可以追踪同一对象时，优先采用配对设计**。

## 代码示例

```python
import numpy as np
from scipy import stats

# 设置随机种子保证可复现
np.random.seed(42)

# ========== 1. 单样本 t 检验 ==========
# 场景：检验样本均值是否显著不等于 50
sample = np.random.normal(loc=52, scale=5, size=30)
t_stat, p_value = stats.ttest_1samp(sample, popmean=50)
print(f"单样本 t 检验: t={t_stat:.3f}, p={p_value:.4f}")

# ========== 2. 独立样本 t 检验 (Welch's) ==========
# 场景：比较实验组和对照组的均值
# 默认 equal_var=False，即 Welch's t-test
group_a = np.random.normal(loc=100, scale=15, size=100)
group_b = np.random.normal(loc=108, scale=20, size=100)
t_stat, p_value = stats.ttest_ind(group_a, group_b, equal_var=False)
print(f"独立样本 t 检验: t={t_stat:.3f}, p={p_value:.4f}")

# 计算均值差和 95% 置信区间
diff = np.mean(group_a) - np.mean(group_b)
se = np.sqrt(np.var(group_a, ddof=1)/len(group_a) + np.var(group_b, ddof=1)/len(group_b))
df = (np.var(group_a, ddof=1)/len(group_a) + np.var(group_b, ddof=1)/len(group_b))**2 / \
     ((np.var(group_a, ddof=1)/len(group_a))**2/(len(group_a)-1) + \
      (np.var(group_b, ddof=1)/len(group_b))**2/(len(group_b)-1))
ci = stats.t.interval(0.95, df, loc=diff, scale=se)
print(f"均值差: {diff:.2f}, 95% CI: [{ci[0]:.2f}, {ci[1]:.2f}]")

# ========== 3. 配对 t 检验 ==========
# 场景：同一批用户改版前后的指标对比
before = np.random.normal(loc=50, scale=10, size=40)
after = before + np.random.normal(loc=3, scale=5, size=40)  # 平均提升 3
t_stat, p_value = stats.ttest_rel(before, after)
print(f"配对 t 检验: t={t_stat:.3f}, p={p_value:.4f}")

# 计算 Cohen's d (配对)
d_mean = np.mean(after - before)
d_std = np.std(after - before, ddof=1)
cohens_d = d_mean / d_std
print(f"Cohen's d: {cohens_d:.3f}")
```
