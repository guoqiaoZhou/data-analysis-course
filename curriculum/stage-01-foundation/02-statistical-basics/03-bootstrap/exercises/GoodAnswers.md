# 好问练习解答 - Bootstrap 重抽样方法

> 对应知识点：[[stage-01-foundation/02-statistical-basics/03-bootstrap/Bootstrap重抽样方法|Bootstrap 重抽样方法]]

---

### Q1: 老板要我估计"典型创作者"月收入的置信区间，但数据严重右偏，均值被头部创作者拉得极高——传统方法为什么失效？

**Hints:**
- 中位数的标准误有没有解析公式？和均值的标准误有什么区别？
- 中心极限定理保护的是均值的抽样分布，中位数的抽样分布收敛速度如何？

**Answer:**
传统方法失效的两个原因：

1. **中位数没有简单标准误**：均值的标准误=σ/√n，有解析公式；中位数的抽样分布依赖于总体分布的密度函数在 median 处的值，没有通用的样本估计公式。

2. **正态近似不可靠**：中心极限定理保证大样本下均值趋近正态，但中位数的收敛速度更慢（需要n>1000甚至更大），且右偏分布下正态近似严重低估右尾。

Bootstrap通过有放回重抽样，直接从数据中"模拟"中位数的抽样分布，不依赖任何解析公式或分布假设。

**决策规则：**
- 如果估计均值且样本量>30 → 传统方法即可，Bootstrap可作验证
- 如果估计中位数、分位数、相关系数等复杂统计量 → Bootstrap是首选
- 如果数据严重偏态或有异常值 → Bootstrap比正态近似更可靠
- Bootstrap重抽样次数B≥10000是底线，B=1000仅作快速探索

### Q2: Bootstrap和置换检验（Permutation Test）都能做假设检验，它们的核心区别是什么？

**Hints:**
- Bootstrap的"有放回抽样"是在模拟什么？置换检验的"打乱标签"是在模拟什么？
- 两种方法构建的"零分布"（null distribution）是否相同？

**Answer:**
核心区别在目的和机制：

| 维度 | Bootstrap | 置换检验 |
|------|-----------|----------|
| **目的** | 估计统计量的抽样分布，构造置信区间 | 构造零假设下的检验统计量分布 |
| **抽样方式** | 有放回地从原样本抽 | 无放回地打乱标签 |
| **零分布** | 以原样本为中心的经验分布 | 假设"无效应"时的分布 |
| **适用范围** | 置信区间、标准误估计 | 假设检验（两组比较） |
| **对原假设的依赖** | 不直接依赖 | 必须假设零假设成立 |

**决策规则：**
- 如果需要置信区间或标准误 → 用Bootstrap
- 如果只需要p值（两组是否不同） → 置换检验更精确，因为它直接模拟零假设
- 如果两者结论矛盾 → 检查Bootstrap是否正确地中心化了（Bootstrap检验需要额外步骤调整零假设）
- 常见错误：把"打乱标签"当成Bootstrap——那是置换检验，不是Bootstrap

### Q3: 样本量只有35人，我用Bootstrap（B=500）估计置信区间，结果"显著"但无法复现——我错在哪？

**Hints:**
- Bootstrap的准确性依赖于"原样本对总体的代表性"。n=35时，原样本本身代表性如何？
- B=500和B=10000对置信区间尾部估计的稳定性有什么区别？

**Answer:**
你犯了两个错误：

1. **小样本下Bootstrap覆盖率不足**：Bootstrap假设"原样本的经验分布近似总体分布"。n=35时，经验分布与真实分布差距很大，Bootstrap分布系统性偏窄，置信区间覆盖率可能远低于95%。

2. **B=500不足以估计尾部**：95%置信区间需要估计2.5%和97.5%分位数，B=500时每个尾部只有约12个观测，分位数估计极不稳定。

**决策规则：**
- n<50时 → Bootstrap仅作辅助，主结论需用精确方法（如二项检验精确置信区间）
- n<100时 → Bootstrap可用，但需与精确方法交叉验证
- B≥10000是底线，B=500仅用于教学演示或快速探索
- 任何Bootstrap结论，必须报告B值和敏感性分析（B=5000 vs B=10000是否一致）

### Q4: 新功能灰度实验组仅80人，传统Z检验p=0.28（不显著），Bootstrap置信区间包含0——我该放弃还是继续扩大样本？

**Hints:**
- Bootstrap置信区间包含0，是否等价于"效应为0"？
- 区间的宽度反映了什么？宽度与样本量的关系是什么？

**Answer:**
不应该放弃，也不应该全量上线——应该扩大样本。Bootstrap CI=[-0.02, 0.11]包含0，说明"不能确认效应存在"，但不等于"效应不存在"。区间较宽（宽度0.13）反映样本量不足导致的估计不精确，而非效应本身很小。

**决策规则：**
- 如果CI包含0但全部偏向正值（如[-0.02, 0.11]） → "可能有效但样本不足，建议扩大样本重测"
- 如果CI对称跨0（如[-0.05, 0.05]） → "效应极小或不存在，建议放弃"
- 如果CI下限>0 → "确认有效，可全量上线"
- 样本量规划：用Bootstrap的SE估计，计算达到目标精度所需的n（n = (Z·SE/δ)²）

### Q5: 时间序列数据的自相关性让简单Bootstrap失效，但块Bootstrap又引入了新偏差——非独立数据该怎么处理？

**Hints:**
- 简单Bootstrap假设观测独立同分布（i.i.d.），时间序列违反了这个假设的哪一部分？
- 块Bootstrap的"块长度"选择，本质上是在平衡什么矛盾？

**Answer:**
时间序列数据的Bootstrap需要专门方法，核心矛盾是"保留相关性结构"与"保持足够有效样本量"之间的权衡：

**决策规则：**
- 数据有短期自相关（如几天内的用户行为） → 用块Bootstrap，块长度取自相关系数衰减到0的滞后阶数
- 数据有长期趋势或季节性 → 块Bootstrap可能不够，考虑平稳化Bootstrap（如差分后Bootstrap）
- 不知道相关性结构 → 用"最优块长度"选择算法（如Hall-Horowitz-Jing方法），或报告多个块长度的敏感性分析
- 如果块长度选择对结论敏感 → 结论本身不可信，需要更长的观测期或更复杂的模型（如ARMA Bootstrap）

**关键原则：**
- 简单Bootstrap对时间序列是"错误的方法"，不是"近似的方法"——它系统性地低估标准误
- 块Bootstrap的块长度没有唯一正确答案，敏感性分析是必做项
- 如果数据相关性极强（如金融高频数据），Bootstrap可能完全不适合，需用参数化方法
