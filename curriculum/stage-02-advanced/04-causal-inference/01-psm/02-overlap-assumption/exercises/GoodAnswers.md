# 好问练习解答 - 重叠假设

> 对应知识点：[[stage-02-advanced/04-causal-inference/01-psm/02-overlap-assumption/重叠假设|重叠假设]]

---

### Q1: 为什么PSM需要"重叠假设"？没有重叠会发生什么？

**Hints:**
- 思考PSM的核心逻辑：为处理组个体找一个"在倾向得分上相似"的对照组个体
- 如果没有相似的人，匹配会匹配到什么？这种匹配的质量如何？

**Answer:**
重叠假设要求：对于每一个协变量组合，处理组和对照组都有非零概率接受处理，即 $0 < P(D=1|X=x) < 1$。它是PSM的"生命线"，因为PSM的核心是为每个处理组个体在特征空间中找到"镜像"对照。若某处理组个体的倾向得分在对照组中不存在（如0.95的黑卡用户 vs 最高0.3的非会员），则找不到"相似者"，匹配只能退化为"找最不差的"——匹配质量极差，反事实不可信。

**没有重叠的后果：**
- 强行匹配 → 匹配对差异巨大，估计偏差严重
- 限制共同支持域 → 丢弃无重叠样本，外部效度损失
- 结论不可信 → 无法回答"对这部分人，处理效应是什么"

**决策规则：** 做PSM前必须先画倾向得分分布图（直方图/密度图），目视检查重叠。若重叠区域<50% → 考虑换方法或重新设计研究。

### Q2: "限制共同支持域"和"截断（Trimming）"有什么区别？

**Hints:**
- 限制共同支持域是"处理组和对照组的交集"，截断是"去掉两端的极端值"
- 思考：如果处理组倾向得分范围是[0.3, 0.9]，对照组是[0.1, 0.7]，两者的结果分别是什么？

**Answer:**
| 维度 | 限制共同支持域 | 截断（Trimming） |
|---|---|---|
| 逻辑 | 保留处理组和对照组倾向得分的**交集** | 丢弃倾向得分**极端**的样本（两端） |
| 处理组是否可能丢弃 | 是（超出对照组范围的） | 是（超出截断点的） |
| 对照组是否可能丢弃 | 是（超出处理组范围的） | 是（超出截断点的） |
| 截断点来源 | 数据驱动（由两组分布决定） | 主观设定（如0.1和0.9） |
| 结果 | 更严格，通常保留样本更少 | 更灵活，可能保留更多样本 |

**决策规则：**
- 若两组分布差异大但有部分重叠 → 限制共同支持域（更保守，避免主观截断点）
- 若两组分布整体重叠，但两端有极端值 → 截断（如去掉倾向得分<0.05或>0.95的样本）
- 若截断点主观性强且影响结论 → 做截断点敏感性分析（尝试多个截断点，看结论是否稳健）

### Q3: 为什么限制共同支持域后，结论不能推广到被丢弃的群体？

**Hints:**
- 被丢弃的群体往往是"最极端"的处理组用户——他们可能是业务最关心的群体
- 思考：ATT的定义是"处理组的平均效应"，但限制共同支持域后的ATT是"可匹配处理组的平均效应"，两者一样吗？

**Answer:**
错误在于将"共同支持域内的ATT"等同于"全体处理组的ATT"。被丢弃的群体（如倾向得分>0.8的黑卡用户）往往是业务核心用户——他们贡献了大部分GMV，且处理效应可能与可匹配用户完全不同（如天花板效应使边际提升更小）。

**外部效度损失的本质：** PSM的结论被限定在"共同支持域"这个子样本上，而被丢弃的样本可能正是决策者最关心的群体。

**正确汇报方式：**
- "在共同支持域内（覆盖处理组的X%），ATT=+300元"
- "对倾向得分>0.8的高端用户（覆盖Y%），因无重叠无法估计"
- "若高端用户的效应与可匹配用户不同，全体用户的ATE可能偏离+300元"
- 建议：若共同支持域覆盖<70% → 声明"结论仅适用于大部分用户，对极端用户的效应未知"

**决策规则：** 共同支持域覆盖比例 ≥ 80% → 可推广；50%-80% → 谨慎推广，需声明限定；<50% → 不宜推广，需换方法或补充数据。

### Q4: 一线城市实施"限塑令"，二三线城市作为对照，为什么PSM可能失效？

**Hints:**
- PSM要求处理组和对照组在特征空间有重叠，但一线城市和二三线城市的经济结构差异是系统性的
- 思考：DID和合成控制法在这个场景下的识别假设分别是什么？

**Answer:**
PSM失效的原因：一线城市商家和二三线城市商家在规模、客单价、品类结构等维度上**系统性差异**，倾向得分分布几乎无重叠，无法找到"相似"的对照。

**替代方案对比：**
| 方法 | 为什么适用 | 怎么做 |
|---|---|---|
| **DID** | 不依赖"个体相似"，依赖"趋势平行" | 比较一线城市限塑令前后 vs 二三线城市同期变化，假设"如果没有限塑令，一线城市会和二三线城市走同样趋势" |
| **合成控制法** | 用多个城市加权构造"虚拟一线城市"，不找单一相似城市 | 见[[stage-02-advanced/04-causal-inference/02-did/03-synthetic-control/合成控制法|合成控制法]] |
| **PSM（若坚持用）** | 仅在一线城市内部找对照 | 比较一线城市中"已实施限塑令的商家" vs "未实施的商家"（但需假设实施是准随机的） |

**决策规则：**
- 处理单元独特、对照单元差异大 → 不用PSM，用合成控制法
- 有面板数据、政策实施时间明确 → 用DID，但必须先做[[stage-02-advanced/04-causal-inference/02-did/01-parallel-trends/DID平行趋势检验|平行趋势检验]]
- 若DID平行趋势不成立 → 用合成控制法作为DID的替代

### Q5: 面对"无重叠"困境，是丢弃样本、换方法、还是重新设计研究？

**Hints:**
- 三种方案的本质是：①缩小问题范围；②换识别策略；③承认数据局限
- 思考：什么情况下"承认无法回答"比"强行回答"更有价值？

**Answer:**
**决策框架：**

| 情境 | 推荐方案 | 判断标准 |
|---|---|---|
| 无重叠的是"边缘群体"，核心群体可匹配 | 限制共同支持域 | 共同支持域覆盖≥70%，且边缘群体非决策关键 |
| 处理单元独特，但有大量潜在对照单元 | 合成控制法 | 对照单元≥15个，政策前数据≥12期 |
| 有面板数据，处理时间明确 | DID + 平行趋势检验 | 事前窗口≥4-6期，且趋势可图形验证 |
| 无重叠且上述方法均不适用 | 承认局限，只做描述性分析 | 强行因果推断的偏误风险 > 不做因果推断的信息损失 |

**核心原则：**
- 若结论将直接影响重大决策（如投入千万级预算）→ 宁可承认"数据无法回答"，也不做不可靠的因果推断
- 若结论仅用于"方向性判断" → 可尝试多种方法交叉验证，明确标注不确定性
- 若所有方法都指向同一方向 → 增强信心；若方法间结论矛盾 → 暂停决策，补充数据或实验

**决策规则：** 当"因果推断的可信度"无法达到业务决策所需阈值时，诚实报告"当前数据无法支持因果结论"，并建议下一步数据收集或实验设计。
