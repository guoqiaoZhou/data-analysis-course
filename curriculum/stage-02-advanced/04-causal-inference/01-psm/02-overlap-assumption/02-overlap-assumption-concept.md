# 重叠假设（Overlap / Common Support）

## 基本原理

重叠假设要求：对于每一个协变量组合，处理组和对照组都有非零概率。即倾向得分分布必须有重叠区域，否则无法找到匹配。

**形式化：**
$$0 < P(D=1|X=x) < 1, \quad \forall x$$

**实际含义：**
- 处理组中不能存在"绝无可能在对照组中找到相似个体"的个体
- 对照组中也不能存在"绝无可能接受处理"的个体

## 使用场景

| 场景 | 问题 | 处理 |
|---|---|---|
| 处理组和对照组特征差异大 | 无重叠区域 | 限制共同支持域，丢弃无重叠样本 |
| 极端倾向得分 | 接近0或1 | 截断（trimming）或分层 |
| 高维协变量 | 难以可视化重叠 | 用倾向得分分布图判断 |
| 政策评估 | 某些地区/群体完全未实施 | 这些群体无法估计效应 |

## 局限性

1. **样本损失**：限制共同支持域会丢弃样本，降低效率
2. **外部效度损失**：丢弃的样本可能正是最关心的群体
3. **高维难判断**：协变量多时用倾向得分代理，但倾向得分模型可能误设
4. **边界效应**：共同支持域边界处的匹配质量差

## 同类算法对比

| 方法 | 处理无重叠 | 优势 | 劣势 |
|---|---|---|---|
| **限制共同支持域** | 丢弃无重叠样本 | 简单直接 | 样本损失，外部效度问题 |
| **截断（Trimming）** | 截断极端倾向得分 | 保留更多样本 | 截断点主观 |
| **熵平衡（Entropy Balancing）** | 重新加权 | 不丢弃样本 | 权重可能极端 |
| **合成控制** | 构造加权对照组 | 适用于聚合数据 | 只适用于少量处理单元 |
| **双重稳健估计** | 模型辅助 | 一个模型对即可 | 两个都错时偏差大 |

## 真实案例

### 案例1：高端用户匹配失败（C端运营）

**背景：** 评估"黑卡会员"效果，黑卡用户消费极高。

**数据：**
- 黑卡用户：倾向得分集中在0.8-1.0
- 普通用户：倾向得分集中在0.0-0.3
- 重叠区域：0.3-0.8，但样本稀少

**问题：**
- 黑卡用户中消费最高的群体无匹配
- 若限制共同支持域，丢失最关心的高端用户

**处理：**
- 分层分析：将黑卡用户分层，低层（可匹配）用PSM，高层（无匹配）用其他方法
- 声明：对高端用户的效应无法估计

### 案例2：政策评估的地域差异（离线分析）

**背景：** 评估某城市"限塑令"对商家订单的影响。

**数据：**
- 实施城市：一线城市，商家规模大
- 未实施城市：二三线城市，商家规模小

**问题：**
- 一线城市商家在二三线城市找不到匹配
- 倾向得分分布几乎无重叠

**处理：**
- 限制分析范围：仅在一线城市内比较（实施 vs 未实施，但时间不同）→ DID
- 或：用合成控制法，用多个二三线城市加权构造"伪一线城市"

### 案例3：功能使用的自选择（B端运营）

**背景：** 评估"高级报表"功能对商家活跃度的影响。

**数据：**
- 使用功能商家：大型商家，倾向得分0.7-1.0
- 未使用商家：小型商家，倾向得分0.0-0.3

**分析：**
- 重叠区域窄，匹配后样本减少60%
- 匹配后ATT = +10%，但置信区间宽
- 结论：功能可能有效果，但估计不精确，建议扩大功能覆盖后重新评估

## 关键词

- **共同支持域（Common Support）**：处理组和对照组倾向得分的重叠区域
- **截断（Trimming）**：丢弃倾向得分极端的样本
- **SMD（标准化均值差异）**：匹配后协变量平衡度，<0.1为可接受
- **倾向得分分布图**：可视化重叠，通常用直方图或密度图
- **外部效度**：估计结果能推广到哪些群体
