# 敏感性分析

## 问题引入

经过PSM匹配和重叠假设检验，你向老板汇报了会员制度的真实效果：匹配后会员消费提升约300元。老板点点头，但随即抛出一个尖锐的问题："你控制了年龄、性别、城市、历史消费这些能看到的因素，但万一还有你没看到的因素呢？比如，开通会员的人可能本来就更有'消费意愿'——这种意愿你看不到，却同时驱动了他们开通会员和多消费。如果存在这种遗漏变量，你的300元结论还成立吗？"

你再次愣住了。确实，PSM只能控制"可观测的混杂因素"——那些在数据表格里有记录的特征。但真实世界中，总有我们无法测量的因素：用户的消费心态、对平台的情感依赖、甚至只是"喜欢尝试新事物"的性格倾向。这些因素既影响用户是否开通会员，又影响他们的消费金额，却不在你的数据集中。

这就是观察性研究的"阿喀琉斯之踵"：**未观测混杂（Unobserved Confounding）**。无论你的匹配方法多么精巧，只要存在未被控制的混杂因素，因果估计就可能偏误。问题是——我们不知道遗漏变量有多强，也不知道它们能在多大程度上解释掉我们观测到的300元效应。

老板追问："那你说说，需要多强的遗漏变量才能推翻你的结论？"你无法回答。因为你学过的[[stage-02-advanced/04-causal-inference/01-psm/01-matching-methods/PSM匹配方法|PSM匹配方法]]和[[stage-02-advanced/04-causal-inference/01-psm/02-overlap-assumption/重叠假设|重叠假设]]只能处理观测到的世界，对"看不到的因素"无能为力。

你需要一种方法，能够量化回答："如果存在一个未观测混杂因素，它需要多强才能解释掉我的结论？"这就是敏感性分析的使命——它不试图找出遗漏变量是什么，而是评估"结论对遗漏变量的敏感程度"，为你的因果推断加上一道"压力测试"。

## 基本原理

敏感性分析评估：未观测混杂需要多强，才能解释掉已观测的处理效应。核心问题："我的结论对遗漏变量有多敏感？"

**Rosenbaum Bounds：**
- 假设未观测混杂使个体接受处理的 odds 变化 $\Gamma$ 倍
- 计算在不同 $\Gamma$ 下，效应是否仍然显著
- 若 $\Gamma$ 较小（如1.5）就不显著，说明结论脆弱

**E-value：**
- 能解释掉效应的最小未观测混杂强度
- E-value = $RR_{obs} + \sqrt{RR_{obs}(RR_{obs}-1)}$，其中 $RR_{obs}$ 是观测到的风险比
- 含义：未观测混杂需使结果的RR达到此值，才能完全解释观测效应

## 使用场景

| 场景 | 说明 | 示例 |
|---|---|---|
| 观察性研究结论可靠性 | 评估未观测混杂的影响 | PSM后发现显著效应，但担心遗漏变量 |
| 政策评估 | 政策非随机实施 | 某城市先实施政策，评估效果 |
| 业务决策依据 | 无实验数据，需基于观察性数据决策 | 评估历史活动效果 |
| 审稿/汇报 | 回应"是否考虑了所有混杂？" | 在报告中附敏感性分析 |

## 局限性

1. **只能评估强度，不能识别变量**：知道需要多强的混杂，但不知道是什么变量
2. **假设单混杂**：实际可能有多个未观测混杂
3. **无法排除所有替代解释**：只能评估未观测混杂，不能评估模型误设等
4. **结果解释主观**：多大的$\Gamma$算"强"？因领域而异

## 同类算法对比

| 方法 | 原理 | 优势 | 劣势 |
|---|---|---|---|
| **Rosenbaum Bounds** | 假设未观测混杂的 odds ratio | 非参数，直观 | 只适用于匹配方法 |
| **E-value** | 计算最小解释效应的混杂强度 | 简单，适用于任何设计 | 假设单混杂 |
| **Oster Bounds** | 基于R²的敏感性分析 | 适用于回归调整 | 需要假设混杂与已观测混杂的关系 |
| **Imbens' Sensitivity** | 假设未观测混杂的分布 | 更一般 | 参数化假设 |
| **阴性对照（Negative Controls）** | 预期无效应的结果变量 | 直接检验 | 需要找到合适的阴性对照 |

## 真实案例

### 案例1：会员制度效果的稳健性（C端运营）

**背景：** PSM估计会员制度ATT = +￥300（p=0.02）。

**Rosenbaum Bounds：**
- $\Gamma$ = 1.0：显著（p=0.02）
- $\Gamma$ = 1.5：p=0.08（边缘）
- $\Gamma$ = 2.0：p=0.25（不显著）

**解释：**
- 若未观测混杂使会员 odds 变化1.5倍，结论边缘
- 若变化2倍，结论不成立
- 业务判断：消费意愿可能使 odds 变化2倍以上
- 结论：效应可能存在，但不宜作为强决策依据

### 案例2：运营策略的E-value（B端运营）

**背景：** 评估"运营经理对接"效果，$RR_{obs}$ = 1.5（处理组vs对照组的GMV比）。

**E-value：**
- E-value = 1.5 + $\sqrt{1.5 \times 0.5}$ = 1.5 + 0.87 = 2.37

**解释：**
- 未观测混杂需使GMV的RR达到2.37，才能解释掉观测效应
- 业务判断：商家规模、行业等已观测，剩余未观测因素很难达到2.37
- 结论：效应较稳健

### 案例3：阴性对照检验（离线分析）

**背景：** 评估"搜索优化"对下单转化的影响。

**阴性对照：**
- 预期不受搜索优化影响的结果："客服咨询次数"
- 若搜索优化也显著影响客服咨询次数，提示存在混杂

**结果：**
- 搜索优化对下单转化：+2.1pp（p=0.01）
- 搜索优化对客服咨询：+0.1pp（p=0.8，不显著）
- 结论：阴性对照通过，效应较可信

## 关键词

- **Rosenbaum Bounds**：匹配方法中，假设未观测混杂的 odds ratio 范围
- **E-value**：能解释掉效应的最小未观测混杂强度
- **阴性对照（Negative Controls）**：预期无效应的结果变量
- **Gamma ($\Gamma$)**：未观测混杂导致的 odds ratio 倍数
- **稳健性（Robustness）**：结论对假设偏离的敏感程度
