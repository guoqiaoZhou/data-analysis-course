# 好问练习解答 - 敏感性分析

> 对应知识点：[[stage-02-advanced/04-causal-inference/01-psm/03-sensitivity-analysis/敏感性分析|敏感性分析]]

---

### Q1: 为什么PSM匹配后还要做敏感性分析？它回答什么问题？

**Hints:**
- PSM只能控制"可观测混杂"，但真实世界总有"看不到的因素"
- 思考：老板问"需要多强的遗漏变量才能推翻结论"，这个问题的价值是什么？

**Answer:**
PSM消除的是"数据表格里有记录"的混杂，但真实世界中总有未测量因素：消费心态、情感依赖、性格倾向、信息获取渠道等。这些**未观测混杂**既影响处理分配又影响结果，PSM对此无能为力。

敏感性分析的核心使命是：**量化评估"结论对未观测混杂的抵抗能力"**。它不试图找出遗漏变量是什么，而是回答："如果存在一个未观测混杂因素，它需要多强才能解释掉我观测到的300元效应？"

**决策规则：**
- 所有观察性研究的因果推断 → 必须附敏感性分析
- 若敏感性分析显示结论脆弱（如Γ<1.5就不显著）→ 结论不宜作为强决策依据
- 若敏感性分析显示结论稳健（如Γ>3.0仍显著）→ 增强因果推断可信度，但仍不能"证明"无未观测混杂

### Q2: Rosenbaum Bounds和E-value有什么区别？

**Hints:**
- Rosenbaum Bounds是"给定混杂强度，检验显著性"；E-value是"计算推翻结论的最小混杂强度"
- 思考：两者的输出形式不同，分别适合什么汇报场景？

**Answer:**
| 维度 | Rosenbaum Bounds | E-value |
|---|---|---|
| 核心问题 | "在混杂强度Γ下，结论是否仍显著？" | "推翻结论所需的最小混杂强度是多少？" |
| 输出 | 不同Γ下的p值区间 | 单个数字（最小RR） |
| 数学基础 | 假设未观测混杂使odds变化Γ倍 | E-value = RR_obs + √(RR_obs(RR_obs-1)) |
| 适用方法 | 主要适用于匹配方法（PSM） | 适用于任何设计（回归、DID等） |
| 汇报形式 | "Γ=1.5时p=0.08，Γ=2.0时p=0.25" | "E-value=2.37" |
| 解释 | 需要理解Γ的含义 | 更直观："未观测混杂需使结果RR达到此值" |

**决策规则：**
- 使用PSM匹配方法 → 优先报告Rosenbaum Bounds（与匹配方法配套）
- 使用回归或DID → 优先报告E-value（通用性强）
- 向非技术决策者汇报 → E-value更直观（"需要多强的混杂才能推翻"）
- 技术汇报/论文 → 同时报告两者，交叉验证

### Q3: 为什么"敏感性分析通过"不等于"没有未观测混杂"？

**Hints:**
- 敏感性分析通常假设"单混杂"，但现实中可能有多个未观测混杂
- 思考：多个弱混杂叠加，与单个强混杂，对结论的影响是否相同？

**Answer:**
错误在于将"敏感性分析的通过"等同于"不存在未观测混杂"。敏感性分析有三个根本局限：

1. **假设单混杂**：Rosenbaum Bounds和E-value都假设只有一个未观测混杂因素。但现实中可能有多个弱混杂同时存在（如"消费意愿"+"信息渠道"+"社交影响"），每个的Γ都不大，但叠加后总效应可能很大。

2. **不能识别变量**：知道需要Γ=2.0，但不知道是什么变量。如果现实中确实存在Γ>2.0的变量（如"商家规模"在某些场景下），结论就被推翻了。

3. **无法排除其他威胁**：敏感性分析只评估未观测混杂，不评估模型误设、测量误差、样本选择偏差等其他威胁。

**正确解读方式：**
- "敏感性分析显示，单个未观测混杂需达到Γ=2.0才能推翻结论"
- "但多个弱混杂叠加可能达到同等效果"
- "若领域知识表明不存在Γ>2.0的未观测因素 → 结论较可信"
- "若领域知识表明可能存在 → 结论需谨慎"

**决策规则：** Γ>3.0且领域知识支持 → 结论稳健；1.5<Γ<3.0 → 结论中等稳健，需补充其他检验；Γ<1.5 → 结论脆弱，不宜作为决策依据。

### Q4: 阴性对照检验和敏感性分析有什么区别？

**Hints:**
- 敏感性分析是"假设存在混杂，评估需要多强"；阴性对照是"直接找一个预期无效应的结果，检验是否出现伪效应"
- 思考：阴性对照通过意味着什么？阴性对照失败意味着什么？

**Answer:**
| 维度 | 敏感性分析 | 阴性对照检验 |
|---|---|---|
| 逻辑 | "假设存在未观测混杂，评估其强度" | "找一个预期不受处理影响的结果，直接检测混杂" |
| 对混杂的态度 | 间接评估 | 直接检测 |
| 需要额外数据 | 不需要（基于已有估计） | 需要（找到合适的阴性对照变量） |
| 通过的含义 | "需要很强的混杂才能推翻" | "未检测到明显的混杂信号" |
| 失败的含义 | "结论可能脆弱" | "存在混杂，结果不可信" |
| 局限 | 假设单混杂，不能识别变量 | 需要找到"合适的"阴性对照，难度高 |

**互补使用：**
- 敏感性分析通过 + 阴性对照通过 → 双重证据支持结论稳健
- 敏感性分析通过 + 阴性对照失败 → 提示存在未观测混杂，敏感性分析可能低估了风险
- 敏感性分析失败 + 阴性对照通过 → 可能是敏感性分析假设过强，或阴性对照选择不当
- 两者都失败 → 结论不可信，需重新设计研究

**决策规则：**
- 能找到合适的阴性对照（预期无效应且与主要结果相关）→ 优先做阴性对照，再补充敏感性分析
- 找不到合适的阴性对照 → 必须做敏感性分析，并在汇报中声明局限
- 重大决策 → 两者都做

### Q5: 敏感性分析显示结论"脆弱"，下一步该做什么？

**Hints:**
- "脆弱"意味着观察性数据无法给出可信的因果结论
- 思考：什么情况下"承认无法回答"比"强行回答"更有价值？什么情况下可以补救？

**Answer:**
**决策框架：**

| 情境 | 推荐方案 | 判断标准 |
|---|---|---|
| 有额外数据可获取（如更详细的用户画像） | 补充协变量，重新PSM | 新增协变量可能覆盖主要未观测混杂路径 |
| 可设计自然实验（如政策在不同时间/地区 staggered 实施） | 改用DID或合成控制法 | 利用时间/空间变化，减少对交叉截面匹配的依赖 |
| 可设计RCT（如小范围随机实验） | 直接做RCT验证 | RCT是因果推断金标准，可完全消除未观测混杂 |
| 上述均不可行 | 承认局限，只做描述性分析 | 强行因果推断的偏误风险 > 不做因果推断的信息损失 |

**核心原则：**
- 若结论将用于重大资源分配（如千万级预算）→ 敏感性分析脆弱时，必须补充RCT或自然实验
- 若结论仅用于"方向性判断"或"优先级排序" → 可保留观察性分析，但明确标注"因果效应估计存在不确定性"
- 若领域知识强烈支持"不存在强未观测混杂" → 可适度放松标准，但需在汇报中论证

**决策规则：** 当敏感性分析显示结论脆弱（Γ<1.5或E-value<2.0），且该结论将直接影响重大决策时，**优先设计验证实验**，而非依赖观察性数据。
