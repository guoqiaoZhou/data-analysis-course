# 好问练习解答 - 事件研究法

> 对应知识点：[[stage-02-advanced/04-causal-inference/02-did/02-event-study/事件研究法|事件研究法]]

---

### Q1: 传统DID给出了"平均5.5分钟"，为什么还需要事件研究法？

**Hints:**
- 传统DID只估计一个"政策后 vs 政策前"的平均差异
- 思考：如果效应随时间变化，平均数字会掩盖什么信息？

**Answer:**
传统DID的局限：只给出**静态平均效应**，掩盖了三种关键信息：①**效应何时出现**（即时/滞后）；②**效应如何演变**（增强/衰减/稳定）；③**是否存在副作用**（先升后降/反转）。

事件研究法的核心价值：**同时回答两个问题**：
1. **平行趋势检验**：政策前系数是否显著为0（验证DID核心假设）
2. **动态效应估计**：政策后系数展示效应的完整轨迹

**业务决策意义：**
- 效应即时显现 → 政策设计合理，无需调整
- 效应滞后 → 有学习/适应成本，不宜过早评估
- 效应衰减 → 政策不持久，需配套措施
- 效应透支 → 刺激提前释放需求，长期不可持续

**决策规则：** 所有DID分析都应补充事件研究法，除非只有政策前后各1期数据（短面板无法实施）。

### Q2: 事件研究法和"分阶段DID"有什么区别？

**Hints:**
- 事件研究法是一个回归模型，同时估计所有期系数
- 分阶段DID是多个独立回归，每个回归估计一个时期的效应
- 思考：两者的自由度、标准误、模型假设有何差异？

**Answer:**
| 维度 | 事件研究法 | 分阶段DID |
|---|---|---|
| 模型结构 | 单回归，多虚拟变量 | 多回归，每期独立 |
| 统计效率 | 高（共享方差估计，自由度大） | 低（每期独立估计方差） |
| 标准误 | 较小（模型约束更强） | 较大（每期样本少） |
| 系数解释 | 相对基准期的差异 | 每期独立的"政策后 vs 政策前"差异 |
| 平行趋势检验 | 内置（政策前系数） | 需额外检验 |
| 灵活性 | 低（模型设定统一） | 高（每期可不同控制变量） |

**决策规则：**
- 标准场景（统一处理时间，关心动态效应）→ 事件研究法（效率高，内置平行趋势检验）
- 处理时间异质性大，或每期需要不同控制策略 → 分阶段DID（灵活，但效率低）
- 样本量小 → 事件研究法（共享信息，标准误更小）
- 需要每期独立稳健性检验 → 分阶段DID（每期可独立做安慰剂检验）

### Q3: 为什么基准期选择不当会"扭曲整个系数图"？

**Hints:**
- 基准期系数被约束为0，所有其他期系数是相对于基准期的差异
- 思考：如果基准期本身已经"不正常"，其他期的"相对差异"还正常吗？

**Answer:**
基准期是"参照点"，其系数被约束为0。若基准期已受预期效应影响（如政策前1个月因灰度测试而行为改变），则：

- **政策前系数**：政策前2期相对基准期的差异会被高估（因为基准期已经"高"了）
- **政策后系数**：政策后效应相对基准期会被低估（基准期已经包含了部分"预期效应"）
- **平行趋势检验**：政策前系数可能显著（因为基准期不正常），导致平行趋势"假失败"

**修正策略：**
1. 调查政策实施前是否存在预热、灰度、泄露等预期效应
2. 若存在 → 将基准期前移至预期效应开始之前（如政策前第2期或更早）
3. 重新估计后，对比两次系数图的变化幅度
4. 若变化幅度>20% → 报告修正后的结果，并解释基准期选择的影响

**决策规则：** 基准期选择必须基于业务知识（政策实施流程），不能机械默认"政策前1期"。若存在预期效应 → 前移基准期；若不确定 → 尝试多个基准期，报告敏感性。

### Q4: 交错处理下传统事件研究法出现"后期系数为负"，是政策真的有害还是方法偏差？

**Hints:**
- 交错处理中，早期处理组会被用作晚期处理组的对照组
- 思考：已接受处理的单元作为"对照组"，这个"对照"还干净吗？

**Answer:**
**方法偏差，而非政策有害。** 传统事件研究法假设所有处理单元在同一时间接受处理。交错处理下：

- 早期处理组（如城市A在Q1上线）在晚期（Q2-Q4）被用作"对照组"对照晚期处理组（城市B在Q2上线）
- 但城市A已经接受了处理，其趋势包含了真实处理效应，不再是"干净的对照"
- Goodman-Bacon分解显示：这些"坏比较"的权重为负，导致估计量可能产生与真实效应方向相反的偏误

**改用Callaway & Sant'Anna（2021）方法后：**
- 只使用"从未处理组"或"尚未处理组"作为对照组
- 消除负权重问题，所有系数应为正（若真实效应为正）
- 计算更复杂，需要专门软件包（如R的`did`包或Python的`staggered`）

**决策规则：**
- 处理时间异质性大（staggered）→ 必须使用CS或SA方法
- 传统方法出现反直觉结果（如政策明显有益但系数为负）→ 识别为负权重问题，换修正方法
- 处理时间统一（所有单元同时处理）→ 传统事件研究法可用

### Q5: 事件研究法显示"激励效应逐月衰减"，运营团队想"加大补贴力度"，你该支持还是反对？

**Hints:**
- "衰减"有两种解释：①补贴力度不够，需要加大；②需求被透支，加大补贴只会进一步透支
- 思考：如何区分"补贴不足"和"透支效应"？

**Answer:**
**反对简单加大补贴。** 事件研究法的动态模式需要结合业务机制解读：

| 动态模式 | 机制解读 | 决策建议 |
|---|---|---|
| **即时高 → 快速回落** | 透支效应（存量需求集中释放） | 不改补贴力度，改补贴结构（阶梯式/持续性） |
| **逐步增强 → 稳定** | 滞后效应（学习/适应成本） | 维持现有补贴，给足时间 |
| **稳定高 → 缓慢衰减** | 边际效用递减 | 微调补贴结构，而非简单加量 |
| **先升后降再升** | 复杂机制（如竞争反应） | 深入分析，不宜简单决策 |

**判断"透支 vs 补贴不足"的标准：**
- 若第1-2期提升主要来自"存量用户集中行为"（如创作者集中释放存量内容）→ 透支
- 若新增用户在补贴后增长，但留存差 → 可能是补贴吸引了"羊毛党"
- 若政策后各期"新增量"稳定，但"总量"回落 → 时间转移效应

**决策规则：**
- 透支效应 → 改变补贴结构（从"按播放量一次性奖励"改为"持续优质内容阶梯奖励"）
- 补贴不足 → 优化补贴精准度（定向补贴高潜力用户），而非简单加量
- 机制不明 → 先做用户分层分析，明确衰减原因后再决策
