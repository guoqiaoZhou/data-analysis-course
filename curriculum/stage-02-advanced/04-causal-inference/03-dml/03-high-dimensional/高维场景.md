# 高维场景

## 基本原理

高维场景指协变量维度 $p$ 与样本量 $n$ 相当甚至更大的情况（$p \approx n$ 或 $p \gg n$）。传统统计方法在高维场景下失效，DML通过机器学习处理高维nuisance估计。

核心挑战：
- **维度灾难**：当 $p$ 增大时，数据在高维空间极度稀疏
- **过拟合**：传统回归在高维下完美拟合噪声，估计方差爆炸
- **计算不可行**：某些传统方法（如精确矩阵求逆）在高维下计算困难

DML在高维场景下的理论保证：

在 $p \gg n$ 但存在稀疏结构（只有少量协变量真正重要）时，如果nuisance函数满足稀疏性假设，Lasso等正则化方法可以达到 $o(n^{-1/4})$ 的收敛速度，满足DML的要求。

稀疏性假设：
$$g(X) = \sum_{j=1}^p \beta_j X_j, \quad \|\beta\|_0 = s \ll n$$

即结果回归函数仅依赖于 $s$ 个协变量，$s$ 远小于 $n$。

Lasso的收敛速度：
$$\|\hat{g} - g\|_2 = O_P\left(\sqrt{\frac{s \log p}{n}}\right)$$

当 $s \log p / \sqrt{n} \to 0$ 时，满足DML的 $o(n^{-1/4})$ 要求。

## 使用场景

| 场景 | 说明 | 示例 |
|---|---|---|
| 用户画像高维标签 | 用户有数百至数千个标签特征 | 估计推送策略效果，控制1000+用户行为标签 |
| 文本/图像特征 | 将非结构化数据转化为高维向量 | 用BERT embedding（768维）作为协变量 |
| 基因/生物信息 | 基因表达数据维度极高 | 药物效果评估，控制数万个基因表达水平 |
| 网络/图特征 | 社交网络中的高维邻接信息 | 评估社交推荐效果，控制用户社交关系特征 |
| 时间序列高维面板 | 多个时间序列的联合分析 | 评估政策效果，控制多个地区、多个指标的交互 |

## 局限性

1. **稀疏性假设可能不成立**：如果真实nuisance函数依赖于大量协变量（非稀疏），Lasso等方法的收敛速度可能不满足DML要求
2. **特征选择不稳定**：高维下特征选择结果高度依赖于样本，不同子样本选出的特征集差异大
3. **计算资源需求大**：高维场景下训练机器学习模型（尤其是调参）需要大量计算资源
4. **可解释性丧失**：高维DML的nuisance模型通常是黑盒，难以解释哪些协变量在控制混杂

## 同类算法对比

| 方法 | 适用条件 | 优势 | 劣势 |
|---|---|---|---|
| DML + Lasso | 高维稀疏线性关系 | 理论保证完善，计算高效 | 无法处理非线性，需稀疏性假设 |
| DML + Elastic Net | 高维，特征间相关性强 | 处理共线性，比Lasso稳定 | 需调两个超参数 |
| DML + Random Forest | 高维，非线性，特征交互 | 自动捕捉非线性和交互 | 高维稀疏场景不如Lasso |
| DML + Neural Network | 超高维，复杂模式 | 表达能力最强 | 样本量要求极高，理论保证弱 |
| 双重Lasso（Double Lasso） | 高维，关注变量选择 | 同时进行变量选择和因果推断 | 仅适用于线性模型 |
| Post-Lasso DML | 高维，先筛选后估计 | 提高稀疏场景效率 | 变量选择错误会传播偏差 |

## 行业使用经验

### 经验1：先做维度诊断，不盲目上高维模型
在大厂实践中，不是所有场景都需要高维DML。经验流程：
1. 先用低维方法（如AIPW+逻辑回归）估计，作为基准
2. 逐步增加协变量维度，观察估计是否稳定
3. 当估计随维度增加明显变化时，说明遗漏变量偏差严重，需要高维DML
4. 如果低维和高维估计接近，优先使用低维（可解释性更好）

### 经验2：特征工程比模型选择更重要
高维DML的效果很大程度上取决于特征质量。大厂实践：
- 优先引入"业务强相关"特征（如历史行为、 demographics）
- 避免引入"衍生特征"的过多组合（如所有两两交互），会导致维度爆炸但信息增益有限
- 对高维稀疏特征（如one-hot编码的类目）进行降维或嵌入
- 经验：100个精心构造的特征往往比10000个原始特征效果更好

### 经验3：使用双重稳健性作为诊断工具
DML框架天然具有双重稳健性：只要倾向得分或结果回归之一估计正确，ATE估计就一致。在高维场景下，可以：
1. 分别用不同模型估计两个nuisance函数
2. 比较不同组合下的ATE估计
3. 如果估计差异大，说明至少一个nuisance函数估计有问题
4. 大厂标准：当不同组合的ATE差异>10%时，需要重新审视特征和模型

## 真实案例

### 案例1：短视频平台个性化推送效果（C端运营/离线分析）
某短视频平台评估"个性化推荐"对人均时长的因果效应，用户画像包含2000+标签（包括 demographics、历史行为、内容偏好等）。

**分析方案**：
- 高维挑战：2000+特征，样本量10万，$p/n = 0.02$
- 使用DML + Lasso：Lasso自动筛选重要特征估计nuisance函数
- 对比实验：同时用DML + Random Forest作为稳健性检验

**结果**：
- Lasso选出约150个非零系数特征，ATE估计为+12分钟/日
- Random Forest估计为+11分钟/日，两者接近，增强信心
- 如果两者差异大（如>20%），则需要检查模型设定

### 案例2：金融风控策略效果评估（B端运营）
某金融科技公司评估"动态额度调整策略"对逾期率的影响，需要控制用户多维信用特征。

**分析方案**：
- 协变量：用户信用评分、历史还款、消费行为、社交关系等500+特征
- 部分特征来自外部数据源（如运营商数据），维度高且稀疏
- 使用DML + Elastic Net（处理特征间的共线性）
- 对稀疏特征（如one-hot编码的职业类别）进行PCA降维

**结果**：策略使逾期率降低2.5个百分点，Elastic Net选出约80个关键特征，其中信用评分和历史还款行为权重最高。

### 案例3：医疗数据分析中的高维控制（离线分析/运营）
某互联网医疗平台评估"在线问诊服务"对用户后续就医行为的影响，需要控制用户健康相关的高维特征。

**分析方案**：
- 协变量：用户 demographics、历史问诊记录、药品购买记录、症状描述文本（经BERT编码为768维向量）
- 总维度约1500，样本量8000
- 使用DML + 两层策略：
  - 第一层：用Lasso筛选文本embedding中的重要维度
  - 第二层：在筛选后的特征上用Random Forest估计nuisance

**结果**：在线问诊服务使用户30天内复诊概率降低8%，说明线上服务有效替代了部分线下就医需求。

## 关键词

- **维度灾难（Curse of dimensionality）**：高维空间中数据稀疏性导致传统统计方法失效的现象
- **稀疏性（Sparsity）**：真实模型仅依赖于少量特征的性质，高维统计的核心假设
- **正则化（Regularization）**：通过惩罚大系数来控制模型复杂度的技术，如L1/L2惩罚
- **Lasso（Least Absolute Shrinkage and Selection Operator）**：使用L1惩罚的正则化方法，能产生稀疏解
- **Post-selection inference**：先进行变量选择再做推断的方法，需注意选择偏差问题
- **双重稳健性（Double robustness）**：只要倾向得分或结果回归之一正确，估计就一致的性质
