# 好问练习解答 - Nuisance函数估计

> 对应知识点：[[stage-02-advanced/04-causal-inference/03-dml/01-nuisance-estimation/Nuisance函数估计|Nuisance函数估计]]

---

### Q1: 为什么DML不直接用传统逻辑回归估计倾向得分，而要用机器学习来估计Nuisance函数？

**Hints:**
- 思考"维度"和"函数形式"两个维度：传统方法在什么条件下失效？
- 回忆PSM的局限性：低维场景表现好，高维+非线性时力不从心
- DML的核心分解是什么？机器学习负责哪部分，因果推断负责哪部分？

**Answer:**
传统逻辑回归在以下两种情况下失效：
1. **高维场景**：当协变量维度$p$与样本量$n$相当或更大时（$p \approx n$或$p \gg n$），传统回归的估计方差爆炸，甚至无法求解
2. **非线性关系**：当协变量与结果/处理的关系是非线性或存在复杂交互时，线性模型设定错误（misspecification）导致估计有偏

DML的分解逻辑：机器学习负责"预测"（利用其处理高维和非线性的能力估计nuisance函数），因果推断负责"识别"（在正交化后的残差上给出无偏的效应估计）。

**决策规则**：
- 如果协变量维度<20且关系大致线性 → 传统方法（AIPW+逻辑回归）足够
- 如果维度≥20或存在已知非线性关系 → 使用DML+机器学习
- 如果需要估计异质性效应（CATE）→ DML是更自然的选择

### Q2: Nuisance函数估计中的"正交化"和传统回归中的"控制协变量"是一回事吗？

**Hints:**
- 传统回归中"控制$X$"的数学含义是什么？和DML的残差构造有何不同？
- 思考"模型误设"的后果：如果$X$与$Y$的关系不是线性的，传统回归会怎样？
- Neyman正交性的核心性质是什么？为什么它让DML对nuisance估计误差不那么敏感？

**Answer:**
**不是一回事。**

**传统回归"控制协变量"**：在模型$Y = \beta_0 + \beta_1 D + \beta_2 X + \epsilon$中，假设$Y$与$X$的关系是线性的。如果这个假设错误（如真实关系是$g(X)$而非$\beta_2 X$），则估计量有偏——遗漏的$g(X) - \beta_2 X$部分会进入误差项，与$D$相关时导致内生性。

**DML正交化**：通过构造特殊的得分函数$\psi(W; \theta, \eta) = [Y - g(X) - \theta D][D - m(X)]$，使得因果效应估计$\theta$对nuisance函数$\eta = (g, m)$的小扰动不敏感。数学上，得分函数对$\eta$的导数在真实值处为零（Neyman正交性），这意味着nuisance估计的一阶误差不会传播到$\theta$的估计中。

**关键区别**：
- 传统回归：模型设定错误直接导致因果估计有偏
- DML正交化：nuisance函数的估计误差被"二阶化"，只要收敛速度满足$o(n^{-1/4})$，$\theta$的估计仍保持$\sqrt{n}$一致性

**决策规则**：
- 如果确信$Y$与$X$、$D$与$X$都是近似线性关系 → 传统回归简洁有效
- 如果关系复杂、高维、或不确定函数形式 → DML正交化提供稳健性保障

### Q3: 为什么用同一批数据训练nuisance模型又估计因果效应，会导致严重偏误？

**Hints:**
- XGBoost在训练数据上的预测特性是什么？残差在训练集上有什么特点？
- "自己预测自己"的类比：学生既当出题人又当考生，分数能反映真实水平吗？
- 交叉拟合（cross-fitting）如何解决这个偏差？

**Answer:**
**本质：过拟合偏差（Overfitting bias）。**

XGBoost在训练数据上的预测几乎是"完美"的——它会拟合训练数据中的每一个噪声波动。当你用训练数据上的预测值$\hat{g}(X)$计算残差$Y - \hat{g}(X)$时，这个残差被人为压低了：
- 残差中不仅包含真实的$\epsilon$，还包含"未被模型拟合的噪声"
- 但由于模型过度拟合，这部分"未被拟合的噪声"被人为减小
- 残差与处理变量$D$的相关性被人为改变

具体后果：
- 残差$\tilde{Y} = Y - \hat{g}(X)$与$\tilde{D} = D - \hat{m}(X)$的协方差被人为扭曲
- 第二阶段估计的$\theta$严重偏误，通常偏向零（因为残差中的信号被"洗掉"了）

**解决方案**：[[stage-02-advanced/04-causal-inference/03-dml/02-cross-validation/交叉验证策略|交叉验证策略]]——将样本分为K份，轮流用K-1份训练nuisance模型，在剩余1份上计算残差，确保残差与模型训练相互独立。

**决策规则**：
- 任何DML分析都必须使用交叉拟合，不能用同一批数据训练和估计
- K=5是计算效率和统计效率的平衡点；小样本用K=10

### Q4: 外卖平台会员效果评估，nuisance模型该用XGBoost还是Lasso？

**Hints:**
- 两种模型的核心假设差异：Lasso要求线性，XGBoost捕捉非线性
- DML对nuisance函数的收敛速度要求是什么？（$o(n^{-1/4})$）
- 200维特征中，稀疏性假设是否成立？如何验证？

**Answer:**
**决策标准：函数形式 + 稀疏性 + 样本量。**

**如果满足以下条件，优先用Lasso**：
- 真实nuisance函数近似线性（或可通过特征工程转化为线性）
- 200维特征中只有少量（如<50个）真正重要（稀疏性假设成立）
- 样本量足够大（$s \log p / \sqrt{n} \to 0$）
- 需要可解释性（Lasso给出明确的系数）

**如果满足以下条件，优先用XGBoost**：
- 已知存在非线性关系（如年龄与消费的倒U型）
- 特征间存在复杂交互（如"活跃时段 × 品类偏好"）
- 稀疏性假设不成立（大量特征都有影响，但影响较小）
- 样本量充足（>10万），能支撑非线性模型的收敛

**大厂实践中的折中方案**：
1. 先用Lasso做基准估计（理论保证强）
2. 同时用XGBoost估计（捕捉非线性）
3. 比较两者ATE估计：如果接近（差异<10%），说明非线性不严重，用Lasso；如果差异大，说明非线性重要，用XGBoost或进一步诊断

**决策规则**：
- 如果低维线性估计和高维非线性估计接近 → 优先用简单模型（可解释性好）
- 如果差异大 → 非线性重要，用复杂模型，但需检查收敛速度是否满足要求
- 如果XGBoost的R²或AUC明显低于业务常规水平 → 改进特征工程，而非换模型

### Q5: 当倾向得分和结果回归的nuisance估计质量不一致时，DML的估计还可靠吗？

**Hints:**
- DML的"双重稳健性"（Double Robustness）定义是什么？
- 双重稳健性在什么条件下成立？两个nuisance函数都需要满足收敛速度吗？
- 如何诊断nuisance估计质量？有哪些可操作的检查步骤？

**Answer:**
**DML具有双重稳健性：只要倾向得分$m(X)$或结果回归$g(X)$之一估计正确（满足$o(n^{-1/4})$收敛），ATE估计就一致。**

**具体情形分析**：

| 情形 | 倾向得分 | 结果回归 | ATE估计可靠性 |
|---|---|---|---|
| A | 好 | 好 | 最可靠，方差最小 |
| B | 好 | 一般 | 仍然一致（双重稳健性），方差可能稍大 |
| C | 一般 | 好 | 仍然一致（双重稳健性），方差可能稍大 |
| D | 差 | 差 | 不一致，估计有偏 |

**诊断步骤**：
1. **检查nuisance模型性能**：倾向得分的AUC应>0.65（最好>0.7），结果回归的R²应显著高于零（最好>0.1）
2. **交叉验证稳定性**：重复5-10次DML流程，观察ATE估计的变异系数（CV=标准差/均值），CV>20%说明不稳定
3. **不同模型组合的稳健性检验**：
   - 组合1：Lasso估计$m(X)$ + Lasso估计$g(X)$
   - 组合2：XGBoost估计$m(X)$ + XGBoost估计$g(X)$
   - 组合3：Lasso估计$m(X)$ + XGBoost估计$g(X)$
   - 如果不同组合的ATE差异>10%，说明至少一个nuisance函数估计有问题
4. **检查重叠性（Overlap）**：倾向得分分布不应有极端值（<0.05或>0.95），否则DML在这些区域不可靠

**决策规则**：
- 如果至少一个nuisance模型性能达标 → DML估计一致，可采信
- 如果两个nuisance模型都差 → 重新审视特征工程或增加样本量
- 如果不同模型组合差异大 → 优先改进表现差的nuisance模型，或检查重叠性
- 如果重叠性不满足 → 考虑 trimming（剔除极端倾向得分样本）或换用其他因果推断方法
