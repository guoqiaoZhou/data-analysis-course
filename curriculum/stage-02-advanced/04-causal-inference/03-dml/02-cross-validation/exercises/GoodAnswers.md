# 好问练习解答 - 交叉验证策略

> 对应知识点：[[stage-02-advanced/04-causal-inference/03-dml/02-cross-validation/交叉验证策略|交叉验证策略]]

---

### Q1: 为什么DML不能简单地用全量数据训练nuisance模型，而必须做交叉拟合？

**Hints:**
- 思考"自己预测自己"的类比：学生既当出题人又当考生，分数能反映真实水平吗？
- XGBoost在训练数据上的残差有什么特点？和测试数据上的残差有何不同？
- 过拟合偏差如何传播到第二阶段的因果效应估计？

**Answer:**
**本质问题：过拟合偏差（Overfitting bias）。**

如果用同一批数据训练nuisance模型又估计因果效应：
1. XGBoost会过度拟合训练数据中的噪声，使得训练集上的预测值$\hat{g}(X)$和$\hat{m}(X)$几乎完美
2. 残差$\tilde{Y} = Y - \hat{g}(X)$被人为压低——不仅包含真实误差$\epsilon$，还包含"被模型拟合掉的噪声"
3. 残差与处理变量$D$的协方差被人为扭曲，导致第二阶段估计的$\theta$严重偏误

类比：学生既当出题人又当考生——题目都是自己做过的，分数当然好看，但不反映真实水平。

**交叉拟合的解决机制**：
- 将样本分为K份，轮流用K-1份训练nuisance模型，在剩余1份上计算残差
- 确保用于估计因果效应的残差与nuisance模型的训练过程相互独立
- 数学上：$\mathbb{E}[(Y - \hat{g}_{-k}(X))(D - \hat{m}_{-k}(X)) | \mathcal{D}_{-k}] \approx 0$

**决策规则**：
- 任何DML分析都必须使用交叉拟合，不能用同一批数据训练和估计
- 如果不用交叉拟合，即使nuisance模型"预测精度"再高，因果估计也是不可信的

### Q2: DML中的"交叉拟合"和传统机器学习中的"交叉验证"是一回事吗？

**Hints:**
- 传统交叉验证的目的是什么？（模型选择、超参数调优、性能评估）
- DML交叉拟合的目的是什么？（消除过拟合偏差、保证残差独立性）
- 如果混用两者，比如用交叉验证的"最佳模型"直接在全量数据上预测，会发生什么？

**Answer:**
**不是一回事。**

**传统交叉验证（Cross-validation）**：
- **目的**：模型选择、超参数调优、估计模型泛化性能
- **操作**：训练多个模型，比较验证集性能，选出最佳模型
- **最终模型**：通常用全量数据重新训练选定的最佳模型（或取K折模型的平均）
- **关键**：关注的是"哪个模型最好"

**DML交叉拟合（Cross-fitting）**：
- **目的**：消除"自己预测自己"的过拟合偏差，保证残差与模型训练独立
- **操作**：每折的nuisance模型只在该折外的数据上训练，残差在该折上计算，不重新训练最终模型
- **最终输出**：所有折的残差合并后用于第二阶段估计
- **关键**：关注的是"残差是否独立"

**混用的后果**：
- 如果用传统交叉验证选出"最佳XGBoost"，然后用这个模型在全量数据上预测计算残差 → 回到了"自己预测自己"的问题，过拟合偏差仍然存在
- 正确做法：交叉拟合中，每折的nuisance模型只在该折外的数据上训练，不在全量数据上重新训练

**决策规则**：
- 模型选择/调参 → 用传统交叉验证（可以在交叉拟合的每折内部做）
- 因果效应估计 → 必须用交叉拟合，确保残差独立性
- 两者可以嵌套：外层交叉拟合（保证残差独立），内层交叉验证（选择nuisance模型的超参数）

### Q3: 为什么K=5是交叉拟合的"甜点"，而不是K越大越好？

**Hints:**
- K增大时，每折的训练样本和验证样本分别怎么变化？
- 训练样本减少会导致什么问题？（nuisance模型估计质量）
- 验证样本减少会导致什么问题？（第二阶段估计的方差）
- 计算成本如何随K变化？

**Answer:**
**K=5在统计效率、计算效率和估计稳定性之间提供了最佳平衡。**

**K过大的问题**：
1. **训练样本减少**：K=20时，每折只用95%数据训练，虽然比K=5的80%多，但差异有限；更重要的是，nuisance模型（如XGBoost）在80%数据上通常已能学到主要模式
2. **验证样本减少**：K=20时，每折验证样本只有5%，第二阶段在该折上的估计方差增大；合并所有折后，总估计的方差不一定比K=5小
3. **计算成本倍增**：K-fold需要训练K次nuisance模型，K=20的计算量是K=5的4倍
4. **边际收益递减**：从K=5到K=10，统计效率提升有限；从K=10到K=20，提升更微弱

**K过小的风险**：
- K=2时，每折只用50%数据训练nuisance模型，估计质量明显下降
- 验证样本虽然大（50%），但nuisance模型的欠拟合会传播到残差

**大厂实践**：
- 样本量>10万：K=3或K=5即可
- 样本量1万-10万：K=5
- 样本量<1万：K=10，最大化训练数据
- 样本量<5000：考虑DML是否适用，或增加K到10-20

**决策规则**：
- 样本量>10万 → K=3-5
- 样本量1万-10万 → K=5
- 样本量<1万 → K=10
- 不要为了追求"更多训练数据"而盲目增大K，边际收益递减且计算成本倍增

### Q4: 时间序列因果推断中，为什么不能用随机交叉拟合，而要用滚动窗口？

**Hints:**
- 随机分割如何破坏时间顺序？举一个具体的例子
- "未来信息泄露到过去"是什么意思？在因果推断中为什么致命？
- 滚动窗口交叉验证和随机交叉拟合的操作差异是什么？

**Answer:**
**随机交叉拟合会破坏时间依赖关系，导致"未来信息泄露到过去"（look-ahead bias）。**

**问题示例**：
假设你评估2023年1-6月的推荐算法效果，随机分割把6月的数据和1月的数据混在一起。某折的训练集包含"6月用户行为模式"，验证集包含"1月用户行为"。nuisance模型在训练时"学到"了6月的模式，用来预测1月的用户——这在现实中不可能发生（1月时不可能知道6月会发生什么）。

**在因果推断中的致命性**：
- 时间序列数据通常存在趋势、季节性和自相关
- 随机分割使得nuisance模型在训练时"看到"了未来的趋势，预测过去时过于准确
- 残差被人为压低，因果效应估计偏误
- 更严重：如果处理效应本身随时间变化（如新奇效应衰减），随机混合会扭曲效应的时间结构

**滚动窗口（Rolling Window）的操作**：
1. 按时间顺序分割数据（如前80%训练，后20%验证）
2. 用历史数据训练nuisance模型，在后续时间段上计算残差
3. 可以多次滚动（如用1-3月训练，估计4月；用2-4月训练，估计5月）
4. 保持"用过去预测未来"的时间顺序

**决策规则**：
- 如果数据存在时间依赖性（用户行为、经济指标、政策效果等）→ 必须用滚动窗口或前向验证
- 如果数据是截面数据且时间不重要（如随机抽样的一次性调查）→ 随机交叉拟合可接受
- 大厂标准：任何涉及用户行为随时间变化的分析，默认使用滚动窗口

### Q5: 当交叉拟合结果在不同随机种子下差异很大时，该如何判断DML估计的可靠性？

**Hints:**
- 随机分割引入的方差来源是什么？（样本异质性、nuisance模型不稳定）
- 如何区分"随机性导致的正常波动"和"模型本身不稳定"？
- 大厂标准做法是什么？（重复次数、变异系数阈值）
- 如果变异系数>20%，问题可能出在哪里？

**Answer:**
**判断标准：变异系数（CV = 标准差 / 均值）。**

**诊断流程**：
1. **重复交叉拟合**：用5-10个不同随机种子重复整个DML流程，记录每次的ATE估计
2. **计算变异系数**：$CV = \sigma / |\mu|$，其中$\sigma$是ATE估计的标准差，$\mu$是均值
3. **判断阈值**：
   - CV < 10%：结果稳定，可采信
   - CV = 10%-20%：结果基本稳定，但需谨慎解读，建议增加样本量或简化nuisance模型
   - CV > 20%：结果不稳定，不可采信，需要诊断问题根源

**CV过高的可能原因**：
1. **样本量太小**：每折的验证样本不足，估计方差大
2. **nuisance模型过拟合**：XGBoost过于复杂，在不同训练子集上学到不同的噪声模式
3. **重叠性（Overlap）问题**：某些子群体的倾向得分接近0或1，不同随机分割时这些样本在不同折中的分布差异大
4. **处理效应异质性大**：ATE在不同子样本中差异大，说明存在显著的CATE异质性，此时报告ATE本身可能不合适
5. **异常值影响**：少数极端样本在不同折中的分布差异大

**解决方案**：
- 增加样本量（如果可行）
- 简化nuisance模型（减少XGBoost深度、增加正则化）
- 检查重叠性，剔除极端倾向得分样本（trimming）
- 如果异质性大，考虑报告CATE而非ATE
- 增加重复次数（如20次），用bootstrap估计置信区间

**决策规则**：
- CV < 10% → 结果稳定，可采信
- CV = 10%-20% → 谨慎采信，建议报告均值±标准差
- CV > 20% → 结果不稳定，需要诊断并解决问题后才能采信
- 任何DML分析报告都应包含交叉验证稳定性检验
