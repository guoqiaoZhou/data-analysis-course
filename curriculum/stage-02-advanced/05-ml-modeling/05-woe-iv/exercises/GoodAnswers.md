# 好问练习解答 - WOE-IV分箱

> 对应知识点：[[stage-02-advanced/05-ml-modeling/05-woe-iv/WOE-IV分箱|WOE-IV分箱]]

---

### Q1: 你的消费金融公司要从300+特征中筛选评分卡入模变量。为什么逻辑回归+相关系数筛选不能满足评分卡的需求？

**Hints:**
- 评分卡的核心输出是什么？是一个概率值，还是一套可人工审核的规则？
- 相关系数能告诉你"特征与目标相关"，但能告诉你"特征在哪个区间风险最高"吗？
- WOE/IV的"分箱-编码-筛选"三步分别解决了评分卡的什么需求？

**Answer:**

评分卡的核心需求是"可解释的规则体系"，而非单纯的预测精度。逻辑回归+相关系数的缺陷：

| 需求 | 相关系数+LR | WOE/IV |
|---|---|---|
| 特征离散化 | 无，LR直接处理连续值 | 有，分箱后每个区间对应一个分数 |
| 区间可解释性 | 无，系数是全局的 | 有，每箱WOE值直接反映风险倾向 |
| 预测力量化 | 相关系数不区分方向 | IV综合量化区分能力 |
| 人工审核 | 不可审核 | 可逐箱审核WOE趋势是否合理 |
| 异常值稳健 | 差，极端值影响系数估计 | 好，极端值归入边界区间 |

WOE/IV的三步价值：
1. **分箱**：将连续特征转化为业务可理解的区间（如"月收入5k-8k"）
2. **WOE编码**：每个区间赋予对数几率比，直接反映"该区间相对于整体的风险倾向"
3. **IV筛选**：量化整个特征的区分能力，用于排序和初筛

决策规则：
- 如果场景是评分卡/高可解释性要求 → 必须用WOE/IV
- 如果场景是黑盒模型/精度优先 → 可用其他方法（如XGBoost直接建模）
- 如果业务方要求"人工可审核" → 禁用纯连续特征+相关系数方案

### Q2: WOE/IV分箱和决策树分箱都能将连续特征离散化，如果你的场景不是评分卡，该用哪个？如果是评分卡，决策树分箱有什么隐患？

**Hints:**
- 决策树分箱的"最优切分"优化的是什么目标？WOE分箱优化的又是什么？
- 评分卡为什么要求WOE单调性？决策树分箱能保证单调吗？
- 文档中提到的"决策树分箱容易过拟合"在什么场景下最危险？

**Answer:**

两个业务线的分箱选择：

| 场景 | 推荐方法 | 原因 |
|---|---|---|
| 信用评分卡 | WOE/IV（卡方/等频分箱） | 与LR天然契合、IV量化预测力、可强制单调性 |
| 电商CTR预测 | 决策树分箱 或 等频+One-Hot | 不需要WOE单调性约束，树模型自动学习非线性 |
| 风控初筛（非评分卡） | 决策树分箱 | 自动发现最优切分，无需人工调参 |
| 监管报送模型 | WOE/IV（强制单调） | 监管机构要求可解释、可审核的分箱规则 |

评分卡场景中决策树分箱的隐患：
1. **不保证WOE单调性**：决策树按信息增益切分，可能产生"高-低-高"的WOE趋势，无法通过监管审核
2. **过拟合风险**：决策树分箱在训练集上"最优"，但切分点对样本波动敏感，跨时间稳定性差
3. **不可解释性**：决策树的切分边界可能落在无业务意义的数值上（如"月收入=7,342元"），审批人员无法理解

决策规则：
- 评分卡 + 监管要求 → WOE/IV，禁用决策树分箱
- 非评分卡 + 精度优先 → 决策树分箱或直接用树模型
- 评分卡 + 无监管 → WOE/IV为主，决策树分箱仅作参考

### Q3: 你将"月收入"等宽分为5档，最高档的WOE值高达2.5，IV跃升到0.45。审核时发现最高档只占总样本的0.3%，WOE估计极不稳定。如何区分"真强特征"和"样本稀疏造成的虚假高IV"？

**Hints:**
- WOE的公式中，分子和分母分别是什么？当样本稀疏时，哪个部分会剧烈波动？
- IV是各分箱WOE差异的加权和，如果某个WOE不可靠，IV是否也不可靠？
- 文档中提到的"每箱坏样本数≥30"的经验阈值从何而来？

**Answer:**

区分"真强特征"和"虚假高IV"的验证流程：

1. **样本量检查**（第一道防线）：
   - 每箱总样本数 ≥ 100（或总体的5%）
   - 每箱坏样本数 ≥ 30（风控场景的经验阈值，保证WOE估计的置信区间不过宽）
   - 不满足 → 合并该箱与相邻箱，重新计算WOE和IV

2. **WOE稳定性检查**（第二道防线）：
   - 用Bootstrap重采样（如1000次），计算每箱WOE的置信区间
   - 如果置信区间宽度 > 1.0（或跨越0点）→ WOE不稳定，不可信
   - 如果IV在Bootstrap下的标准差 > 0.05 → IV不稳定

3. **时间稳定性检查**（第三道防线）：
   - 按时间切分数据（如上半年/下半年），分别计算WOE和IV
   - 如果跨期IV差异 > 0.1 → 特征不稳定，可能是短期噪声

4. **业务逻辑检查**（第四道防线）：
   - 最高档的WOE方向是否符合业务常识？
   - 如果不符合 → 即使统计稳定，也需深入排查数据定义

决策规则：
- 样本量不足 → 合并分箱，重新计算
- Bootstrap不稳定 → 删除该特征或增加样本量
- 跨期不稳定 → 暂不纳入评分卡，观察后续表现
- 全部通过 → 确认为真强特征

### Q4: 你的评分卡要求WOE曲线单调，但"平台合作年限"的WOE呈U型。你面临选择：强制单调还是保留U型？有没有第三种方案？

**Hints:**
- 文档中的案例2是怎么处理U型WOE的？
- 评分卡要求单调性的本质原因是什么？是数学需要还是监管需要？
- 如果不入评分卡，这个特征还能以什么形式进入模型？

**Answer:**

三种方案及适用场景：

| 方案 | 操作 | 优势 | 劣势 | 适用场景 |
|---|---|---|---|---|
| 强制单调 | 合并区间使WOE单调 | 满足评分卡要求，可入模 | 丢失U型信息，降低预测力 | 监管严格的评分卡 |
| 保留U型 | 与另一特征组合分箱 | 保留全部信息，预测力最高 | 分箱复杂，解释性下降 | 非评分卡模型或内部模型 |
| 子群化建模 | 按U型的不同阶段分别建模 | 既保留信息又保持单调 | 模型数量增加，维护成本高 | 业务上U型有明确子群含义 |

文档案例2的解决方案（第三种方案的变体）：
- 将"合作年限"与"历史违约次数"组合分箱：无违约记录的供应商按年限正序WOE，有违约记录的单独分箱
- 结果：AUC提升4%，且每个分箱的业务解释清晰

决策规则：
- 如果U型有明确的业务子群含义（如"新供应商审核严格"vs"老供应商信用好"）→ 子群化建模或组合分箱
- 如果U型无明确业务解释 → 检查分箱是否不当，尝试调整边界
- 如果监管强制要求单调 → 强制合并，接受信息损失

### Q5: 你发现"最近一笔还款是否逾期"的IV高达0.8，几乎等同于用目标变量的代理预测目标。你该如何建立一套IV阈值规则，区分"因果倒置"、"强因果"和"过拟合"？

**Hints:**
- "因果倒置"和"强因果"在业务逻辑上的区别是什么？
- 文档中提到的"IV>0.5需检查是否与目标存在逻辑因果倒置"具体怎么操作？
- 过拟合导致的IV高和真实强特征的IV高，在统计上如何区分？

**Answer:**

IV阈值规则——"三层判断法"：

**第一层：IV值初筛**
- IV < 0.02 → 删除
- 0.02 ≤ IV < 0.1 → 弱特征，谨慎使用
- 0.1 ≤ IV < 0.3 → 核心候选集
- 0.3 ≤ IV < 0.5 → 强特征，需人工审核
- IV ≥ 0.5 → 强制进入第二层审核

**第二层：业务逻辑审核（针对IV≥0.5）**
- 问自己：该特征是"目标的原因"还是"目标的结果/代理"？
  - "近30天逾期次数" → 可能是原因（历史行为预测未来）→ 保留
  - "最近一笔还款是否逾期" → 几乎是目标代理（逾期=违约）→ 删除
  - "是否收到催收电话" → 是结果（违约后才催收）→ 删除
- 审核方法：画出特征与目标的时间关系图，特征必须**严格先于**目标发生

**第三层：统计稳定性审核（针对所有入模特征）**
- Bootstrap检验：IV的置信区间是否包含0.3以下？
- 时间稳定性：跨期IV变化是否 < 0.1？
- 子群稳定性：在不同产品/地区，IV是否均 > 0.1？

决策规则：
- IV ≥ 0.5 + 因果倒置 → 删除
- IV ≥ 0.5 + 强因果 + 统计稳定 → 保留（但需向监管解释）
- IV ≥ 0.5 + 统计不稳定 → 删除（过拟合噪声）
- IV 0.3-0.5 + 全部通过 → 核心强特征
