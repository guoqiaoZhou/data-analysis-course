# X-Learner

## 问题引入

假设你是一家内容平台的算法工程师，正在评估流量扶持对创作者产出的影响。你已经尝试了S-Learner（处理变量被淹没）和T-Learner（对照组样本少时欠拟合），效果都不理想。现在，你面临一个更棘手的现实：流量扶持不是随机分配的——平台优先扶持"有潜力"的创作者，而"有潜力"的定义由算法根据历史表现自动判定。这意味着处理组和对照组的样本量严重不平衡（90%获得扶持，10%未获得），而且处理分配完全由可观测特征决定。

T-Learner在这种情况下几乎不可用：对照组只有10%的样本，模型严重欠拟合。S-Learner也不行：处理变量在90%的样本中都是1，模型几乎学不到处理效应的信号。你需要一种方法，能够同时解决两个问题：①样本不平衡（处理组远多于对照组）；②处理分配非随机（由特征决定）。

你想到，可以利用"倾向得分"——估计每个创作者获得扶持的概率。对于处理组中那些"几乎不可能获得扶持"的创作者（倾向得分极低），他们的反事实状态应该更多地参考对照组模型的估计；反之，对于"几乎必然获得扶持"的创作者，应该更多地参考处理组模型的估计。通过倾向得分加权，可以让两个模型"各尽其能"。

但具体怎么操作？你需要的不仅是加权思想，而是一套完整的四阶段流程：先分别建模处理组和对照组的结果分布，再交叉估计伪处理效应，然后分别建模这些伪效应，最后用倾向得分加权合并。这个流程比S-Learner和T-Learner都复杂，但理论上能更好地利用全量样本信息，尤其在不平衡场景下。

你学过的[[stage-02-advanced/06-causal-ml/02-meta-learners/01-s-learner/S-Learner|S-Learner]]和[[stage-02-advanced/06-causal-ml/02-meta-learners/02-t-learner/T-Learner|T-Learner]]分别代表了"统一建模"和"分别建模"两个极端，而你需要一种"交叉建模+加权合并"的中间方案，在不平衡场景下取得更好的样本效率和估计精度。

这就是X-Learner的设计动机——它通过四阶段流程实现"交叉借用"，让样本量大的一方帮助样本量小的一方，同时用倾向得分加权来处理非随机分配带来的选择偏差。

## 基本原理

X-Learner 是 Meta-Learner 框架中针对**处理组与对照组样本量不平衡**场景的特化设计。其核心思想是"交叉借用"——让处理组模型去估计对照组样本的"反事实处理效应"，让对照组模型去估计处理组样本的"反事实处理效应"，再通过倾向得分加权合并。

核心思想：当处理组样本远多于对照组（或反之）时，T-Learner的对照组模型因样本不足而欠拟合。X-Learner通过"交叉估计"让样本量大的一方帮助样本量小的一方，缓解不平衡问题。

**核心公式**：

第一阶段（同T-Learner）：

$$\hat{\mu}_1(x) = \mathbb{E}[Y | X=x, W=1], \quad \hat{\mu}_0(x) = \mathbb{E}[Y | X=x, W=0]$$

第二阶段（交叉估计伪处理效应）：

$$\hat{D}_i^1 = Y_i - \hat{\mu}_0(X_i) \quad \text{（处理组样本的伪处理效应）}$$

$$\hat{D}_i^0 = \hat{\mu}_1(X_i) - Y_i \quad \text{（对照组样本的伪处理效应）}$$

第三阶段（分别建模伪处理效应）：

$$\hat{\tau}_1(x) = \mathbb{E}[\hat{D}^1 | X=x] \quad \text{（用处理组样本建模）}$$

$$\hat{\tau}_0(x) = \mathbb{E}[\hat{D}^0 | X=x] \quad \text{（用对照组样本建模）}$$

第四阶段（倾向得分加权合并）：

$$\hat{\tau}(x) = g(x) \cdot \hat{\tau}_0(x) + (1-g(x)) \cdot \hat{\tau}_1(x)$$

其中 $g(x) = \hat{P}(W=1|X=x)$ 为倾向得分估计。当处理组样本多时，$g(x)$ 大，更信任对照组模型估计的 $\hat{\tau}_0$（因为处理组样本多，$\hat{\mu}_1$ 更可靠，从而 $\hat{D}^0$ 更可靠）。

## 使用场景

| 场景 | 说明 | 示例 |
|---|---|---|
| 处理组/对照组样本严重不平衡 | 一组占90%，另一组占10% | 自然实验或观察性研究中，处理组远多于对照组 |
| 观察性研究（非随机实验） | 处理分配由特征决定，非随机 | 用户自发选择使用某功能，非被随机分配 |
| 需要利用全量样本信息 | 希望处理组和对照组数据互相借用 | 小样本场景下，T-Learner分别建模浪费数据 |
| 倾向得分可可靠估计 | 处理分配机制已知或可建模 | 知道用户是否被处理取决于哪些特征 |
| T-Learner对照组欠拟合 | T-Learner因对照组样本少而表现差 | 对照组仅数千样本，模型无法稳定估计 |
| 需要更精确的CATE估计 | 业务对CATE精度要求高，值得额外复杂度 | 精准营销场景，CATE误差直接影响ROI |

## 局限性

1. **实现复杂度高**：四阶段流程（结果模型→伪效应→效应模型→加权合并）比S/T-Learner复杂得多，代码实现和调试成本高
2. **倾向得分估计引入新误差**：$g(x)$ 的估计误差会直接传递到最终处理效应估计中，若倾向得分模型误设，X-Learner的优势可能丧失
3. **伪处理效应的噪声传递**：$\hat{D}_i^1 = Y_i - \hat{\mu}_0(X_i)$ 包含 $\hat{\mu}_0$ 的预测误差，该噪声进入第三阶段的训练数据，可能导致 $\hat{\tau}_1$ 偏误
4. **计算成本高**：需要训练4个模型（2个结果模型+2个效应模型+1个倾向得分模型），计算量是T-Learner的2倍以上
5. **小样本下优势不明显**：当总样本量<1万时，X-Learner的复杂结构可能过拟合，不如简单的S-Learner稳健

## 同类算法对比

| 方法 | 适用条件 | 优势 | 劣势 |
|---|---|---|---|
| X-Learner | 样本不平衡、观察性研究、倾向得分可靠 | 对不平衡稳健、利用全量信息、通常更精确 | 实现复杂、倾向得分误差传递、计算成本高 |
| S-Learner | 快速基线、处理效应不弱、特征维度中等 | 简单、样本效率高、实现容易 | 处理变量易被淹没、无法利用组间结构差异 |
| T-Learner | 两组分布差异大、样本量均衡 | 分别建模灵活、实现较简单 | 对照组少时欠拟合、样本效率低、方差叠加 |
| R-Learner | 需要半参数效率、有可靠的倾向得分估计 | 渐近效率最高、理论性质好 | 需要交叉拟合、实现最复杂、小样本可能差 |
| 因果森林 | 高维特征、异质性结构未知 | 自动发现异质性、有置信区间 | 计算慢、结果黑盒、小样本不稳定 |

## 行业使用经验

### 经验1：倾向得分模型需要单独调优

X-Learner的效果对倾向得分估计质量敏感。大厂实践中，倾向得分模型（通常用逻辑回归或梯度提升）需要单独调优，甚至用交叉验证选择。一个常见错误是随意用一个未调参的逻辑回归估计 $g(x)$，导致加权严重偏误。某电商平台发现，将倾向得分模型从默认逻辑回归换成调参后的LightGBM，X-Learner的CATE估计MSE降低25%。

### 经验2：样本极度不平衡时考虑"单边X-Learner"

当处理组样本是对照组的10倍以上时，标准X-Learner的第四阶段加权几乎完全依赖 $\hat{\tau}_0$（因为 $g(x) \approx 1$），此时可简化为"单边X-Learner"：仅用处理组样本估计伪效应 $\hat{D}^1$，直接用 $\hat{\tau}_1$ 作为CATE估计，跳过加权合并。某内容平台在95/5的实验设计下采用此简化，效果与完整X-Learner几乎一致，但计算量减少40%。

### 经验3：用X-Learner作为S-Learner的"升级候选"

大厂工作流通常是：先用S-Learner快速出基线→若样本不平衡严重或S-Learner效果不佳→升级到X-Learner。不建议一上来就用X-Learner，因为其复杂度和调参成本只有在S-Learner不够用时才值得。某外卖平台的增益模型开发流程明确：S-Learner AUUC < 0.6 时才触发X-Learner评估。

## 真实案例

### 案例1：（C端运营/离线分析）

**某电商平台"猜你喜欢"算法改版的效果分析**

背景：平台向95%用户推送新推荐算法（处理组），5%用户保持旧算法（对照组），评估对点击率和GMV的影响。

做法：X-Learner处理严重不平衡。第一阶段分别建模两组点击率；第二阶段处理组样本用对照组模型估计反事实点击率，得到伪处理效应；第三阶段用处理组样本建模伪效应；第四阶段倾向得分加权（因处理组占比高，加权偏向对照组模型估计的效应）。

发现：新算法对"历史浏览品类数>5 + 近7天活跃>3天"的用户处理效应+18%（显著），但对"历史浏览品类数<3 + 低频用户"处理效应-5%（显著为负）。

落地：新算法全量上线但排除"低频窄兴趣"用户群体，整体GMV提升12%，避免了负向伤害。

### 案例2：（B端运营）

**SaaS平台"客户成功经理"介入的效果评估**

背景：平台向高价值企业客户分配客户成功经理（CSM），但CSM资源有限，仅20%客户获得服务（处理组），80%无CSM（对照组）。

做法：X-Learner处理不平衡。第一阶段分别建模两组的续约率；第二阶段交叉估计伪处理效应；第三阶段分别建模；第四阶段倾向得分加权（因处理概率低，加权偏向处理组模型）。

发现：CSM对"年合同额10-50万 + 使用功能数<5 + 近30天登录<5次"的"沉睡高价值客户"处理效应最强（+35%续约率），而对"使用功能数>10 + 高频登录"的活跃客户处理效应仅+3%。

落地：CSM资源从"普惠高价值客户"改为"重点唤醒沉睡高价值客户"，续约率提升22%。

### 案例3：（离线分析/运营）

**内容平台"创作者训练营"的参与效果分析**

背景：训练营名额有限，仅15%创作者被邀请（处理组），85%未邀请（对照组）。

做法：X-Learner处理不平衡。分别建模两组的内容产出和粉丝增长，交叉估计伪处理效应，倾向得分加权合并。

发现：训练营对"粉丝数1-5万 + 历史月更<4次 + 垂类为知识科普"的创作者处理效应最强（+50%内容产出，+30%粉丝增长），对"粉丝数>20万"的头部创作者处理效应接近0。

落地：训练营名额从"按粉丝数排序"改为"按潜力模型排序"，重点邀请低频更新的腰部知识类创作者，平台整体内容生态质量提升。

## 关键词

- **X-Learner**：通过交叉估计伪处理效应、倾向得分加权合并的Meta-Learner，专为样本不平衡设计
- **交叉估计**：处理组模型估计对照组样本的反事实、对照组模型估计处理组样本的反事实，实现信息借用
- **伪处理效应（Imputed Treatment Effect）**：用反事实预测与实际观测的差值构造的"替代处理效应"，用于第三阶段训练
- **倾向得分（Propensity Score）**：给定特征条件下接受处理的概率 $P(W=1|X=x)$，用于X-Learner的加权合并
- **样本不平衡**：处理组和对照组样本量差异大，常见于观察性研究或资源受限的实验设计

## 代码示例

以下代码演示该知识点的核心概念。

```python
from sklearn.ensemble import GradientBoostingRegressor

# S-Learner: 单模型，T 作为特征
s_learner = GradientBoostingRegressor().fit(np.column_stack([X, D]), Y)
cate_s = s_learner.predict(np.column_stack([X, np.ones(n)])) -          s_learner.predict(np.column_stack([X, np.zeros(n)]))

# T-Learner: 分别训练
treat_model = GradientBoostingRegressor().fit(X[D==1], Y[D==1])
ctrl_model = GradientBoostingRegressor().fit(X[D==0], Y[D==0])
cate_t = treat_model.predict(X) - ctrl_model.predict(X)

print("S-Learner 简单但可能偏误; T-Learner 无偏但方差大")
```
