# 好问练习 - X-Learner

> 对应知识点：[[stage-02-advanced/06-causal-ml/02-meta-learners/03-x-learner/X-Learner|X-Learner]]

---

## Q1: 为什么需要X-Learner？T-Learner在样本不平衡时出了什么问题？ [L1 概念层]

你是一家内容平台的算法工程师，流量扶持不是随机分配的——平台算法优先扶持"有潜力"的创作者。结果是：90%的创作者获得了扶持，只有10%未获得。你用T-Learner分别建模处理组和对照组，但对照组只有10%的样本，模型严重欠拟合。更糟的是，处理分配完全由可观测特征决定（非随机），对照组的10%并非随机样本，而是"算法判定为无潜力"的创作者——他们的特征分布与处理组完全不同。T-Learner在这种情况下几乎不可用：对照组模型不仅样本少，而且样本有偏。你需要一种方法，能够同时解决"样本不平衡"和"处理分配非随机"两个问题。X-Learner的"交叉借用+倾向得分加权"机制如何同时应对这两个挑战？

## Q2: X-Learner和R-Learner最容易搞混，区别在哪？ [L2 混淆层]

你团队正在选型处理效应估计方法。有人提议用X-Learner："它专门解决样本不平衡，四阶段流程虽然复杂但效果好。"另一个人说："R-Learner理论性质更好，半参数效率最优，而且通过正交化分解直接估计处理效应，更优雅。"你意识到两者的区别不只是"四阶段 vs 两阶段"或"倾向得分加权 vs 残差化回归"那么简单。X-Learner的"交叉估计伪处理效应"和R-Learner的"Robinson正交化分解"在数学上有什么本质区别？为什么X-Learner更适合"样本不平衡的观察性研究"，而R-Learner更适合"大样本、需要统计效率最优"的场景？

## Q3: 初学者最常犯的错是什么？ [L3 错误层]

你刚学会X-Learner，兴奋地用它评估"创作者训练营"的效果。处理组15%（被邀请的创作者），对照组85%（未被邀请）。你按照四阶段流程：分别建模两组结果→交叉估计伪效应→分别建模伪效应→倾向得分加权合并。结果看起来很好："粉丝数1-5万 + 历史月更<4次 + 知识科普"的群体处理效应+50%。但你没有注意到：倾向得分模型用的是默认逻辑回归，未调参。而处理分配机制实际上非常复杂（由创作者历史表现、平台运营策略、外部合作等多因素决定），简单逻辑回归严重误设，倾向得分估计偏差大。由于X-Learner的第四阶段加权严重依赖倾向得分，这个误差直接传递到了最终CATE估计中。你意识到X-Learner的"复杂结构"不是"免死金牌"——它的优势建立在"倾向得分可靠估计"的前提上。为什么初学者总是忽视倾向得分模型的调优？如何诊断倾向得分是否误设？

## Q4: 这个场景该用什么？ [L4 应用层]

你是一家SaaS平台的数据科学家，需要评估"客户成功经理（CSM）介入"对企业续约率的影响。你有两个选择：
- 方案A：X-Learner（处理组20%，对照组80%）
- 方案B：因果森林（全量数据，不区分处理组/对照组比例）

你的场景特征：处理组/对照组严重不平衡（20%/80%），总样本5万企业，特征维度60维，处理分配非随机（CSM资源有限，优先分配给"高价值+低活跃"企业），需要精确估计各企业子群体的处理效应以优化CSM资源分配。为什么方案A更适合？在什么情况下你会反而选择方案B？如果倾向得分模型怎么调参都拟合不好（AUC<0.6），你的选择会改变吗？

## Q5: 多个方法冲突时怎么选？ [L5 综合层]

你是一家电商平台的算法负责人，需要为"猜你喜欢算法改版"搭建CATE估计模型。场景特征：
- 处理组95%（新算法），对照组5%（旧算法）
- 总样本200万用户，特征维度300维
- 处理效应预期中等（+10%左右）
- 需要判断"哪些用户群体对新算法响应好/坏"以决定是否全量上线
- 计算资源充足，但时间窗口紧（2周内必须决策）

你跑了四种方法：
- S-Learner：AUUC = 0.54（处理变量被淹没，几乎无效）
- T-Learner：AUUC = 0.51（对照组仅5%，严重欠拟合）
- X-Learner：AUUC = 0.63，但实现耗时1.5周
- 因果森林：AUUC = 0.61，但计算耗时2周（含CI计算）

CEO要求2周内决策，否则错过窗口期。X-Learner和因果森林都可能来不及。你需要在"效果"、"时间"、"决策风险"之间做权衡。你的决策框架是什么？如果CEO说"宁可推迟上线也不要错误决策"，你的建议会改变吗？
