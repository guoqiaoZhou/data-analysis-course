# T-Learner

## 基本原理

T-Learner（Two Learner）是 Meta-Learner 框架中"分别建模"思想的直接实现。其核心是**为处理组和对照组分别训练独立的机器学习模型**，通过两个模型的预测差异估计处理效应。

核心思想：处理组和对照组的结果分布可能结构不同（不同非线性模式、不同方差），分别建模更灵活，避免统一模型被迫用同一函数形式拟合两组数据的效率损失。

**核心公式**：

训练阶段：

$$\hat{\mu}_1(x) = \mathbb{E}[Y | X=x, W=1] \quad \text{（处理组模型）}$$

$$\hat{\mu}_0(x) = \mathbb{E}[Y | X=x, W=0] \quad \text{（对照组模型）}$$

分别用处理组样本和对照组样本训练两个独立模型。

估计阶段：

$$\hat{\tau}(x) = \hat{\mu}_1(x) - \hat{\mu}_0(x)$$

对同一特征 $x$，分别用两个模型预测，差值即为处理效应估计。

**与 S-Learner 的本质区别**：
- S-Learner：一个模型，$W$ 作为特征
- T-Learner：两个模型，$W$ 决定用哪个模型

## 使用场景

| 场景 | 说明 | 示例 |
|---|---|---|
| 处理组/对照组分布差异大 | 两组的特征分布或结果分布结构明显不同 | 处理组多为新用户（高响应），对照组多为老用户（低响应） |
| 处理组样本充足 | 处理组和对照组都有足够样本分别建模 | 50/50随机实验，两组各数万样本 |
| 结果分布结构不同 | 处理组和对照组的结果服从不同函数形式 | 处理组结果呈指数增长，对照组呈线性增长 |
| 需要组内可解释性 | 分别解释处理组和对照组的预测逻辑 | 单独分析"什么特征让处理组转化"和"什么特征让对照组转化" |
| S-Learner表现差 | S-Learner因处理变量被淹没而估计偏近于0 | 高维特征下S-Learner失效，换T-Learner分别建模 |
| 对照组模型需要特殊处理 | 对照组有独特结构需专门建模 | 对照组包含"自然流失"模式，需专门学习时间衰减 |

## 局限性

1. **对照组样本少时欠拟合**：当对照组样本量远小于处理组（如处理组90%、对照组10%），对照组模型欠拟合，其预测方差大，直接放大处理效应估计的方差
2. **样本效率低**：两组数据不能互相借用信息，总样本效率低于S-Learner。若处理效应结构简单（如全局常数），分别建模是浪费
3. **两模型差异放大方差**：$\hat{\tau}(x) = \hat{\mu}_1(x) - \hat{\mu}_0(x)$ 的方差是两模型方差之和，即使单个模型精度高，差异的精度可能很低
4. **分布外预测不可靠**：若处理组和对照组的特征分布重叠不足，模型需要在训练未覆盖的区域做外推，预测不可靠
5. **无法直接利用全量数据的联合结构**：如处理组和对照组共享某些底层模式（如共同的季节性趋势），T-Learner无法自动借用

## 同类算法对比

| 方法 | 适用条件 | 优势 | 劣势 |
|---|---|---|---|
| T-Learner | 两组分布差异大、样本量均衡、结果结构不同 | 分别建模更灵活、可捕捉组间结构差异、实现仍较简单 | 对照组少时欠拟合、样本效率低、两模型方差叠加 |
| S-Learner | 快速基线、处理效应不弱、特征维度中等 | 简单、样本效率高、实现容易 | 处理变量易被淹没、无法利用组间结构差异 |
| X-Learner | 处理组/对照组样本量不平衡、需要更精确估计 | 利用倾向得分加权、对不平衡更稳健、通常更精确 | 实现复杂、倾向得分估计引入新误差、计算成本高 |
| R-Learner | 需要半参数效率、有可靠的倾向得分估计 | 渐近效率最高、理论性质好 | 需要交叉拟合、实现最复杂、小样本表现可能差 |
| 因果森林 | 高维特征、异质性结构未知 | 自动发现异质性、有置信区间 | 计算慢、结果黑盒、小样本不稳定 |

## 行业使用经验

### 经验1：对照组样本量是最关键的约束条件

T-Learner的瓶颈几乎总是对照组模型。大厂实践中，若对照组样本量<处理组的1/5，T-Learner通常不如S-Learner或X-Learner。经验法则是：对照组样本量需≥5000（分类问题）或≥10000（回归问题）才考虑T-Learner。某电商平台在90/10的实验设计下，T-Learner的对照组模型严重欠拟合，CATE估计的方差是S-Learner的3倍。

### 经验2：两组使用相同模型类型但不同超参数

实践中不建议处理组用复杂模型、对照组用简单模型的"不对称"设计，这会使两组预测不可比。正确做法是：两组使用同一模型类型（如都用LightGBM），但允许不同超参数（对照组样本少，适当降低树深度、增大叶子节点最小样本量）。某内容平台采用此策略后，T-Learner的稳定性显著提升。

### 经验3：T-Learner是检验S-Learner的"诊断工具"

即使最终不采用T-Learner，也建议将其作为S-Learner的对比基准。若T-Learner与S-Learner的CATE估计相关性<0.5，说明处理组和对照组结构差异大，S-Learner的统一模型假设可能不成立，应优先考虑T-Learner或更复杂方法。某外卖平台通过此诊断发现，其S-Learner在新用户群体上的估计系统性地低估了20%。

## 真实案例

### 案例1：（C端运营/离线分析）

**某电商平台大促活动的用户GMV提升分析**

背景：双11期间向部分用户推送"满200减30"优惠券，处理组和对照组各50万用户。

做法：T-Learner分别用LightGBM建模处理组和对照组的30天GMV，输入用户历史行为特征。

发现：处理组模型学习到"历史大促参与次数≥2 + 收藏商品数>10"的用户在高优惠券刺激下GMV激增；对照组模型显示同一群体无优惠券时GMV平稳。两者差异显示该群体处理效应+45%。

落地：大促优惠券定向投放给"历史大促活跃 + 有明确购物意向"的用户，而非全量发放，优惠券核销率提升35%。

### 案例2：（B端运营）

**企业培训SaaS的免费试用转化分析**

背景：向企业客户开放"高级培训课程"免费试用，处理组（获得试用）与对照组（未获得）样本量接近。

做法：T-Learner分别建模两组的30天内付费转化率，输入企业规模、行业、当前使用功能数等特征。

发现：处理组模型显示"员工数50-200 + HR部门活跃 + 已使用培训模块"的企业试用后转化率高；对照组模型显示同一群体即使不试用，因HR活跃也有基础转化率。真实处理效应为+12%（而非表面上的+30%）。

落地：销售聚焦"HR活跃但尚未使用培训模块"的企业，避免对"已自发高转化"企业浪费试用资源。

### 案例3：（离线分析/运营）

**短视频平台创作者"创作工具"功能的效果评估**

背景：向部分创作者开放新版创作工具，评估对内容发布频率的影响。处理组和对照组各2万创作者。

做法：T-Learner分别建模两组的内容发布数，输入创作者粉丝数、垂类、历史发布频率等。

发现：处理组模型显示"粉丝数1-10万 + 垂类为知识科普 + 历史周更<2次"的创作者使用工具后发布数大增；对照组模型显示该群体不使用工具时发布数很低。处理效应+60%。

落地：创作工具优先向"低频更新的知识类腰部创作者"推送，平台知识类内容供给量提升40%。

## 关键词

- **Two Learner**：使用两个独立模型分别拟合处理组和对照组结果分布的Meta-Learner
- **分别建模**：处理组和对照组各自训练模型，不共享参数或结构，灵活性高但样本效率低
- **模型差异方差叠加**：T-Learner的处理效应估计方差等于两模型方差之和，$Var(\hat{\tau}) = Var(\hat{\mu}_1) + Var(\hat{\mu}_0)$
- **分布外预测**：当预测点的特征分布超出训练数据覆盖范围时，模型预测不可靠
- **欠拟合**：模型复杂度不足，未能捕捉数据中的真实模式，常见于小样本场景
