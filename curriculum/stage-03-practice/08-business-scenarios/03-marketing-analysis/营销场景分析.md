# 营销场景分析

> **本文约定**：pp = 百分点（绝对变化），% = 百分比（相对变化）。例如：从10%提升至12%，绝对提升为+2pp，相对提升为+20%。

## 问题引入

某电商平台运营总监在双11复盘会上宣布："本次大促GMV同比增长80%，活动非常成功！"

数据分析师小赵举手："但对照组（未参与大促的用户）同期GMV也增长了60%。实际增量可能只有20个百分点。"

会议室瞬间安静。运营总监问："什么叫对照组？"

小赵解释："就是同期没有收到促销信息的用户，他们的GMV也涨了60%——因为整个行业都在做双11。我们的80%里，大部分是行业红利，不是活动本身带来的。"

更尴尬的是，深入分析后发现：那20%的"增量"里，还有很大一部分是"需求前置"——用户把原本下周要买的提前到了这周。也就是说，活动并没有创造新需求，只是挪动了时间。

运营总监追问："那ROI到底多少？"

小赵翻着报告："我们投入了500万补贴，增量GMV大约..."

这个场景揭示了营销分析的核心困境：**表面上的增长数字往往极具误导性**。没有对照组的ROI是自欺欺人，不考虑"需求前置"的增量是虚高的，单一归因模型（如末次点击）会扭曲渠道价值判断。

如何准确评估营销活动的真实增量效果，避免"把大盘增长当成活动效果"？这就是营销场景分析要解决的问题。

## 基本原理

营销场景分析的核心公式：

**营销ROI**
```
营销ROI = 增量收入 / 营销成本 × 100%
```

> **定义说明**："增量收入"指营销活动带来的、超出自然水平的额外收入（已扣除自然量）。若增量收入为毛收入（未扣除成本），则公式为：`(增量收入 - 营销成本) / 营销成本 × 100%`。分析时必须明确所用定义。

**营销转化率**
```
营销转化率 = 营销触达后产生目标行为的人数 / 营销触达总人数
获客成本(CAC) = 营销总成本 / 新增用户数
```

**促销弹性**
- 价格弹性 = 销量变化率 / 价格变化率
- 促销增量GMV = 促销期间GMV - 预估无促销GMV（需通过对照组或时间序列预测）

**核心思想**：营销分析的本质是"投入产出比"评估。关键在于区分"增量"与"自然量"——营销带来的真正价值是"如果没有营销活动，原本不会发生的那部分交易"。

## 使用场景

| 场景 | 说明 | 示例 |
|---|---|---|
| 营销活动ROI评估 | 评估单次营销活动的投入产出 | 618大促ROI为1:8，但增量GMV仅占30%，其余为需求前置 |
| 渠道效果对比 | 对比不同营销渠道的效率 | 信息流广告CAC为50元，搜索广告CAC为80元，但后者LTV更高 |
| 促销策略优化 | 分析不同促销形式的转化效果 | 满减活动转化率15%，折扣券转化率8%，但后者客单价更高 |
| 营销人群定向评估 | 评估不同人群的营销响应率 | 高活跃用户营销响应率5%，沉默用户响应率2%但唤醒价值高 |
| 营销频次控制 | 避免过度营销导致用户疲劳 | 同一用户月均收到5条以上营销推送，转化率下降40% |

## 局限性

1. 营销增量 attribution 困难，用户可能同时受多个渠道影响，难以准确拆分贡献
2. 营销活动往往伴随季节性、竞争环境变化等外部因素，难以建立纯净对照组
3. 促销可能引发"需求前置"——用户将未来需求提前到促销期，实际增量被高估
4. 营销分析周期较长（需观察用户全生命周期），短期数据可能误导决策
5. 不同品类、不同用户群体的营销弹性差异大，统一分析可能掩盖细分规律

## 同类算法对比

| 方法 | 适用条件 | 优势 | 劣势 |
|---|---|---|---|
| 末次点击归因 | 简单快速评估 | 计算简单，易于理解 | 忽略其他渠道贡献，高估末次渠道 |
| 多触点归因 | 用户旅程复杂场景 | 更公平分配贡献 | 模型复杂，数据要求高 |
| 增量测试（Lift Test） | 精准评估营销增量 | 直接测量因果效应 | 需要对照组，样本量要求大 |
| 营销组合模型(MMM) | 长期宏观预算分配 | 考虑外部因素，适合战略层 | 数据粒度粗，无法到用户级 |
| Uplift模型 | 精准营销人群选择 | 识别" persuadable "用户 | 建模复杂，需要实验数据 |

## 行业使用经验

### 经验1：必须建立"自然对照组"评估增量——我们曾把大盘增长当成活动效果

每次营销活动预留5%-10%用户不触达作为对照组。某电商平台发现某次大促表面GMV增长200%，但对照组同期也增长150%（受行业大势影响），实际增量仅50%。**没有对照组的ROI计算是自欺欺人。即使业务方说"这次全员参与"，也要想办法找到"近似对照组"（如未曝光用户、其他区域用户）。**

### 经验2：区分"获客营销"与"留存营销"的评估周期——我们曾把短信召回ROI高估3倍

获客营销看7日/30日留存，留存营销看复购周期内的回购率。某平台将两者混为一谈，导致对短信召回ROI高估3倍——召回用户的"复购"实际上是把原本会在自然周期内发生的购买提前了。**规范：营销分析必须在报告首页声明"评估周期"和"增量定义"。不同营销类型的周期不同，混用会扭曲决策。**

### 经验3：警惕"补贴依赖"的沉默成本——连续6个月大促后，非促销期GMV暴跌40%

某零售平台连续6个月每月大促，发现非促销期GMV下降40%，用户价格敏感度永久性提升。**建议：建立"促销期/非促销期GMV比值"监控指标。若比值持续扩大，说明用户已被"训练"成等促销才购买。此时继续加大促销是饮鸩止渴，应转向产品价值建设。**

### 经验4：我们曾犯的错误——用"末次点击归因"把全部功劳归给搜索广告

某平台分析渠道贡献，末次点击归因显示搜索广告贡献60%成交。但多触点归因（MTA）分析后发现：用户平均接触3.2个渠道后才成交，信息流广告的"种草"作用被严重低估。据此将预算从搜索转向信息流，整体CAC降低25%。**教训：单一归因模型会扭曲渠道价值判断。至少对比末次点击、首次点击、线性归因三种模型，看结论是否稳健。**

## 真实案例

### 案例1：（C端运营/离线分析）
某电商平台"双11"大促复盘，表面GMV同比增长80%（相对提升）。但分析师通过对照组（未参与大促的用户）发现，对照组同期GMV也增长60%（相对提升，受行业大盘带动）。实际绝对增量=+20pp（80%-60%），大量为需求前置。次年调整策略，从"满减驱动"转向"新品首发驱动"，增量占比提升至45% [P1已修复]。

### 案例2：（B端运营）
某SaaS企业分析年度营销预算分配，发现70%预算投入信息流广告（CAC 800元），仅30%投入老客户推荐计划（CAC 200元）。但推荐客户的12个月留存率（85%）显著高于信息流客户（60%）。调整预算比例为5:5后，整体客户生命周期价值（LTV）提升30%，虽然首月新增客户数下降15%，但年度营收增长22%。

### 案例3：（离线分析/运营）
某外卖平台分析"满减券"与"直降"两种促销形式。数据显示满减券使用率60%，直降使用率90%。但深入分析发现，满减券用户中有40%是为了凑单而加购非必要商品，退单率高达25%；直降用户退单率仅8%。运营团队据此调整策略，对高频用户用满减（提升客单价），对新用户用直降（降低决策门槛），整体营销ROI提升18%。

## 关键词

- **营销ROI**：营销投入与产出的比值，核心评估指标
- **增量GMV**：营销活动带来的超出自然水平的额外GMV
- **CAC（获客成本）**：获取一个新用户的平均成本
- **LTV（用户生命周期价值）**：单个用户在全生命周期内的总价值
- **需求前置**：用户将未来购买计划提前到促销期，造成增量高估
- **归因模型**：将成交贡献分配给不同营销渠道的逻辑框架
- **Uplift模型**：预测用户对营销干预的响应程度，识别"可被说服"的用户
- **营销弹性**：营销投入变化对产出的敏感程度
