### Q1: 补贴后时效下降7分钟，为什么不能确定是补贴效果？

**Hints:**
- 思考"观测数据"与"实验数据"的根本区别：没有随机分组，任何对比都可能是混杂的结果。
- 回顾"混杂因子"概念：哪些因素可能同时影响"是否补贴"和"履约时效"？

**Answer:**
没有随机实验的情况下，"补贴前后对比"是观测数据比较，无法排除混杂干扰。

- **可能的混杂因子**：天气（补贴期天气好，骑手配送更快）、新调度系统（同期上线，独立改善时效）、季节性（淡季订单少，时效自然改善）、区域差异（补贴先在配送密度高的区域试点）。这些因素与"补贴"同时变化，导致"时效改善"的功劳无法归属。
- **日常分析方法的局限**：对比均值只能回答"发生了什么"（时效从35→28），无法回答"为什么"（是补贴还是其他因素）。相关性分析更危险——它可能发现"补贴投入"与"时效改善"强相关，但相关≠因果。
- **因果推断的必要性**：在无法做A/B测试时（如政策评估、全量活动），需要用因果推断方法从观测数据中分离因果效应。Python生态的DoWhy + EconML提供了"识别-估计-验证"的完整流程。
- **决策规则**：如果可以做随机实验（如分城市随机试点），优先做A/B测试；如果无法随机化（如全量政策），必须用因果推断工具，并在报告中明确声明"未观测混杂"风险和敏感性分析结果。

### Q2: DoWhy和CausalML为什么通常组合使用而非二选一？

**Hints:**
- 思考两个工具的核心定位：DoWhy解决"因果正确性"，CausalML解决"估计精度和工程化"。
- 回顾"识别-估计-验证"流程：哪个工具覆盖哪个环节？

**Answer:**
两个工具的分工明确，互补而非替代。

- **DoWhy的定位**：负责"因果正确性"——因果图建模（识别混杂路径）、refutation测试（验证结果稳健性）。它回答"这个效应是否可信"。
- **EconML（与DoWhy组合）**：负责"估计精度"——DML、因果森林等高级估计器，在高维控制变量下获得一致估计。它回答"效应量是多少"。
- **CausalML的定位**：负责"工程化落地"——T-learner、S-learner、X-learner直接输出sklearn模型，可通过PMML/ONNX嵌入实时打分服务。它回答"如何在线预测Uplift"。
- **决策规则**：
  - **如果场景是离线因果效应估计（如评估活动效果）**，用DoWhy + EconML组合。DoWhy确保因果逻辑自洽，EconML确保估计精度。
  - **如果场景是实时Uplift建模（如精准营销人群圈选）**，用CausalML独立。它与sklearn生态无缝衔接，模型轻量、推理快，适合生产环境。
  - **如果场景需要完整流程（识别+估计+验证+工程化）**，用DoWhy做离线验证确认方法有效后，将核心逻辑用CausalML重写并嵌入生产链路。

### Q3: 为什么没有refutation测试，结果通常会被打回？

**Hints:**
- 思考"统计显著性"与"因果稳健性"的区别：p<0.05是否意味着结果可信？
- 回顾"安慰剂检验"的作用：如果处理变量是随机的，效应应该消失——这个逻辑如何验证结果？

**Answer:**
Refutation测试是因果推断的"安全网"——它检验结果是否对假设违反敏感，而传统统计检验做不到这一点。

- **错误根源**：分析师认为"因果图建模 + DML估计 + p<0.05"就足够了。但p值只检验"随机误差"，不检验"系统性偏差"（如未观测混杂、模型误设）。
- **Refutation的三类测试**：
  - **安慰剂检验（Placebo Treatment）**：随机打乱处理变量，如果"随机处理"也产生显著效应，说明原结果可能是巧合。
  - **随机混杂注入（Random Common Cause）**：添加一个随机生成的混杂因子，如果效应大幅变化，说明原结果对混杂敏感。
  - **子样本验证（Data Subsets Validation）**：在不同子样本上重复估计，如果效应不稳定，说明结果可能依赖特定数据分布。
- **为什么会被打回**：没有refutation，评审无法判断"+3.2分钟"是真实因果效应还是模型巧合。补做refutation后发现结果对未观测混杂敏感，结论从"显著"降级为"趋势性"——这个降级是科学的，避免了过度自信的决策。
- **决策规则**：任何因果推断分析报告必须附带至少两类refutation测试。没有refutation的结果只能作为"探索性发现"，不能作为"决策依据"。

### Q4: 改版和审核策略同期上线，如何分离两个因素的贡献？

**Hints:**
- 思考"多重处理"（Multiple Treatments）的因果推断挑战：两个干预同时发生，如何识别各自的效应？
- 回顾"因果图"的作用：能否将"内容审核"建模为混杂因子而非处理变量？

**Answer:**
分离多重处理的贡献需要结合因果图设计和准实验方法，单一方法无法解决。

- **方法1：因果图 + 后门调整（DoWhy）**：将"内容审核"建模为混杂因子（它同时影响"改版"决策和"消费时长"），通过控制"内容审核强度"来分离"改版"的净效应。前提是"内容审核"的度量可量化（如审核通过率、内容下架率）。
- **方法2：双重差分（DID）**：利用"改版"和"审核策略"在不同城市/用户群体上线的时间差。如果改版先上线、审核策略后上线，可以用"交错DID"（Staggered DID）分别估计两个政策的效应。这需要处理时间异质性（不同单元处理时间不同）。
- **方法3：合成控制法（SCM）**：如果改版只在部分城市上线，可以为改版城市构建"合成对照组"（加权组合未改版城市），使其在改版前与处理城市高度匹配。审核策略的影响被"合成"过程部分吸收，改版的净效应更纯净。
- **决策规则**：
  - 如果两个干预有"时间差"或"空间差"，优先用DID或SCM。
  - 如果两个干预完全同步、无差异，只能用因果图控制可观测混杂，并在报告中明确声明"无法分离两个干预的独立贡献，估计的是联合效应"。
  - 无论用哪种方法，都必须做refutation测试，检验结果对"未观测混杂"和"模型选择"的敏感性。

### Q5: "无未观测混杂"假设无法验证，因果推断结果还有意义吗？

**Hints:**
- 思考"科学方法"的本质：物理学定律也依赖不可直接验证的假设（如"自然规律不变"），为什么仍然有用？
- 回顾"敏感性分析"的作用：如果未观测混杂存在，结果在什么范围内仍然稳健？

**Answer:**
假设不可验证不等于结果无意义，关键在于"透明度"和"敏感性分析"——让决策者知道"在什么条件下结论成立"。

- **因果推断 vs A/B测试的取舍**：
  - **如果可以做随机实验**，优先做A/B测试。随机化消除了观测和未观测混杂，是因果推断的金标准。
  - **如果无法随机化**（如政策评估、历史事件、伦理限制），因果推断是"次优但可行"的方案。此时必须明确声明假设，并用敏感性分析量化假设违反的影响。
- **敏感性分析的方法**：
  - **偏倚因子分析**：假设存在一个未观测混杂，其强度为已观测混杂的X%，计算效应估计在什么X%范围内仍然显著。
  - **Rosenbaum边界**：在倾向得分匹配后，检验"未观测混杂需要多强才能推翻结论"。
  - **Oster检验**：估计"未观测混杂需要解释多少结果变异才能消除效应"。
- **决策规则**：
  - 因果推断报告必须包含"假设声明"（明确列出所有识别假设）和"敏感性分析"（展示假设违反对结论的影响）。
  - 如果敏感性分析显示"中等强度的未观测混杂即可推翻结论"，则结论只能作为"参考性发现"。
  - 如果敏感性分析显示"极强的未观测混杂才能推翻结论"，则结论可信度较高，可作为"倾向性决策依据"。
  - 最终决策应结合"因果推断结果 + 业务常识 + 其他证据"，而非单一依赖统计输出。
