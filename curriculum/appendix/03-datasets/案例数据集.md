## 项目01：电商大促因果推断

### 数据集概述

- **数据集名称**：电商618大促用户行为模拟数据
- **记录数**：10,000 条用户记录
- **核心教学点**：选择偏差、异质性处理效应、面板数据（DID）
- **生成脚本**：`project-01-ecommerce-causal/data/generate_data.py`

### 字段说明

| 字段名 | 类型 | 取值范围 | 说明 |
|--------|------|----------|------|
| `user_id` | int | 10001-20000 | 用户唯一标识 |
| `treatment` | int | 0/1 | 是否领取优惠券（1=领取，0=未领取） |
| `gmv` | float | ≥0 | 618期间的成交总额（元） |
| `age` | int | 18-65 | 用户年龄 |
| `gender` | int | 0/1 | 性别（0=女，1=男） |
| `city_tier` | int | 1-4 | 城市等级（1=一线，4=四线） |
| `membership_level` | int | 0-3 | 会员等级（0=普通，3=钻石） |
| `historical_gmv` | float | ≥0 | 过去一年历史消费总额（元） |
| `historical_orders` | int | ≥0 | 过去一年历史订单数 |
| `activity_score` | float | 0-100 | 用户活跃度评分 |
| `coupon_sensitivity` | float | 0-1 | 优惠券敏感度（越高越容易被影响） |

### 数据生成逻辑

1. **用户画像生成**：年龄、性别、城市等级等基础属性从真实分布中抽样
2. **历史行为**：历史GMV和订单数与用户画像相关（一线城市、高会员等级用户消费更高）
3. **优惠券发放**（选择偏差设计）：`treatment=1`的用户并非随机选择。平台倾向于向历史消费较高、活跃度较高、优惠券敏感度较高的用户发放优惠券
4. **GMV生成**：618期间的GMV由基础消费倾向 + 优惠券效应（异质性）+ 随机扰动决定
5. **异质性效应**：优惠券对高敏感度用户效果更大，高消费用户存在天花板效应

### 教学要点

- **选择偏差**：处理组的历史GMV显著高于对照组，直接比较会高估效应
- **异质性效应**：需用CATE方法（如因果森林）识别不同用户群体的响应差异
- **面板数据扩展**：可构造pre_gmv/post_gmv用于DID分析

## 项目02：APP新功能AB实验

### 数据集概述

- **数据集名称**：APP新功能AB实验数据
- **记录数**：约20,000条（用户×日期）
- **核心教学点**：AA检验、CUPED方差缩减、后分层分析、多重检验
- **生成脚本**：`project-02-app-experiment/data/generate_data.py`

### 字段说明（实验主数据）

| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| `user_id` | string | 用户唯一标识 | `U_10001` |
| `group` | string | 实验分组：`control`或`treatment` | `treatment` |
| `dau_date` | date | 实验日期 | `2024-01-15` |
| `dwell_time` | float | 当日停留时长（秒），右偏分布 | 1865.3 |
| `ctr` | float | 点击率（点击次数/曝光次数） | 0.085 |
| `retention_next_day` | int | 次日是否留存：0=否，1=是 | 1 |
| `historical_dwell_time` | float | 历史平均停留时长（CUPED用） | 1723.5 |
| `user_type` | string | 用户类型：`new`或`old` | `old` |
| `device_type` | string | 设备类型：`iOS`或`Android` | `iOS` |

### 字段说明（用户属性数据）

| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| `user_id` | string | 用户唯一标识 | `U_10001` |
| `age_group` | string | 年龄段：`18-24`, `25-34`, `35-44`, `45+` | `25-34` |
| `gender` | string | 性别：`M`, `F`, `unknown` | `M` |
| `city_tier` | string | 城市级别：`tier1`, `tier2`, `tier3`, `tier4+` | `tier1` |
| `content_preference` | string | 内容偏好 | `entertainment` |

### 数据质量问题设计

| 问题类型 | 出现比例 | 处理策略 |
|----------|----------|----------|
| 缺失值（dwell_time） | ~2% | 用历史均值填充 |
| 缺失值（ctr） | ~3% | 用0填充 |
| 缺失值（historical_dwell_time） | ~5% | 用全局均值填充 |
| 异常值（dwell_time > 7200秒） | 少量 | 用99分位数截断 |
| 异常值（ctr > 0.5） | 少量 | 用0.5截断 |

### 教学要点

- **AA检验**：实验前验证两组基线是否平衡
- **CUPED**：用historical_dwell_time作为协变量缩减方差
- **后分层**：按user_type、device_type分层提升估计精度
- **多重检验**：同时监控5个指标时的FWER控制

## 项目03：骑手配送费动态定价

### 数据集概述

- **数据集名称**：骑手配送费动态定价模拟数据
- **记录数**：10,000+条（区域×时段×日期面板数据）
- **时间跨度**：3个月（90天）
- **核心教学点**：选择偏差、异质性CATE、因果森林、Meta-Learners
- **生成脚本**：`project-03-rider-pricing/data/generate_data.py`

### 标识字段

| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| `order_id` | string | 订单唯一标识 | `ORD_20240115_001` |
| `region_id` | categorical | 区域编码（10个区域） | `R01`, `R02` |
| `hour` | int | 时段（0-23） | `8`, `12`, `18` |
| `date` | date | 日期 | `2024-01-15` |
| `day_of_week` | int | 星期几（0=周一, 6=周日） | `0`, `5` |
| `is_weekend` | bool | 是否周末 | `True`, `False` |

### 处理变量（Treatment）

| 字段名 | 类型 | 说明 | 取值范围 |
|--------|------|------|----------|
| `price` | float | 配送费（元） | 3.0 - 15.0 |
| `price_level` | categorical | 价格档位 | `low`, `medium`, `high` |
| `price_increase` | float | 相比基准价的涨幅（元） | 0.0 - 8.0 |
| `price_increase_pct` | float | 相比基准价的涨幅比例 | 0.0 - 1.5 |

**价格生成机制**：
- 基准价：根据区域×时段的"历史均价"确定
- 动态加价：雨天/高峰/偏远区域自动加价
- **关键设计**：价格不是随机的——存在选择偏差

### 结果变量（Outcomes）

| 字段名 | 类型 | 说明 | 取值范围 |
|--------|------|------|----------|
| `order_volume` | int | 时段订单量 | 50 - 500 |
| `ontime_rate` | float | 准时率 | 0.6 - 0.95 |
| `rider_supply` | int | 在线骑手数 | 20 - 200 |
| `gmv` | float | 时段GMV（万元） | 1.0 - 50.0 |
| `user_complaint_rate` | float | 用户投诉率 | 0.01 - 0.15 |

### 协变量（Covariates）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `weather` | categorical | 天气：`sunny`, `rainy`, `snowy` |
| `temperature` | float | 温度（摄氏度） |
| `humidity` | float | 湿度（%） |
| `wind_speed` | float | 风速（km/h） |
| `region_type` | categorical | 区域类型：`cbd`, `residential`, `suburb`, `university` |
| `region_density` | float | 区域订单密度（0.1-1.0） |
| `avg_income` | float | 区域平均收入（万元/年） |
| `competitor_price` | float | 竞品配送费（元） |
| `holiday` | bool | 是否节假日 |
| `promotion_active` | bool | 是否有营销活动 |
| `historical_ontime_rate` | float | 历史准时率（7天平均） |
| `historical_rider_supply` | int | 历史骑手供给（7天平均） |
| `historical_order_volume` | int | 历史订单量（7天平均） |

### 构造字段（教学用）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `true_cate` | float | 真实条件平均处理效应（用于评估估计精度） |
| `selection_bias` | float | 选择偏差强度（教学用） |
| `price_elasticity` | float | 真实价格弹性（区域×时段特异） |

### 异质性效应设计

```
真实CATE = f(weather, region_type, hour)

雨天CBD高峰: CATE = +0.15  (加价有效，骑手供给弹性高)
雨天郊区平峰: CATE = -0.05  (加价无效，用户流失效应 > 骑手效应)
晴天住宅高峰: CATE = +0.08  (加价中等有效)
雪天任何区域: CATE = +0.20  (加价高度有效)
```

### 选择偏差设计

```
价格不是随机的：
  price = baseline + weather_premium + peak_premium + region_premium + noise

雨天本来订单就少（不是因为价格）：
  order_volume = baseline - weather_penalty + price_effect + ...

简单回归会混淆：
  "雨天价格高，订单少" → 错误归因于价格
```

### 脏数据设计

| 问题类型 | 出现比例 | 处理方式 |
|----------|----------|----------|
| 缺失值 | 5% | `price`和`rider_supply`字段 |
| 异常值 | 2% | `order_volume`出现极端值 |
| 重复记录 | 1% | 同一订单ID出现多次 |
| 逻辑错误 | 1% | 雨天但温度35°C（数据录入错误） |

### 数据文件说明

| 文件名 | 说明 | 用途 |
|--------|------|------|
| `raw/rider_pricing_raw.csv` | 原始数据（含脏数据） | 数据清洗练习 |
| `raw/rider_pricing_clean.csv` | 清洗后数据 | 直接分析用 |
| `processed/rider_pricing_processed.csv` | 特征工程后数据 | 建模用 |
| `processed/rider_pricing_train.csv` | 训练集 | 模型训练 |
| `processed/rider_pricing_test.csv` | 测试集 | 模型验证 |

### 教学要点

- **选择偏差**：价格由天气/高峰/区域决定，简单回归会混淆天气效应和价格效应
- **异质性CATE**：不同区域×时段×天气组合下，价格效应方向和大小都不同
- **因果森林**：自动发现"高响应子群"，无需预设交互项
- **Meta-Learners**：对比S/T/X/R-Learner在不同场景下的表现

## 数据生成规范

所有项目数据均通过Python脚本生成，遵循以下规范：

1. **随机种子固定**：`np.random.seed(42)`或`random.seed(42)`
2. **分布合理**：基于真实业务场景设计数据生成逻辑
3. **问题内置**：选择偏差、异质性、脏数据等教学点通过生成逻辑实现
4. **可复现**：同一脚本多次运行结果一致
