# 数据字典

## 原始数据 (raw/ecommerce_618.csv)

| 字段名 | 类型 | 取值范围 | 说明 |
|--------|------|----------|------|
| user_id | int | 1-10000 | 用户唯一标识 |
| treatment | int | 0/1 | 是否领取优惠券（1=领取，0=未领取） |
| gmv | float | ≥0 | 618期间的成交总额（元） |
| age | int | 18-65 | 用户年龄 |
| gender | int | 0/1 | 性别（0=女，1=男） |
| city_tier | int | 1-4 | 城市等级（1=一线，4=四线） |
| membership_level | int | 1-4 | 会员等级（1=普通，4=钻石） |
| historical_gmv | float | ≥0 | 过去一年历史消费总额（元） |
| historical_orders | int | ≥0 | 过去一年历史订单数 |
| activity_score | float | 0-100 | 用户活跃度评分 |
| coupon_sensitivity | float | 0-1 | 优惠券敏感度（越高越容易被优惠券影响） |
| pre_gmv | float | ≥0 | 618前一个月的GMV（元） |
| post_gmv | float | ≥0 | 618后一个月的GMV（元） |
| device_type | str | iOS/Android/Web | 主要使用设备 |
| channel | str | app/mini_program/web | 主要访问渠道 |

## 生成逻辑说明

数据通过以下逻辑模拟真实场景：

1. **用户画像生成**：年龄、性别、城市等级等基础属性从真实分布中抽样
2. **历史行为**：历史GMV和订单数与用户画像相关（一线城市、高会员等级用户消费更高）
3. **优惠券发放**：treatment=1的用户并非随机选择。平台倾向于向：
   - 历史消费较高的用户
   - 活跃度较高的用户
   - 优惠券敏感度较高的用户
   发放优惠券。这造成了**选择偏差**。
4. **GMV生成**：618期间的GMV由以下因素决定：
   - 基础消费倾向（由历史行为决定）
   - 优惠券效应（treatment=1的用户获得额外GMV提升，但效应大小因人而异）
   - 随机扰动
5. **面板数据**：pre_gmv和post_gmv用于DID分析，模拟618前后的消费变化

## 数据特点

- 样本量：10,000条
- 处理组占比：约40%
- 选择偏差：处理组的历史GMV显著高于对照组
- 异质性效应：优惠券对高敏感度用户效果更大

## 使用建议

- 原始数据位于 `data/raw/ecommerce_618.csv`
- 清洗后的数据可保存至 `data/processed/`
- 注意检查缺失值和异常值
