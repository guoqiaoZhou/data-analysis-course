## 案例1：电商优惠券效果评估 ✅ 已完成

**真实场景**：618大促，平台给部分用户发券，老板问"这券到底有没有用？"

**为什么复杂**：
- 不是随机发券（高消费用户更可能领券）
- 不能简单比较"领券vs没领券"的GMV
- 需要回答：如果**强制**给所有用户发券，GMV会提升多少？

**必须用**：
1. **潜在结果框架** — 明确"因果"vs"相关"的鸿沟
2. **PSM匹配方法** — 构造"相似"的对照组，模拟随机化
3. **重叠假设检查** — 确保有可比的对照组（Common Support）
4. **DID平行趋势** — 如果有前后数据，验证"没券的话趋势是否相同"
5. **敏感性分析** — 回答"如果有未观测混杂，结论还成立吗？"

**不用**：
- 合成控制（没有"对照城市"）
- 因果森林（不需要区域异质性）
- 随机实验（已经发生了，不是设计阶段）

**交付物**：因果推断分析报告（含稳健性检验）

## 案例2：APP新功能AB实验

**真实场景**：内容APP想上线新推荐算法，用实验验证效果

**为什么复杂**：
- 多个指标（CTR、停留时长、次日留存）
- 用户行为方差大（需要CUPED降方差）
- 实验前需要确认分组平衡（AA检验）
- 多个实验组（A/B/C/D）时多重检验问题

**必须用**：
1. **实验设计** — 样本量计算、实验周期、最小检测效应
2. **随机化与分流** — 如何分配用户（用户ID哈希？设备ID？）
3. **Pre-AA检验** — 实验前确认两组在关键指标上无差异
4. **CUPED方差缩减** — 用历史CTR作为协变量，提高检验力
5. **多重检验问题** — 3个指标、5个实验组，怎么控制FWER？
6. **后分层分析** — 新用户和老用户效果是否不同？
7. **分析决策** — 显著但效应小，上不上线？不显著但趋势好，再测一周？

**不用**：
- PSM（已经随机了，不需要匹配）
- DID（没有前后对比设计）
- 因果森林（不需要异质性，只需要平均效应）

**交付物**：实验设计文档 + 分析报告 + 上线决策建议

## 案例3：用户流失预警模型

**真实场景**：SaaS平台用户流失严重，需要提前识别高风险用户并干预

**为什么复杂**：
- 这是**预测问题**，不是因果问题（不需要知道"为什么流失"，只需要知道"谁会流失"）
- 需要模型可解释（业务方要知道"为什么这个用户高风险"）
- 特征可能有数百个，需要筛选和转换
- 模型需要稳定（不同时间段表现一致）

**必须用**：
1. **XGBoost** — 处理非线性关系，自动处理缺失值
2. **WOE-IV分箱** — 将连续特征离散化，增强可解释性
3. **SHAP全局重要性** — 回答"哪些特征对预测最重要"
4. **SHAP局部解释** — 回答"这个具体用户为什么被判定高风险"
5. **Bootstrap** — 估计模型性能的置信区间，评估稳定性
6. **交叉验证策略** — 时间序列交叉验证（不能用随机K折，因为数据有时间顺序）

**不用**：
- 任何因果推断方法（这是预测，不是因果）
- DID/PSM/合成控制（没有处理变量）
- 固定效应（用户-level模型，不需要控制用户固有特征）

**交付物**：流失预测模型 + 特征解释报告 + 高风险用户名单

## 案例4：城市限行政策评估

**真实场景**：某城市实施机动车限行，评估对空气质量的因果效应

**为什么复杂**：
- 不能随机限行（伦理+政治）
- 空气质量受很多因素影响（季节、天气、工业排放）
- 需要构造"如果没有限行，空气质量会怎样"的反事实
- 政策实施时间明确，适合时间序列方法

**必须用**：
1. **DID平行趋势** — 比较限行前后的空气质量变化，验证"平行趋势"假设
2. **事件研究法** — 看限行效应是即时出现还是滞后，是否持续
3. **合成控制法** — 用其他城市加权构造"虚拟对照城市"
4. **固定效应模型** — 控制城市固有特征（如地理位置、产业结构）
5. **功效分析** — 限行前后数据是否足够检测出效应？

**不用**：
- PSM（没有时间维度，匹配后仍可能有时间趋势差异）
- 随机实验（不可能随机限行）
- XGBoost/SHAP（这是因果推断，不是预测）

**交付物**：政策评估报告（含稳健性检验）

## 案例5：骑手配送费动态定价

**真实场景**：外卖平台动态调整配送费，雨天/高峰/偏远区域自动加价，平衡订单量和骑手供给

**为什么复杂**：
- 效应高度异质：雨天加价vs晴天加价，效应完全不同
- 不能随机定价（用户体验+商业逻辑）
- 需要知道"每个区域×每个时段"的最优价格
- 模型可能误设（价格效应非线性）

**必须用**：
1. **因果森林分裂准则** — 自动发现异质性子群（如"雨天+偏远区域"效应特别大）
2. **因果森林置信区间** — 每个子群的效应估计精度如何？
3. **S-Learner / T-Learner / X-Learner** — 对比不同建模策略，选最优
4. **R-Learner** — 高效估计，减少模型误设偏差
5. **双重稳健学习** — 倾向得分或结果模型有一个对就行
6. **DML（高维控制）** — 控制数百个协变量（区域、时段、天气、历史订单等）
7. **Nuisance函数估计** — 用ML估计倾向得分和结果模型
8. **交叉验证策略** — 选择最优Meta-Learner

**不用**：
- AB实验（不可能随机定价，用户体验差）
- PSM（高维匹配困难，区域×时段组合太多）
- DID（没有明确的时间断点）

**交付物**：动态定价策略建议（按区域×时段细分）

## 案例6：金融风控评分卡升级

**真实场景**：银行信用卡审批，从传统评分卡升级到ML模型，降低坏账率

**为什么复杂**：
- 监管要求模型可解释（不能用黑盒）
- 需要稳定性（不同季度表现一致）
- 特征可能有数百个（征信、消费、社交等）
- 坏账率很低（1-3%），类别不平衡

**必须用**：
1. **Logistic回归** — 传统评分卡基础，可解释性强
2. **WOE-IV分箱** — 特征离散化，增强可解释性，符合监管要求
3. **XGBoost** — 新模型，处理非线性，提升预测精度
4. **GBDT-LR** — 组合方案：GBDT自动特征组合 + LR可解释
5. **SHAP全局重要性** — 回答"哪些特征对违约预测最重要"
6. **Bootstrap** — 评估模型稳定性（不同季度AUC的置信区间）
7. **卡方检验** — 验证特征与违约的关联性（监管要求）

**不用**：
- 因果推断（这是预测问题，不是估计"某因素导致违约"）
- DID/PSM（没有处理变量）
- 因果森林（不需要异质性，只需要平均违约概率）

**交付物**：评分卡模型 + 特征解释文档 + 监管报备材料

## 案例7：内容平台创作者激励计划

**真实场景**：平台推出"创作者激励计划"（流量扶持+现金奖励），评估对内容产量的影响

**为什么复杂**：
- 优质创作者更可能入选（选择偏差）
- 效应可能滞后（入选后1个月才开始增产）
- 激励可能挤出非激励内容（总产能不变，只是转移）
- 需要知道"对平台整体DAU的影响"（外部性）

**必须用**：
1. **潜在结果框架** — 明确"入选vs不入选"的产能差异
2. **PSM匹配方法** — 构造可比对照创作者
3. **DID平行趋势** — 入选前后产能变化
4. **事件研究法** — 动态效应：效应是即时还是滞后？持续多久？
5. **合成控制法** — 用未入选创作者加权构造"虚拟对照组"
6. **固定效应模型** — 控制创作者固有特征（如创作领域、粉丝基数）
7. **功效分析** — 入选创作者数量是否足够检测出效应？

**不用**：
- XGBoost/SHAP（这是因果推断，不是预测）
- 随机实验（不可能随机排除优质创作者）

**交付物**：激励计划效果评估报告

## 案例8：线下门店选址决策

**真实场景**：连锁品牌开新店，用数据辅助选址

**为什么复杂**：
- 需要预测新店销售额（回归问题）
- 也需要预测盈亏概率（分类问题）
- 选址特征复杂（人流、竞争、租金、交通等）
- 模型需要可解释（老板要知道"为什么选这里"）

**必须用**：
1. **OLS回归** — 预测新店销售额
2. **Logistic回归** — 预测盈亏概率
3. **Bootstrap** — 预测置信区间（"销售额预计100-150万"）
4. **固定效应模型** — 控制城市固有特征（如消费水平、竞争密度）
5. **交叉验证策略** — 留一城市交叉验证（不能用随机K折，城市间差异大）
6. **SHAP局部解释** — 回答"这个选址为什么预测销售额高"

**不用**：
- 因果推断（这是预测问题，不是估计"某因素导致销售额"）
- DID/PSM（没有处理变量）
- XGBoost（需要可解释，OLS+Logistic更合适）

**交付物**：选址评估报告（含预测区间和解释）

## 设计原则验证

1. **真实性**：每个案例来自真实业务场景，不是为覆盖知识点而硬凑
2. **必要性**：每个知识点的出现是"不得不"而非"可以放"
3. **不堆砌**：案例1不用合成控制（没有对照城市），案例3不用因果推断（是预测问题）
4. **渐进性**：从简单（案例1：PSM+DID）到复杂（案例5：因果森林+Meta-Learner）
