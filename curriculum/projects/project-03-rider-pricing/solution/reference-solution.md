## 一、问题理解（20分）

### 1.1 业务背景理解

**核心矛盾**：
- 运营团队：加价→骑手增加→准时率提升（忽略用户流失）
- 财务团队：加价→订单量下降→GMV受损（忽略骑手供给弹性）
- CEO：要"每个区域×每个时段的最优价格"（忽略效应异质性）

**本质问题**：定价效应高度异质——不存在"万能最优价"，只有"因地制宜的最优价"。

### 1.2 关键挑战识别

| 挑战 | 说明 | 应对 |
|------|------|------|
| 不能随机定价 | 用户体验+商业逻辑 | 观察性因果推断 |
| 效应高度异质 | 雨天vs晴天完全不同 | 因果森林发现子群 |
| 模型可能误设 | 价格效应非线性 | 双重稳健+DML |
| 选择偏差 | 雨天本来订单就少 | DML正交化 |

## 二、方法选择（20分）

### 2.1 方法选择决策树

```
Q1: 能随机实验吗？ → 不能
Q2: 处理变量连续？ → 是
Q3: 需要异质性？ → 是
Q4: 协变量维度高？ → 是
结论：DML + Meta-Learners + 因果森林
```

### 2.2 为什么不用其他方法？

| 方法 | 问题 | 为什么不用 |
|------|------|-----------|
| 简单回归 | 忽略选择偏差 | 会混淆天气效应和价格效应 |
| AB实验 | 不可行 | 不能随机定价，用户体验差 |
| PSM | 高维匹配困难 | 区域×时段=240种组合，匹配质量差 |
| 传统回归调整 | 维度灾难 | 协变量太多，模型不稳定 |

### 2.3 核心方法详解

**DML**：
- 用ML估计nuisance函数（倾向得分、结果模型）
- 样本分割避免过拟合
- 正交化减少模型误设影响

**Meta-Learners**：
- S-Learner：简单，但可能忽略处理效应
- T-Learner：灵活，但方差大
- X-Learner：利用倾向得分，适合处理组少
- R-Learner：残差化，对误设更稳健

**因果森林**：
- 自动发现异质性子群
- 不需要预设交互项
- 提供置信区间

**双重稳健**：
- 倾向得分和结果模型双重保险
- 只要一个对，估计就一致

## 五、建模分析（20分）

### 5.1 DML估计

```python
# 核心代码逻辑
def dml_estimator(X, T, Y, n_splits=5):
    kf = KFold(n_splits=n_splits, shuffle=True, random_state=42)
    thetas = []
    
    for train_idx, test_idx in kf.split(X):
        # 估计nuisance函数
        model_y.fit(X_train, Y_train)
        model_t.fit(X_train, T_train)
        
        # 预测
        Y_pred = model_y.predict(X_test)
        T_pred = model_t.predict(X_test)
        
        # 残差
        Y_residual = Y_test - Y_pred
        T_residual = T_test - T_pred
        
        # 估计因果效应
        theta = np.mean(T_residual * Y_residual) / np.mean(T_residual ** 2)
        thetas.append(theta)
    
    return np.mean(thetas), np.std(thetas)
```
