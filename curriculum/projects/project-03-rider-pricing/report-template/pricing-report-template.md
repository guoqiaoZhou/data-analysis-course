## 二、业务背景（Situation）

### 2.1 问题描述

外卖平台面临骑手供需失衡：
- 雨天/高峰：订单激增，骑手不足，准时率下降
- 偏远区域：骑手不愿接单，订单积压
- 平峰时段：骑手闲置，运力浪费

### 2.2 各方立场

| 团队 | 立场 | 数据验证 |
|------|------|----------|
| 运营 | 加价→骑手增加→准时率提升 | 部分正确 |
| 财务 | 加价→订单量下降→GMV受损 | 部分正确 |
| CEO | 要"最优价格" | 需要差异化 |

### 2.3 核心矛盾

**本质问题**：定价效应不是单一的，而是高度异质的。

不存在一个"万能最优价"，只有"因地制宜的最优价"。

## 三、分析方法（Methodology）

### 3.1 方法选择

| 方法 | 用途 | 为什么选它 |
|------|------|-----------|
| DML | 处理高维协变量 + 选择偏差 | 传统方法维度灾难 |
| S/T/X/R-Learner | 对比不同建模策略 | 不同场景最优策略不同 |
| 因果森林 | 自动发现异质性子群 | 不需要预设交互项 |
| 双重稳健 | 模型误设保护 | 双重保险 |
| 交叉验证 | 选择最优Learner | 数据驱动 |

### 3.2 为什么不用其他方法？

| 方法 | 为什么不用 | 问题 |
|------|----------|------|
| 简单回归 | 混淆因果 | 忽略选择偏差 |
| AB实验 | 不可行 | 不能随机定价 |
| PSM | 高维困难 | 区域×时段组合太多 |

### 3.3 数据说明

- **数据来源**：平台全量订单数据
- **数据量**：20,000+条（清洗后）
- **时间跨度**：3个月
- **数据结构**：面板数据（区域×时段×日期）

## 五、业务洞察（Insights）

### 5.1 对运营团队的回应

**运营说**："雨天必须加价30%，这是行业惯例！"

**数据回应**：
- 加价20%和30%效果一样
- 多出来的10%只是给用户添堵
- **建议**：雨天加价20%即可

### 5.2 对财务团队的回应

**财务说**："所有区域都必须有正ROI！"

**数据回应**：
- 偏远区域加价反而降低GMV
- 用户流失效应 > 骑手供给效应
- **建议**：接受"差异化ROI"

### 5.3 对CEO的回应

**CEO说**："1周内出结论！"

**数据回应**：
- 1周：方向性结论（哪些区域有效）
- 2周：精确估计（最优价格）
- **建议**：分阶段汇报

## 九、方法论说明（Methodology Appendix）

### 9.1 因果推断框架

```
核心问题：价格对订单量的因果效应

识别策略：
1. 用DML处理高维协变量
2. 用样本分割避免过拟合
3. 用正交化减少模型误设影响
4. 用双重稳健提供双重保险

估计策略：
1. S/T/X/R-Learner对比
2. 因果森林发现异质性
3. 交叉验证选择最优策略
```

### 9.2 稳健性检验

|------|------|------|

| 改变样本范围 | 通过 | 结论对样本选择不敏感 |
| 安慰剂检验 | 通过 | 方法可靠 |

### 9.3 局限性

2. **时间局限**：3个月数据，未覆盖季节性变化
3. **场景局限**：未考虑极端天气（台风、暴雨等）
